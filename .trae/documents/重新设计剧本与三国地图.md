# 重新设计剧本与三国地图

## 概述

用户希望：
1. 移除旧的"geo 地图"相关剧本（历代帝王等），保留随机剧本
2. 重新设计一个三国剧本，使用三国时期诸侯作为势力
3. 重新设计地图系统，不再是方正的矩形地图，而是更接近真实地形的不规则地图
4. 河流需要考虑不可跨越的障碍特性

## 当前状态分析

### 地图系统
- **当前地图生成**：`src/core/map/generator.ts` 使用 d3-delaunay 在矩形 `bounds` 内撒点生成 Voronoi 图
- **地形赋值**：`src/core/map/terrain.ts` 使用 simplex-noise 给每个州赋 elevation/moisture，分类为 plain/forest/mountain/desert/river
- **河流**：当前河流只是把某些州标记为 `terrain: 'river'`，没有"不可跨越"的逻辑
- **地图边界**：矩形，通过 `edgeFalloff` 让边缘海拔偏低形成"岛屿"效果
- **渲染**：PixiJS 通过 `MapCanvas.tsx` 渲染 Voronoi 多边形

### 剧本系统
- **当前剧本**（`src/core/scenario/presets/`）：
  - `random.ts` — 随机剧本（保留）
  - `three-kingdoms.ts` — 三国鼎立（蜀汉/曹魏/东吴，已存在但简单）
  - `historical-emperors.ts` — 历代帝王（移除）
  - `custom.ts` — 自定义（保留）
- **势力出生**：支持 `random:n|s|e|w|nw|ne|sw|se|center` 九宫格方位指令
- **剧本应用**：`apply.ts` 解析 spawn 指令，分配空州

### 关键约束
- TDD 中明确提到"三国剧本"需要"固定中国地图"（9.1 节）
- 当前没有不规则地图形状生成能力
- 当前没有河流不可跨越的逻辑

## 决策

### 地图形状方案
**采用噪声裁剪 + 边缘遮罩方案**，不引入 GeoJSON 或外部数据：
1. 在地图生成阶段，用噪声函数生成一个"陆地掩膜"（land mask）
2. 掩膜外的 Voronoi 州标记为"海洋"（不可占领、不可通行）
3. 掩膜内的州正常参与游戏
4. 通过调整噪声参数可以生成不同的陆地形状（大陆、半岛、群岛等）
5. 三国剧本可以使用特定的 seed 生成类似中国地形的轮廓

### 河流不可跨越方案
1. 河流州标记为 `terrain: 'river'`
2. 扩张逻辑中，河流州作为"天然屏障"：
   - 扩张时跨越河流的州需要额外判定（降低胜率或增加难度）
   - 或者：河流州本身不可被占领，只能绕行
3. 采用**方案 A**：河流州可被占领但扩张跨越河流边界时降低胜率（更符合历史，河流是屏障但不是绝对不可逾越）

### 剧本调整
1. **移除** `historical-emperors.ts`（历代帝王）
2. **保留** `random.ts`、`custom.ts`
3. **增强** `three-kingdoms.ts`：
   - 势力：蜀汉（刘备）、曹魏（曹操）、东吴（孙权）
   - 增加更多三国诸侯可选：袁绍、吕布、董卓、马腾、刘表、张鲁等
   - 出生方位更精确（西/北/东南）

## 实施步骤

### Phase 1: 地图不规则形状

#### 1.1 新增陆地掩膜生成
**文件**: `src/core/map/generator.ts`

在 `generateMap` 函数中，Voronoi 生成后、地形赋值前：
- 用 simplex-noise 生成陆地掩膜（land mask）
- 每个州的 centroid 采样掩膜值
- 低于阈值的州标记为"海洋"（新增 `TerrainKind: 'ocean'`）
- 海洋州不参与剧本分配、不参与扩张

#### 1.2 扩展地形类型
**文件**: `src/core/map/types.ts`

```typescript
export type TerrainKind = 'plain' | 'forest' | 'mountain' | 'desert' | 'river' | 'ocean';
```

#### 1.3 地形赋值适配海洋
**文件**: `src/core/map/terrain.ts`

- `assignTerrains` 函数接收陆地掩膜参数
- 海洋州直接赋 `terrain: 'ocean'`，不计算 elevation/moisture
- 海洋州颜色用深蓝色

#### 1.4 渲染适配海洋
**文件**: `src/ui/canvas/MapCanvas.tsx`

- `TERRAIN_COLOR` 新增 ocean 颜色（深蓝 `0x1a3050`）
- `TERRAIN_LABEL` 新增 ocean 标签（"海洋"）
- 海洋州不响应点击/悬停（或点击无效果）

#### 1.5 剧本分配跳过海洋
**文件**: `src/core/scenario/apply.ts`

- `pickRegionForDirective` 跳过 `terrain === 'ocean'` 的州

#### 1.6 扩张逻辑跳过海洋
**文件**: `src/core/sim/expansion.ts`

- 扩张时跳过海洋州作为源或目标

## 假设与决策

1. **不引入 GeoJSON**：用户提到"通过脚本创建"，意味着程序化生成而非加载真实地理数据。使用噪声掩膜方案更轻量。
2. **海洋州完全不可通行**：简化实现，海洋州不参与任何游戏逻辑。后续如需"海岸线"可扩展。
3. **河流是减速屏障而非绝对障碍**：符合历史（渡河困难但不是不可能），也避免游戏逻辑过于复杂。
4. **保留随机剧本为默认**：用户明确说"保留随机剧本"。
5. **三国剧本保留 3 家核心 + 新增 8 家群雄变体**：给用户两种选择。
