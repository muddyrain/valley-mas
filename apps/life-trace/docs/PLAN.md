## 产品定位

Life Trace 是一个 AI 主导的个人生活计划、踪迹记录和家庭库存管理 PWA。它把天气、提醒、日程、打卡、图片记录、家庭库存和 AI 建议组合成一个生活闭环：用户先计划生活，再完成计划，并沉淀为可回看的生活踪迹。

核心闭环：

```text
今日天气 -> 创建生活计划 -> 完成计划 -> 自动生成生活踪迹 -> 在踪迹流中回看
```

家庭库存闭环：

```text
拍照/手动记录商品 -> AI 识别和补全信息 -> 用户编辑确认 -> 入库 -> 到期提醒/使用记录 -> 生成生活踪迹
```

## 下一步优先级

### P0：透明封面真机验收

状态：待验收。

目标：

- 验证 remove.bg 主路在真实商品图上的成功率、耗时和错误提示。
- 验证 IMG.LY 兜底首次模型加载、手机内存占用、失败兜底和坏图过滤。
- 确认透明封面只作为 `thumbnailUrl` 候选，不覆盖真实图片。

验收样例：

- 桌上饮料瓶。
- 食品包装袋或纸盒。
- 透明瓶身。
- 复杂桌面背景。
- remove.bg 未配置、额度耗尽或请求失败。
- 弱网和手机低内存场景。

通过标准：

- 云端失败时能自动进入本地兜底或给出可读失败提示。
- 本地生成结果不会出现明显全透明、全黑、空图或未分离背景却直接保存。
- 用户仍可选择真实图或裁剪图做封面。

### P1：家务与保养 MVP

状态：未开始。

目标：

- 管理周期性家务、设备保养和生活维护事项。
- 与提醒、家庭空间、计划、成就和踪迹联动，但不混入 Pantry 库存。

页面建议：

- 路由：`/maintenance`、`/maintenance/:id`。
- 入口：我的页、Today 风险提醒、家庭空间详情、Inbox 转化。
- 首屏：今日待处理、即将到期、家庭分工、最近完成。

任务类型：

- 家务：洗床单、清理冰箱、拖地、整理桌面。
- 设备：换滤芯、空调清洁、路由器/服务器检查、车辆保养。
- 证件/事务：证件到期、域名续费、保修截止、体检预约。
- 家庭共享：可选分配给 household 成员。

MVP 能力：

- 创建周期性保养任务。
- 支持一次性、每天、每周、每月、每季度、每年、自定义间隔。
- 完成后自动计算下次时间。
- 可选择生成踪迹和成就进度。
- 家庭空间内可查看共享任务，个人任务默认私有。

建议数据结构：

- `life_trace_maintenance_tasks`：任务基础信息、周期、下次到期、所属 household、负责人可选、状态。
- `life_trace_maintenance_logs`：完成记录、备注、图片、完成人。

验收标准：

- 用户能创建一个“每 30 天清理一次冰箱”的任务。
- 完成后能看到下一次到期时间。
- 家庭任务和个人任务权限清晰。
- 不和 Pantry 到期提醒混用；Pantry 管物品，保养管周期性动作。

## 页面与模块范围

- Today 首页采用“柔和生活日记 / 今日节奏”首屏：问候与天气、四格今日节奏、快速记录条、时间线式今日计划、生活卡片和今日小结优先呈现。

- Inbox、轻账本等快捷入口。

### 计划

- 计划筛选：全部、今天、周末、已提醒。
- 内置计划类型：电影、吃饭、运动、阅读、聚会、普通事项。
- 计划卡片展示提醒状态。
- 计划卡片和详情页支持加入系统日历。
- 创建计划 Drawer。
- 创建和编辑计划时支持选择地点库中的常用地点，并保留地点文本字段兼容旧数据。
- 计划详情使用 `/plans/:planId` 二级页面承载，进入详情时隐藏底部 Tab。
- 计划列表、创建、完成和删除通过服务端同步。
- 完成计划后生成踪迹，并通过服务端同步。

### AI

- Life AI 指挥中心。
- 智能体对话、历史消息、历史分组、搜索和动作追踪。
- 快捷操作：生成今日建议、创建计划、智能菜谱、分析图片、生成踪迹、每周回顾。
- 图片生活分析：用于电影海报、美食照片、生活照片，结果可生成计划或踪迹。
- 商品拍照分析：从 AI 页跳转到 `/ai/photo-item-analysis`，走 AI 拍照识别，最终入 Pantry 库存。
- 最近商品识别：AI 页展示草稿优先摘要，完整记录进入 `/ai/photo-item-history`。
- 今日穿搭和拍照识别衣物：从 AI 页进入衣橱相关能力。
- 智能菜谱：基于当前 Pantry 家庭空间的食品库存生成临期优先菜谱。

### Pantry

- 库存列表、筛选、分页和摘要。
- 库存详情页：真实图、透明封面、基础字段、所属家庭空间、到期状态、提醒规则、条码、备注和操作时间线。
- 创建、编辑、删除库存物品。
- 标记已用完、已丢弃、部分使用、部分丢弃。
- 库存到期、临期、正常状态派生。
- 库存图片、分类、标签、数量、单位、保质期、购买日期、存放位置、备注。
- 家庭空间切换：个人空间和共享家庭空间。
- AI 拍照分析入库入口、识别历史和草稿恢复。

### 家庭空间

- `Household` 是统一空间抽象：`personal` 和 `shared`。
- 每个用户始终拥有一个个人空间，个人空间不可解散。
- 用户加入共享家庭后，个人空间仍然存在。
- 库存永远属于某个 household，不直接属于某个 user。
- 用户加入家庭时，不自动迁移个人库存。
- 家庭解散后，家庭库存保留在原 household 上，进入只读归档态。
- 普通成员可以退出家庭。
- owner 如果要退出且仍有其他成员，需要先转移 owner。
- owner 是最后一位成员时，应走解散家庭流程。

已落地能力：

- 通用 household model。
- personal household 自动创建和回填支持。
- household API：列表、详情、创建、成员、邀请、加入、退出、转移 owner、解散。
- Pantry 按 household 查询和权限校验。
- Pantry 家庭空间切换和家庭管理入口。
- 邀请码复制、重新生成、撤销。
- 共享家庭详情、成员列表和归档家庭只读展示。
- 个人库存移动或复制到共享家庭库存，支持重复预览、合并数量或保留两条。

### 衣橱 / 穿搭

- 使用 `/closet`、`/closet/items/:itemId`、`/closet/outfits/:outfitId` 和 `/ai/photo-clothing-analysis`。
- 入口：Today 今日穿搭卡、AI 页今日穿搭和拍照识别衣物、我的页“衣橱与穿搭”。
- 个人衣橱默认仅本人可见。
- 共享家庭空间采用主动共享衣物池，成员只看到主动共享的衣物和穿搭。
- 支持手动创建、编辑、删除衣物。
- 支持照片生成 AI 衣物草稿，用户确认字段和所属空间后才入库。
- 支持规则优先、AI 增强的穿搭建议。
- 支持保存穿搭、查看穿搭详情和标记已穿。
- 标记已穿后生成 `source=穿搭` 的生活踪迹。
- 共享穿搭只能引用共享衣物，不允许把未共享的个人衣物暴露到家庭空间。

### 踪迹

- 完成后的生活记录流。
- 图片、标签、心情、AI 回忆文案。
- 来源可以是计划、打卡、库存状态、书影音、穿搭或手动记录。
- 踪迹详情使用 `/traces/:traceId` 二级页面承载，进入详情时隐藏底部 Tab。
- 当前通过服务端同步，localStorage 只保留非核心兜底状态。
- 创建和编辑踪迹时支持选择地点库中的常用地点。
- 带地点的计划和踪迹会自动沉淀到地点库。

### 地点库

- 使用 `/places` 和 `/places/:placeId`，不接入地图 SDK。
- 地点从计划和踪迹的地点字段自动归集，按用户隔离，同名地点按规范化名称复用。
- 地点列表展示收藏地点、想去地点、常去地点和最近出现地点。
- 地点详情展示记录次数、首次/最近出现时间、备注，以及关联计划和踪迹。
- 地点支持手动创建、收藏、改名、归档和 JSON 导出。
- Pantry 的冷藏、厨房、储物柜等库存存放位置不进入地点库。

### Inbox

- 使用 `/inbox`。
- 用于快速收集暂时不知道归到哪里的生活碎片。
- 支持文本、链接、单图和标签。
- 支持按状态、类型和关键词筛选。
- 条目状态：`inbox`、`converted`、`archived`。
- 支持手动触发 AI 整理，生成标题、摘要、标签和建议去向。
- 支持手动转为计划草稿或踪迹草稿，用户保存后原 Inbox 条目标记为已转化。
- AI 建议不会自动创建计划或踪迹。

### 书影音日记

- 使用 `/media-diary` 和 `/media-diary/:id`。
- 类型：书籍、电影、剧集、动漫、音乐。
- 字段：标题、原名、作者/导演/艺人、年份、封面图、开始日期、完成日期、状态、评分、短评、摘录、标签。
- 支持手动创建、编辑、删除。
- 支持 AI 辅助补全作者/导演/艺人、年份、标签和短评草稿。
- 保存记录时自动生成 `source=书影音` 的踪迹。
- 编辑记录时同步更新关联踪迹，删除记录时保留历史踪迹。

### 轻账本

- 使用 `/ledger`。
- 做生活型轻记录，不做完整财务系统。
- 字段：金额、币种、分类、支付时间、商家/地点、备注、图片。
- 方向：支出、收入、退款、转账备注。
- 分类：吃饭、交通、购物、书影音、订阅、家用、礼物、医疗、其他。
- 支持手动记一笔、编辑、删除和分页列表。
- 支持本月支出、收入、退款、结余和支出分类统计。
- 支持从 Inbox 文本和 Pantry 商品生成账目草稿。
- 支持 AI chat 记账：用户明确说“记账/记一笔/花了/收入/退款”等并提供金额时直接入账；缺金额时只追问金额。
- 所有草稿必须用户确认后才入账。

### 我的

- 个人资料、天气城市、工作时间、每日简报和通勤方式。
- 我的首页为总览和入口，提醒细项不堆叠在首页。

- 提醒设置页：天气风险、每日简报预览、计划提醒、计划提前提醒、工作日规则、勿扰时间、系统通知、服务端推送绑定和 Pantry 默认提醒。
- AI 个性化保留在我的首页，不和提醒规则混放。
- 库存与家庭：默认家庭空间、家庭管理入口。
- 衣橱与穿搭入口。
- 每日打卡项管理。
- 应用安装、更新检查、分享入口和版本信息。

### Admin 后台

- Life Trace 管理页提供数据概览、用户概览和统一记录查询。

## AI 与图片能力边界

### 通用 AI 规则

- AI 能力只做整理、摘要、建议和草稿生成。
- 所有写入计划、踪迹、账目、衣橱、地点、库存或保养记录的动作必须由用户确认。
- AI 未配置或失败时返回可解释错误，不阻断手动流程。
- 前端不保存真实 AI 密钥，不直连模型。
- 药品、保健品、婴幼儿食品等敏感品类只做库存记录建议，不输出医疗或安全结论。

### 商品拍照分析

接口：

```text
POST /api/v1/life-trace/ai/pantry-photo-analysis
```

请求只走 AI 商品识别，不再提供独立文字识别拍照模式；AI 返回商品草稿时可包含生产日期、到期日、保质期天数和日期线索。只有识别到到期日，或同时识别到生产日期和保质期时才自动回填到期相关字段；仅识别到“保质期 9 个月”这类无日期锚点线索时只提示用户确认，不写入表单日期字段；没有日期线索时仍可作为普通商品拍照录入。

AI 返回草稿字段：

- `name`：商品名称。
- `category`：分类，例如 食材、饮料、零食、药品、日用品、护肤、清洁、其他。
- `brand`：品牌，可为空。
- `spec`：规格，例如 `500ml`、`2kg`、`12片`。
- `quantity`：数量，默认 `1`。
- `unit`：单位，例如 件、瓶、盒、袋、个、包。
- `storageLocation`：建议存放位置。
- `expiresAt`：保质期日期，可为空。
- `purchaseDate`：购买日期，默认今天或空。
- `tags`：标签。
- `confidence`：整体识别置信度。
- `warnings`：识别不确定、保质期不可见、需要用户确认等提示。
- `cropBox`：建议裁剪框，保留服务端响应兼容。
- `source` / `model` / `modelTag`：本次识别使用的 AI provider、模型名和展示标签，例如 `Gemini · gemini-2.5-flash` 或 `ARK · ep-...`。

用户可编辑字段：

- 商品名称。
- 分类。
- 数量和单位。
- 购买日期。
- 保质期。
- 存放位置。
- 所属家庭空间。
- 默认提醒规则。
- 备注。
- AI 缩略图。
- 商品标签：主分类保持食品、日用品、药品、宠物、其他等稳定大类；标签用于记录冷冻、零食、调料等可自定义细分类。

### 透明封面策略

- 入库时保存原图 URL 到真实图片字段。
- AI 缩略图字段默认留空。
- 列表卡片优先展示真实图片。
- 透明封面只在用户手动点击“生成透明封面”时触发。
- 主路走服务端 remove.bg，API key 只放在服务端环境变量 `REMOVE_BG_API_KEY` 中。
- remove.bg 配置缺失、额度耗尽或请求失败时，自动使用浏览器本地 IMG.LY 模型兜底。
- 透明封面结果作为 `thumbnailUrl` 候选，不覆盖真实图片。
- 如果未来希望完全离线且不依赖 IMG.LY 默认模型 CDN，再单独评估自托管模型文件、`rembg` / RMBG sidecar 或其他 provider。

## 技术基线

- 前端：React + Vite + TypeScript。
- UI：Tailwind CSS + shadcn/ui dark-first 风格。
- 图标：lucide-react。
- 状态：Zustand + localStorage 兜底持久化；核心数据通过 Go API 同步。
- PWA：manifest + service worker。
- 后端：复用 Valley MAS Go 服务，Life Trace 能力集中在 `server/internal/lifetrace`。
- AI：服务端统一代理模型调用，不在前端直连模型；Pantry 拍照分析优先使用 Gemini 视觉模型，其他 Life Trace AI 默认复用现有 ARK / Life Trace 文本配置。
- 图片：复用现有上传接口和服务端图片分析能力；用户手动上传/拍照选择的生活图片在前端先压缩，再上传到云端。

依赖策略：

- 默认不新增第三方依赖。
- 新增依赖前必须说明要解决的问题、主要取舍和替代方案，并询问用户是否采用。
- AI 拍照分析复用浏览器原生 `MediaDevices.getUserMedia`、`canvas`、现有上传接口；Pantry 商品识别优先走 Gemini 视觉模型，未配置时回退现有 ARK 视觉/文本模型。
- 条码识别首版使用浏览器原生 `BarcodeDetector`；不可用、识别失败或格式不支持时，提供手动输入编码入口。
- IMG.LY 兜底使用较小的 `isnet_quint8` 模型，优先保证能返回透明 PNG，不追求最高质量。

## PWA 能力

- 已提供 `manifest.webmanifest`。
- 已生成 192px / 512px 应用图标。
- 已注册 service worker，并缓存应用壳、manifest 和图标。
- manifest 与图标采用网络优先刷新，降低应用名称或图标变更后继续使用旧缓存的概率。
- service worker 导航请求遇到托管平台 404 时会回退到应用壳，避免用户在子路由点击“立即更新”后看到 `404: NOT_FOUND`。
- Vercel 部署配置已补 SPA rewrite，直接打开或分享子路由时统一返回 `index.html`。
- 全局应用壳提供安装和新版本提示。
- 我的页提供 PWA 控制台，展示安装状态、离线缓存状态、更新状态、分享能力，并提供安装、检查更新、立即更新和分享应用入口。
- 我的页底部展示当前应用版本，格式为 `v产品版本+构建标识`，例如 `v0.1.0+abc1234`。
- 生产构建会把当前 commit 短 SHA 注入 service worker，让每次提交后的部署都能被浏览器识别为可更新版本。
- 应用分享优先使用系统分享面板，失败时回退为复制应用链接。
- 本地系统通知桥接已实现；服务端 Web Push 采用外部 Cron 扫描兜底。

## 验收与校验

功能或代码变更时按范围运行：

- Life Trace 前端类型检查：`pnpm --filter @valley/life-trace exec tsc --noEmit`。
- Life Trace 前端测试：`pnpm --filter @valley/life-trace exec vitest run`。
- Go 服务改动：`cd server && go test ./...` 或更小范围的 `cd server && go test ./internal/lifetrace`。
- 中文文案、Markdown、协作规则或 skill 改动：`python3 .agents/skills/encoding-guard/scripts/check_mojibake.py <相关文件>`。

前端、游戏或可视交互改动不使用 Playwright 自动验收。涉及移动端布局、拍照、PWA 通知、真实模型输出或真机权限时，最终回复必须给出清晰的手动验收路径。
