# 生活迹 Life Trace MVP 计划

## 产品定位

生活迹 Life Trace 是一个 AI 主导的个人生活计划与踪迹记录应用。它把天气、提醒、日程、打卡、图片记录和 AI 建议组合成一个生活闭环：用户先计划生活，再完成计划，并沉淀为可回看的生活踪迹。

## 当前阶段目标

Life Trace 已从纯前端静态 MVP 推进到“登录门禁 + 天气服务 + 云端计划 + 云端踪迹 + 云端偏好 + 结构化计划时间 + 每日打卡”的早期闭环阶段。当前目标不是继续堆通知能力，而是在稳定数据底座上补服务端 AI：计划、完成后的踪迹、用户偏好、每日打卡和结构化时间都应能成为后续建议、回顾与提醒的可靠上下文。

核心闭环：

```text
今日天气与建议 -> 创建生活计划 -> 完成计划 -> 自动生成生活踪迹 -> 在踪迹流中回看
```

当前闭环中，计划、踪迹、用户偏好、每日打卡和计划时间已进入更可靠的数据结构；今日建议、每周回顾和图片分析已开始接入服务端 AI。下一步从“单次 AI 能力”继续推进到可持续的智能体入口与更细的历史整理体验。

## 技术方案

- 前端：React + Vite + TypeScript
- UI：Tailwind CSS + shadcn/ui dark-first 风格
- 图标：lucide-react
- 状态：Zustand + localStorage 兜底持久化；计划、踪迹、打卡和偏好通过 Go API 同步
- PWA：manifest + service worker
- 数据：天气、计划、结构化计划时间、踪迹、每日打卡和偏好已接入服务端；今日建议、每周回顾和图片分析已开始走服务端 AI，前端保留本地兜底
- 后端：复用 Valley MAS Go 服务，当前已有 Life Trace 天气、计划、踪迹、每日打卡、偏好、今日建议 AI、每周回顾 AI 和图片分析 AI 接口

## 已完成能力

- 已接入账号登录门禁，未登录用户进入登录页。
- 今日页已接入服务端天气接口，支持 QWeather 配置、缓存、手动刷新和 mock 回退。
- 天气建议可转为今日计划。
- 计划已支持服务端列表、创建、完成和删除。
- 踪迹已支持服务端列表、创建和删除。
- 每日打卡已支持服务端列表、完成和取消完成。
- 完成计划后，前端会通过服务端生成一条生活踪迹。
- 首页会展示最近的应用内计划提醒。
- 计划已保留 `timeLabel` 展示文案，并新增 `scheduledDate`、`scheduledTime`、`timezone` 支撑筛选和提醒计算。
- 今日建议可优先从服务端 AI 生成，AI 配置缺失或上游失败时回退到本地建议。
- 每周回顾可优先从服务端 AI 生成并存档，结合云端计划、踪迹、打卡和偏好，失败时回退到本地总结。
- 历史周报支持入口、展开详情、删除，并可从“下周行动”一键生成下周计划。
- 图片分析可优先使用服务端视觉 AI，失败时继续使用本地兜底结果生成计划或踪迹。
- 我的页可配置城市、工作时间、通勤方式、提醒偏好和打卡项，并同步到服务端。
- PWA 已支持 manifest、应用图标、基础离线缓存和安装状态提示。
- PWA 本地系统通知桥接已初步实现：可在我的页申请通知权限，到期计划会尝试触发系统通知，点击通知可回到计划页；该能力待 iPhone/安卓真机手动验收。

## 页面范围

### 今日

- 今日天气主卡
- 未来小时天气
- AI 今日简报
- 今日打卡模块
- 穿衣、护肤、出门、通勤、健康、今日计划建议

### 计划

- 计划筛选：全部、今天、周末、已提醒
- 内置计划类型：电影、吃饭、运动、阅读、聚会、普通事项
- 计划卡片展示提醒状态
- 创建计划 Drawer
- 计划列表、创建、完成和删除通过服务端同步
- 完成计划后生成踪迹，并通过服务端同步

### AI

- Life AI 指挥中心
- 快捷操作：生成今日建议、创建计划、分析图片、生成踪迹、每周回顾
- 示例提示词和最近 AI 操作
- 快捷操作当前接入本地行为：
  - 创建计划：打开创建计划 Drawer
  - 生成今日建议：基于城市、通勤、计划数量生成本地建议
  - 生成踪迹：从最近计划生成生活踪迹
  - 每周回顾：优先基于云端计划、踪迹和打卡项生成服务端 AI 总结，并在 AI 页回看历史周报
- 图片分析：
  - 支持粘贴图片 URL 或选择本地图片预览
  - 支持电影海报、美食照片、生活照片三类分析
  - 优先调用服务端视觉 AI，AI 不可用时回退到本地结果
  - 分析结果可生成计划或踪迹

### 踪迹

- 完成后的生活记录流
- 图片、标签、心情、AI 回忆文案
- 来源可以是计划、打卡或手动记录
- 当前通过服务端同步，localStorage 只保留非核心兜底状态

### 我的

- 城市、工作时间、通勤方式
- 通知偏好
- 打卡项
- 数据和隐私入口
- 当前通过服务端同步偏好，并让今日页读取城市、通勤方式、工作时间和打卡项。

## 下一步优先级

### P0：踪迹云同步（已完成）

目标：让“完成计划 -> 生成踪迹 -> 回顾生活”成为真正跨设备可用的核心闭环。

- 服务端新增 `LifeTraceTrace` 模型。
- 新增 `/api/v1/life-trace/traces` 列表、创建、删除接口。
- 完成计划时，由前端调用 API 创建对应踪迹，并返回可展示数据。
- 前端新增 `api/traces.ts`，`TracesPage` 读取云端踪迹。
- 本地 mock 踪迹只保留为空状态或演示兜底，不再作为登录后的主数据源。

完成结果：计划已经云同步，踪迹也已进入云端，Life Trace 的“生活回顾”不再被单浏览器 localStorage 卡住。

### P1：用户偏好云同步（已完成）

目标：让城市、通勤、工作时间、提醒偏好和打卡项跟随账号，而不是只保存在当前浏览器。

- 新增 Life Trace settings 数据模型。
- 新增 `/api/v1/life-trace/settings` 读取、保存接口。
- 我的页修改后同步到服务端，失败时展示保存错误。
- 今日天气、建议和 AI 入口优先读取服务端偏好。

### P2：计划时间结构化（已完成）

目标：降低提醒和筛选逻辑对 `timeLabel` 字符串解析的依赖。

- 保留 `timeLabel` 作为展示字段。
- 新增 `scheduledDate`、`scheduledTime`、`timezone` 结构化字段。
- 计划筛选、最近提醒和后续 Web Push 基于结构化时间计算。

完成结果：手动计划、天气建议计划和图片分析生成的计划都会尽量写入结构化时间；旧数据继续通过 `timeLabel` 兜底解析，避免历史计划失效。

### P3：服务端 AI 能力（进行中）

目标：基于云端计划、踪迹和偏好生成更稳定的建议。

- 今日建议可从服务端生成，并在前端保留本地兜底。
- 每周回顾可从服务端生成、保存和读取历史，并在前端保留本地兜底。
- 图片分析可从服务端视觉 AI 生成，并在前端保留本地兜底。

当前结果：新增 `/api/v1/life-trace/ai/today-advice`、`/api/v1/life-trace/ai/weekly-review`、`/api/v1/life-trace/weekly-reviews` 和 `/api/v1/life-trace/ai/image-analysis`。服务端会结合偏好、天气、未完成计划、完成计划、踪迹和打卡调用 AI 生成今日建议与每周回顾；周报会保存为可回看的历史记录，并可从“下周行动”生成下周计划；图片分析优先使用视觉模型，视觉模型缺失时可降级到文本模型。今日页和 AI 页优先调用服务端接口，失败时继续使用本地规则，避免页面不可用。

### P3.5：每日打卡云同步（已完成）

目标：让喝水、休息、运动、护肤等每日习惯成为可跨设备同步的数据，为后续 AI 建议和每周回顾提供上下文。

- 新增 `LifeTraceCheckin` 模型和 `life_trace_checkins` 表。
- 新增 `/api/v1/life-trace/checkins` 列表和切换完成状态接口。
- 今日页新增“今日打卡”模块，读取我的页打卡项并支持完成/取消完成。
- 前端新增 `api/checkins.ts`，打卡状态进入 Zustand store。

### P4：PWA Web Push 和外部提醒

目标：在核心数据和结构化时间稳定后，再做真正可送达的提醒。

- 当前标记：本地系统通知桥接已实现但未做手机端验收，暂不继续扩展，后续需要在 iPhone 和安卓上确认权限、到达率、点击回跳和 PWA 主屏幕模式表现。
- 维护浏览器 Push 订阅。
- 服务端按结构化时间触发计划提醒。
- 后续再接天气风险提醒、工作日和通勤提醒。

### P5：下一批功能队列

目标：通知链路暂缓验证时，继续推进不依赖手机系统能力的核心功能。

- 计划体验：创建/编辑时展示“预计提醒时间”和更明确的提醒说明。
- AI 页：把“Life AI 指挥中心”从本地快捷操作升级为真实智能体入口，支持读取今日天气、计划、打卡和踪迹。
- 每周回顾：继续打磨周报历史体验，例如按月份归档、搜索和更清晰的回顾详情。
- 图片分析：继续优化真实视觉 AI 的场景识别、结果解释和计划/踪迹生成质量。
- 踪迹详情：补充详情弹层、图片查看、标签和来源信息。
- 设置页：通知被拒绝时增加浏览器/系统设置引导，iPhone 增加“添加到主屏幕后体验更稳定”的提示。
