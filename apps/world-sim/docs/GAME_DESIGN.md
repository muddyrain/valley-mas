## 一、核心理念

### 设计哲学

> **"你不控制任何人，但你掌控一切。"**

玩家不直接操控小人，而是以神的视角俯瞰世界。文明从零开始自主生长：砍树、建房、繁殖、组建军队、宣战、攻城、灭国。玩家通过**神力工具**干预世界——赐福、降灾、召唤怪物、放置资源，观察世界因你的干预而演化。

### 核心情感目标

- 看到文明从一个小村庄成长为帝国的**成就感**
- 用神力搅动局势、引爆战争的**破坏乐趣**
- 不同种族性格迥异、互相角力的**戏剧张力**
- 每局世界都是独一无二的**惊喜感**

### 核心反馈循环

```
上帝放置文明起点
  → 文明开始自主发展（采集/建造/繁殖）
      ↓ 有趣的 AI 行为让玩家忍不住继续观察
  → 文明扩张，领土接壤，冲突升温
      ↓ 戏剧性叙事钩住玩家注意力
  → 玩家用神力干预（助攻/破坏/搅局）
      ↓ 干预产生因果反馈，局面发生变化
  → 新局面再次让玩家想继续看下去
      ↓ 回到循环起点
```

> 所有系统设计都应服务于这个循环：让玩家**有东西看 → 想干预 → 干预有反馈 → 继续看**。

## 二、世界观与种族设定

### 世界背景

一片混沌的大陆，上帝（玩家）用双手塑造地形，召唤生命，静待文明崛起或毁灭。

### 可玩种族（初版 4 种）

| 种族 | 特性 | 扩张风格 | 视觉风格 |
|---|---|---|---|
| 🧑 人类 | 均衡发展，繁殖速度快 | 稳扎稳打，农业立国 | 棕色皮肤，铁器装备 |
| 🐗 兽人 | 战斗力强，食物消耗高 | 激进扩张，武力征服 | 绿色皮肤，骨甲装备 |
| 🧝 精灵 | 移动快，建造慢，寿命长 | 森林依赖，被动防守 | 蓝色皮肤，弓箭装备 |
| ⛏️ 矮人 | 挖矿效率极高，繁殖慢 | 地下城扩张，资源垄断 | 灰色皮肤，石甲装备 |

### 种族独特行为（AI 行为级差异）

> 这些行为差异决定了不同种族在地图上的视觉表现和 AI 决策优先级，**必须在实现 Agent FSM 时体现**，不能只停留在属性加成。

#### 🧑 人类
- 优先开垦空旷草地，建造农场网格，领土呈方形扩散
- 繁殖速度最快，人口压力是扩张的主要驱动力
- 对外交敏感：会主动维持友好关系，轻易不宣战
- 遇到强敌时倾向结盟而非硬拼

#### 🐗 兽人
- 看到任何相邻文明会在 **1~3 游戏年内主动宣战**，不等领土接壤
- 不重视建造，房屋数量少，优先建兵营
- 粮食消耗是人类的 1.5 倍，资源不足时攻击性显著提升
- 战败后不投降，残余小人散落地图变为流浪战士继续袭扰

#### 🧝 精灵
- **主动保护森林**：周围有树木时绕路，不会主动砍伐（除非木材严重不足）
- 建筑建在森林边缘而非清空地块，领土呈不规则林间形态
- 弓手射程 +2 格（6 格），擅长防守不擅长攻城
- 寿命 80 岁（比人类长 20 年），老龄化慢，精英小人可以积累更高职业等级

#### ⛏️ 矮人
- **主动寻找山地**扎营，首选紧靠山脉的位置建立城镇
- 挖矿速度 ×3，山地地形不减速（其他种族减速 50%）
- 繁殖速度最慢（CD = 8 游戏年，人类为 5 年），但每次生育为双胞胎概率 30%
- 建造地下仓库，资源存储上限 ×2，擅长以资源换外交

### 怪物（上帝可召唤，不属于文明）

| 怪物 | 行为 |
|---|---|
| 🐉 龙 | 随机游荡，攻击任何小人和建筑 |
| 🧟 僵尸 | 感染型，被咬的小人变僵尸 |
| 👾 外星人 | 高科技，毁灭一切 |
| 🐜 虫族 | 数量极多，蜂群式攻击 |

## 三、核心系统设计

### 3.1 世界地图系统

#### 地图结构

- **瓦片地图（Tilemap）**：每个格子是一个 16×16px 像素瓦片
- **地图尺寸**：初始 128×128，支持扩展到 256×256
- **分块加载（Chunk）**：16×16 格为一个 Chunk，按需渲染，减少性能消耗

#### 地形类型（视觉资产规范见 `UI_DESIGN.md §6.1`）

| 地形 | 属性 | 对文明的影响 |
|---|---|---|
| 草地 | 可建造、可耕种 | 适合农业文明 |
| 森林 | 含木材资源 | 可砍伐，砍后变草地 |
| 山地 | 含矿石资源 | 可挖掘，减速移动 |
| 水域 | 不可通行（初期） | 天然屏障 |
| 沙漠 | 可建造，食物产出低 | 不适合初期定居 |
| 雪地 | 可建造，移动减速 | 严酷环境 |
| 熔岩 | 不可通行，造成伤害 | 危险地带 |

#### 地形生成（Procedural Generation）

- 使用 **Simplex Noise** 生成高度图
- 高度决定地形：低 → 水域，中 → 草地/森林，高 → 山地/雪地
- 随机放置初始资源点（铁矿、金矿、神树等）
- 支持玩家选择地图预设（大陆、群岛、雪原、地狱）
- 正式随机地图需要按地形可居住性与势力间最小距离选择出生点，避免多个文明开局贴脸；M1 固定测试局面先用地图宽度约 25% / 75% 的比例布点模拟远距离出生

#### 战争迷雾（Fog of War）

- **玩家（上帝）始终全图可见**，没有视野限制，这是上帝视角的核心体验
- 迷雾只影响 **AI 决策**：文明只对已探索区域做决策，不会主动进攻未知领地
- 小人移动揭开周围视野（半径 3 格），扩大该文明的 AI 可决策范围
- 未被任何文明探索的区域在 UI 上可用轻度暗色标识（可选），帮助玩家感知文明的探索进展

### 3.2 Agent 智能体系统

#### 小人基础属性

```
每个小人（Unit）拥有以下属性：

基础属性：
  id          唯一标识
  name        随机姓名
  race        种族
  factionId   所属势力
  age         年龄（0~60岁，超过自然死亡）
  gender      性别（影响繁殖）

生存属性：
  hp          生命值（0死亡）
  vitality    体力值（0~100，综合饥饿与疲劳，降到0开始掉血）
              ↑ 作为单一生存指标，避免 hunger/energy 双指标抢优先级

初版生存参数：
  vitality 每秒 -1；进入 Rest 后每秒 +9
  vitality = 0 时 hp 每秒 -6
  1 游戏年 = 60 现实秒；age >= 60 时自然死亡

职业属性：
  job         当前职业（农民/建造者/战士/侦察兵/商人）
  skill       职业技能等级（影响效率）

状态机：
  state       当前 FSM 状态
```

#### 小人职业体系

| 职业 | 触发条件 | 行为描述 |
|---|---|---|
| 农民（Farmer）| 默认职业 | 耕地、收割、采集浆果 |
| 伐木工（Lumberjack）| 村庄木材不足 | 砍伐附近树木 |
| 矿工（Miner）| 村庄矿石不足 | 挖掘山地矿点 |
| 建造者（Builder）| 有建造任务 | 建造房屋、城墙、仓库 |
| 战士（Warrior）| 村庄宣战/被攻击 | 行军、攻城、防守边境 |
| 侦察兵（Scout）| 村庄人口 > 阈值 | 探索迷雾区域 |
| 商人（Merchant）| 双方结盟后 | 在盟友城镇间运输资源 |
| 国王（King）| 一个势力只有一个 | 协调决策，提升士气 |

#### FSM 状态机（每个小人共用）

```
状态列表：
  Idle        → 闲置，找下一个任务
  Wander      → 随机游走（侦察兵用）
  Harvest     → 采集资源中
  Build       → 建造中
  Rest        → 去仓库补充体力（vitality < 30 时触发）
  Sleep       → 回家休息（夜晚）
  March       → 行军中（前往目标）
  Attack      → 攻击目标中
  Flee        → 逃跑（HP 低于 20%）
  Dead        → 死亡动画播放后移除

状态转移触发：
  vitality < 30 → 强制进入 Rest 状态（去仓库补给）
  HP < 20%      → 强制进入 Flee 状态
  收到攻击命令  → 进入 March → Attack
  收到建造命令  → 进入 March → Build
  无任务且体力满 → Idle → 随机分配任务
```

#### 寻路系统

- 使用 **A\* 算法**
- 地形权重：草地=1，森林=2，山地=3，水域=∞（不可通行）
- 寻路计算在 **Web Worker** 中异步执行，不阻塞渲染
- 路径缓存：相同起终点 5 秒内复用缓存路径

#### 生命周期

```
出生 → 幼年（0~10岁，不参与劳动）
     → 成年（10~50岁，正常工作）
     → 老年（50~60岁，效率降低 50%）
     → 自然死亡（60岁）

繁殖条件：
  - 成年男女均在同一势力内
  - M1 原型先用“同势力 + 近距离”近似领地判定，后续接入领土系统后切换为真实领土判定
  - 仓库食物 > 20（整体充足即可，不绑定个人）
  - 人口未超过势力上限（硬上限：150 人/势力）
  - 每次生育 CD = 5 游戏年

- 新生小人从父母中点生成，初版使用缩放淡入动画，便于观察出生反馈
```

### 3.3 文明（Faction）系统

#### 文明发展阶段

```
营地（Camp）
  人口：1~5
  建筑：篝火
  能力：基础采集

  ↓ 人口 ≥ 6 且木材 ≥ 50

村庄（Village）
  人口：6~20
  建筑：木屋、仓库、农场
  能力：农业、基础建造

  ↓ 人口 ≥ 21 且石料 ≥ 100 且有铁矿

城镇（Town）
  人口：21~60
  建筑：石屋、铁匠铺、城墙（木）、兵营
  能力：训练士兵、初级外交

  ↓ 人口 ≥ 61 且石料 ≥ 200 且城墙完整
  （金矿不是必须条件，拥有金矿可加速升级进度）

王国（Kingdom）
  人口：61~150
  人口上限：150 人（城堡建成后解锁，上限提升至 200）
  建筑：城堡、石城墙、神殿、市场
  能力：高级外交、大规模战争、贸易

  ↓ 人口 ≥ 151 且征服 ≥ 2 个城镇

帝国（Empire）
  人口：151~200（独立区间，硬上限 200 人）
  建筑：帝都、巨型城墙、传送门
  能力：召唤高级单位、终局能力
  备注：达到帝国阶段意味着文明进入强盛期，是「世界统一」里程碑的前置条件
```

#### 资源体系

| 资源 | 来源 | 用途 |
|---|---|---|
| 🌾 食物 | 农场收割、采集浆果、狩猎 | 小人存活、繁殖 |
| 🪵 木材 | 砍树 | 建造篝火、木屋、仓库、农场，制作基础工具 |
| 🪨 石料 | 挖山 | 建造仓库、石质建筑、城墙、防御工事 |
| ⚔️ 铁矿 | 铁矿点 | 锻造武器、盔甲、军备升级 |
| 💰 金币 | 贸易、金矿 | 外交、购买高级单位 |

#### 初版资源落地规则

- 森林格默认携带木材资源，初始量 10，采完后地形变为草地
- 山地格默认携带石料或铁矿资源，初始量 20，采完后仅清空资源
- 草地格可零散携带少量食物资源，初始量 8
- 资源采集后同时进入势力库存，HUD 仅显示汇总数，不逐格展示
- 资源点会在地图上以代码绘制的小型像素图标标记，方便区分可采集格：木材=树/原木，石料=石块，铁矿=带橙红矿点的岩石，食物=浆果/麦穗
- 采集 AI 以库存目标阈值为停采线：食物 40、木材 200、石料 120、铁矿 60；单位每次只选择当前最高优先级且地图上可采的资源，食物未达标时优先找食物，不会因为木材更近而先砍树
- 当前 M1 可视化原型会为每个起始势力创建独立资源库存和建造队列；人类与兽人都能自动采集、休息、按人口需求补建木屋并触发自身领土扩张
- 当前食物库存已用于繁殖消耗；按人口持续吃饭、缺粮惩罚和种族食物消耗倍率后续接入生存/时间系统

#### 初版建造落地规则

| 建筑 | HP | 建造消耗 | 用途 |
|---|---:|---|---|
| 篝火 | 30 | 木材 4 | 营地视觉占位 |
| 木屋 | 50 | 木材 10 | 初版自动建造目标、领土扩张锚点；M1 按每 5 人约 1 栋补建，后续接入正式住房容量与繁殖/文明升级加成 |
| 仓库 | 80 | 木材 12、石料 4 | 后续库存上限/交付点 |
| 农场 | 45 | 木材 6 | 后续食物生产 |

- M1 原型按势力人口自动生成木屋建造任务，当前规则为每 5 人约 1 栋；建造点从该势力首都点右下 1 格开始排列，验证“资源 → 建造 → 建筑出现 → 领土扩张”的闭环
- M1 原型每个势力的休息点使用自身首都点，避免第二势力单位体力低时跑到其他势力营地
- 建造任务启动时一次性扣除资源，施工进度按现实时间推进
- 建造中的建筑以地基、脚手架和进度条显示；完成后的木屋至少显示屋顶、墙体和门，正式 sprite 待美术任务接入
- 木屋当前只提供可视化建设需求与领土扩张锚点；正式人口容量、居住舒适度和繁殖/文明升级加成归后续文明升级链和人口/住房规则接入
- 文明阶段升级仍待势力系统接入后结算

#### 领土系统

- 每个地块有 `ownerFactionId` 属性
- M1 启动局面生成人类营地和兽人部落两个势力，每个势力各 10 人起始单位，并在首都位置周围预置 2 格半径起始领土，保证势力与领土系统开局可见
- M1 固定测试地图中，两方首都按地图宽度比例拉开，默认横向距离不少于地图宽度的 45%，避免领土过早接壤导致战斗/外交节奏失真
- 单位头顶常驻一条势力色标识，避免单位主体颜色被当前状态（采集、行军、休息等）覆盖后无法辨认归属
- 小人在某地块建造建筑 → 周围 3 格自动划入领土（M1 原型使用方形半径，默认 7×7 区域）
- 领土边界用对应势力颜色染色瓦片叠加层，初版只在领土变化时重绘
- 两个势力领土接壤 → 触发接壤通知，并在 M1 原型中自动宣战

> M1 原型已先落地 `Faction` / `FactionManager` / `TerritorySystem` / `DiplomacySystem` 的基础数据层：人口、首都位置、关系矩阵、仓库字段和领土格数由势力真源维护，WorldScene 的双势力启动局面、光标格子、镜头中心格子、势力摘要、单位归属、每势力资源/建造闭环、领土染色叠加层、接壤通知和最小外交状态摘要已接入。临时首都旗标已从默认地图渲染中移除；后续应由真实建筑或正式 UI 表达势力核心。

### 3.4 战争系统

#### 战争触发条件

```
自动宣战触发（任一满足）：
  - 领土接壤超过 5 格且关系为"中立"超过 3~5 游戏年（随机）
  - 资源严重不足（食物 < 10）且邻近势力有充足资源
  - 种族天性（兽人与任何势力接壤后 1~3 游戏年内随机宣战）
  - 上帝工具主动触发（玩家可强制两势力开战）
```

#### 攻城流程

```
1. 宣战 → 双方进入"战争"外交状态
2. 集结 → 兵营训练士兵，在集结点等待
3. 行军 → 士兵分队前往边境
4. 攻城 → 攻击城墙（城墙有 HP）
5. 破城 → 城墙 HP = 0，进入城内
6. 巷战 → 与守城士兵逐一战斗
7. 占领 → 敌方小人全灭或逃跑 → 局部领土清空为无主，等待后续势力扩张重新争夺
8. 驻守 → 留下驻守部队，其余回城
```

#### 战斗结算

```
单次攻击伤害 = 基础攻击力 × 武器系数 × 随机(0.8~1.2)

防御计算 = 目标HP扣减 = 伤害 - 防御力（最少扣1）

武器系数：
  徒手：0.5
  石斧：1.0
  铁剑：1.8
  弓箭：1.2（远程，射程 4 格）
  攻城锤：3.0（只对建筑有效）

建筑HP：
  木屋：50    石屋：150
  木城墙：200  石城墙：500
  城堡：1000
```

M1 原型战斗落地规则：

- 已接入最小外交：只有处于战争关系的非本势力单位或完整建筑才会被主动攻击
- 宣战后会把双方存活单位自动向共同战场点集结一次，形成最小开战反馈
- 近战检测范围：48px
- 初版所有单位使用固定基础攻击力 12，武器系数 1
- 单位防御力暂定 2，建筑防御力暂定 0
- 攻击冷却 900ms
- HP 低于 20% 时进入 `Flee`，向威胁反方向移动
- 当前逃跑为单个单位低血逃跑，不代表整支队伍会判断“打不过就撤退”；士气、溃败、集结撤退和近战站位上限归后续战争 AI/外交阶段迭代
- 当前追击仍为直线移动，正式 A* 追击与站位绕行归 TASK-006
- 当目标区域内不再存在该势力的存活单位时，战斗结果会触发局部清空：攻击方清掉目标附近的敌占领土，HUD 提示“占领”，并同步刷新领土染色与势力领土计数；这些格子随后变为无主
- 当目标势力已没有任何存活单位时，视为被征服：剩余领土整体变为无主，原势力从势力摘要中移除，M1 原型会移除其运行时建筑队列和据点视图
- 后续扩张 AI 可以把无主领土按资源价值排序，优先争夺高资源地块，而不是机械继承前线战果

#### 战败处理

```
人口归零 → 势力灭亡，剩余领土变为无主，待后续势力争夺
  - 小人死亡或被奴役（由征服者种族决定）
  - 建筑与据点移除，领土不自动转移给胜利方
  - 国王逃跑机制：国王 HP < 10% 时，向己方领土纵深最深处逃跑
    （计算方式：找到距离敌方最远的己方格子作为逃跑目标，而非地图边缘）
```

## 四、游戏流程设计

### 典型游戏流程

```
1. 世界生成
   玩家选择地图预设 → 程序随机生成地形

2. 播种文明
   玩家在地图上放置 2~4 个种族起始营地

3. 观察成长（早期）
   文明自动发展：采集 → 建村 → 繁殖
   玩家偶尔降雨、赠资源帮助成长

4. 文明冲突（中期）
   各文明扩张 → 领土接壤 → 摩擦增加
   玩家可点火引战，或强制和平

5. 大战时代（后期）
   多个王国并立 → 大规模战争 → 弱者灭亡
   玩家可扶持弱者、打压强者

6. 里程碑事件（不是「结束」）
   某文明统一全图 → 触发「世界统一」里程碑
     - 事件日志记录、短暂庆典动画
     - 游戏继续运行，世界不会暂停或重置
     - 玩家可以召唤怪物/灾难破坏帝国，观察它如何应对

   玩家主动毁灭 → 触发「神之毁灭」里程碑（核弹/龙/陨石清空全图）
     - 同样继续运行，残余小人在废墟中重建
```

> **核心设计原则：游戏没有强制结局，世界永远在运转。**  
> 里程碑只是一个叙事节点，不触发游戏结束。玩家随时可以重置世界（主菜单新游戏），  
> 但世界本身永远不会「自动结束」——这是 God Game 与策略游戏最根本的区别。

### 事件系统（随机叙事）

游戏内随机触发文字事件，增加世界感：

```
示例事件：
  "人类王国的国王 Arthur 在攻城战中阵亡，其子 Henry 继承王位。"
  "精灵村庄发现了一处古老神树，生产力大幅提升。"
  "一场瘟疫在兽人部落蔓延，人口减少了 30%。"
  "矮人王国与人类王国达成贸易协定。"
```
