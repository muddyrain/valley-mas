## 一、核心理念

### 设计哲学

> **"你不控制任何人，但你掌控一切。"**

玩家不直接操控小人，而是以神的视角俯瞰世界。文明从零开始自主生长：砍树、建房、繁殖、组建军队、宣战、攻城、灭国。玩家通过**神力工具**干预世界——赐福、降灾、召唤怪物、放置资源，观察世界因你的干预而演化。

### 核心情感目标

- 看到文明从一个小村庄成长为帝国的**成就感**
- 用神力搅动局势、引爆战争的**破坏乐趣**
- 不同种族性格迥异、互相角力的**戏剧张力**
- 每局世界都是独一无二的**惊喜感**

### 核心反馈循环

```
上帝放置文明起点
  → 文明开始自主发展（采集/建造/繁殖）
      ↓ 有趣的 AI 行为让玩家忍不住继续观察
  → 文明扩张，领土接壤，冲突升温
      ↓ 戏剧性叙事钩住玩家注意力
  → 玩家用神力干预（助攻/破坏/搅局）
      ↓ 干预产生因果反馈，局面发生变化
  → 新局面再次让玩家想继续看下去
      ↓ 回到循环起点
```

> 所有系统设计都应服务于这个循环：让玩家**有东西看 → 想干预 → 干预有反馈 → 继续看**。

## 二、世界观与种族设定

### 世界背景

一片混沌的大陆，上帝（玩家）用双手塑造地形，召唤生命，静待文明崛起或毁灭。

### 可玩种族（初版 4 种）

| 种族 | 特性 | 扩张风格 | 视觉风格 |
|---|---|---|---|
| 🧑 人类 | 均衡发展，繁殖速度快 | 稳扎稳打，农业立国 | 棕色皮肤，铁器装备 |
| 🐗 兽人 | 战斗力强，食物消耗高 | 激进扩张，武力征服 | 绿色皮肤，骨甲装备 |
| 🧝 精灵 | 移动快，建造慢，寿命长 | 森林依赖，被动防守 | 蓝色皮肤，弓箭装备 |
| ⛏️ 矮人 | 挖矿效率极高，繁殖慢 | 地下城扩张，资源垄断 | 灰色皮肤，石甲装备 |

### 种族独特行为（AI 行为级差异）

> 这些行为差异决定了不同种族在地图上的视觉表现和 AI 决策优先级，**必须在实现 Agent FSM 时体现**，不能只停留在属性加成。

#### 🧑 人类
- 优先开垦空旷草地，建造农场网格，领土呈方形扩散
- 繁殖速度最快，人口压力是扩张的主要驱动力
- 对外交敏感：会主动维持友好关系，轻易不宣战
- 遇到强敌时倾向结盟而非硬拼

#### 🐗 兽人
- 看到任何相邻文明会在 **1~3 游戏年内主动宣战**，不等领土接壤
- 不重视建造，房屋数量少，优先建兵营
- 粮食消耗是人类的 1.5 倍，资源不足时攻击性显著提升
- 战败后不投降，残余小人散落地图变为流浪战士继续袭扰

#### 🧝 精灵
- **主动保护森林**：周围有树木时绕路，不会主动砍伐（除非木材严重不足）
- 建筑建在森林边缘而非清空地块，领土呈不规则林间形态
- 弓手射程 +2 格（6 格），擅长防守不擅长攻城
- 寿命 80 岁（比人类长 20 年），老龄化慢，精英小人可以积累更高职业等级

#### ⛏️ 矮人
- **主动寻找山地**扎营，首选紧靠山脉的位置建立城镇
- 挖矿速度 ×3，山地地形不减速（其他种族减速 50%）
- 繁殖速度最慢（CD = 8 游戏年，人类为 5 年），但每次生育为双胞胎概率 30%
- 建造地下仓库，资源存储上限 ×2，擅长以资源换外交

### 怪物（上帝可召唤，不属于文明）

| 怪物 | 行为 |
|---|---|
| 🐉 龙 | 随机游荡，攻击任何小人和建筑 |
| 🧟 僵尸 | 感染型，被咬的小人变僵尸 |
| 👾 外星人 | 高科技，毁灭一切 |
| 🐜 虫族 | 数量极多，蜂群式攻击 |

## 三、核心系统设计

### 3.1 世界地图系统

#### 地图结构

- **瓦片地图（Tilemap）**：每个格子是一个 16×16px 像素瓦片
- **地图尺寸**：初始 128×128，支持扩展到 256×256
- **分块加载（Chunk）**：16×16 格为一个 Chunk，按需渲染，减少性能消耗

#### 地形类型（视觉资产规范见 `UI_DESIGN.md §6.1`）

| 地形 | 属性 | 对文明的影响 |
|---|---|---|
| 草地 | 可建造、可耕种 | 适合农业文明 |
| 森林 | 含木材资源 | 可砍伐，砍后变草地 |
| 山地 | 含矿石资源 | 可挖掘，减速移动 |
| 水域 | 不可通行（初期） | 天然屏障 |
| 沙漠 | 可建造，食物产出低 | 不适合初期定居 |
| 雪地 | 可建造，移动减速 | 严酷环境 |
| 熔岩 | 不可通行，造成伤害 | 危险地带 |

#### 地形生成（Procedural Generation）

- 使用 **Simplex Noise** 生成高度图
- 高度决定地形：低 → 水域，中 → 草地/森林，高 → 山地/雪地
- 随机放置初始资源点（铁矿、金矿、神树等）
- 支持玩家选择地图预设（大陆、群岛、雪原、地狱）

#### 战争迷雾（Fog of War）

- **玩家（上帝）始终全图可见**，没有视野限制，这是上帝视角的核心体验
- 迷雾只影响 **AI 决策**：文明只对已探索区域做决策，不会主动进攻未知领地
- 小人移动揭开周围视野（半径 3 格），扩大该文明的 AI 可决策范围
- 未被任何文明探索的区域在 UI 上可用轻度暗色标识（可选），帮助玩家感知文明的探索进展

### 3.2 Agent 智能体系统

#### 小人基础属性

```
每个小人（Unit）拥有以下属性：

基础属性：
  id          唯一标识
  name        随机姓名
  race        种族
  factionId   所属势力
  age         年龄（0~60岁，超过自然死亡）
  gender      性别（影响繁殖）

生存属性：
  hp          生命值（0死亡）
  vitality    体力值（0~100，综合饥饿与疲劳，降到0开始掉血）
              ↑ 不拆分 hunger/energy，减少 FSM 优先级冲突

职业属性：
  job         当前职业（农民/建造者/战士/侦察兵/商人）
  skill       职业技能等级（影响效率）

状态机：
  state       当前 FSM 状态
```

#### 小人职业体系

| 职业 | 触发条件 | 行为描述 |
|---|---|---|
| 农民（Farmer）| 默认职业 | 耕地、收割、采集浆果 |
| 伐木工（Lumberjack）| 村庄木材不足 | 砍伐附近树木 |
| 矿工（Miner）| 村庄矿石不足 | 挖掘山地矿点 |
| 建造者（Builder）| 有建造任务 | 建造房屋、城墙、仓库 |
| 战士（Warrior）| 村庄宣战/被攻击 | 行军、攻城、防守边境 |
| 侦察兵（Scout）| 村庄人口 > 阈值 | 探索迷雾区域 |
| 商人（Merchant）| 双方结盟后 | 在盟友城镇间运输资源 |
| 国王（King）| 一个势力只有一个 | 协调决策，提升士气 |

#### FSM 状态机（每个小人共用）

```
状态列表：
  Idle        → 闲置，找下一个任务
  Wander      → 随机游走（侦察兵用）
  Harvest     → 采集资源中
  Build       → 建造中
  Rest        → 去仓库补充体力（vitality < 30 时触发）
  Sleep       → 回家休息（夜晚）
  March       → 行军中（前往目标）
  Attack      → 攻击目标中
  Flee        → 逃跑（HP 低于 20%）
  Dead        → 死亡动画播放后移除

状态转移触发：
  vitality < 30 → 强制进入 Rest 状态（去仓库补给）
  HP < 20%      → 强制进入 Flee 状态
  收到攻击命令  → 进入 March → Attack
  收到建造命令  → 进入 March → Build
  无任务且体力满 → Idle → 随机分配任务
```

#### 寻路系统

- 使用 **A\* 算法**
- 地形权重：草地=1，森林=2，山地=3，水域=∞（不可通行）
- 寻路计算在 **Web Worker** 中异步执行，不阻塞渲染
- 路径缓存：相同起终点 5 秒内复用缓存路径

#### 生命周期

```
出生 → 幼年（0~10岁，不参与劳动）
     → 成年（10~50岁，正常工作）
     → 老年（50~60岁，效率降低 50%）
     → 自然死亡（60岁）

繁殖条件：
  - 成年男女均在同一势力领土内
  - 仓库食物 > 20（整体充足即可，不绑定个人）
  - 人口未超过势力上限（硬上限：150 人/势力）
  - 每次生育 CD = 5 游戏年
```

### 3.3 文明（Faction）系统

#### 文明发展阶段

```
营地（Camp）
  人口：1~5
  建筑：篝火
  能力：基础采集

  ↓ 人口 ≥ 6 且木材 ≥ 50

村庄（Village）
  人口：6~20
  建筑：木屋、仓库、农场
  能力：农业、基础建造

  ↓ 人口 ≥ 21 且石料 ≥ 100 且有铁矿

城镇（Town）
  人口：21~60
  建筑：石屋、铁匠铺、城墙（木）、兵营
  能力：训练士兵、初级外交

  ↓ 人口 ≥ 61 且石料 ≥ 200 且城墙完整
  （金矿不是必须条件，拥有金矿可加速升级进度）

王国（Kingdom）
  人口：61~150
  人口上限：150 人（城堡建成后解锁，上限提升至 200）
  建筑：城堡、石城墙、神殿、市场
  能力：高级外交、大规模战争、贸易

  ↓ 人口 ≥ 151 且征服 ≥ 2 个城镇

帝国（Empire）
  人口：151~200（独立区间，硬上限 200 人）
  建筑：帝都、巨型城墙、传送门
  能力：召唤高级单位、终局能力
  备注：达到帝国阶段意味着文明进入强盛期，是「世界统一」里程碑的前置条件
```

#### 资源体系

| 资源 | 来源 | 用途 |
|---|---|---|
| 🌾 食物 | 农场收割、采集浆果、狩猎 | 小人存活、繁殖 |
| 🪵 木材 | 砍树 | 建造木质建筑、武器 |
| 🪨 石料 | 挖山 | 建造石质建筑、城墙 |
| ⚔️ 铁矿 | 铁矿点 | 锻造武器、盔甲 |
| 💰 金币 | 贸易、金矿 | 外交、购买高级单位 |

#### 领土系统

- 每个地块有 `ownerFactionId` 属性
- 小人在某地块建造建筑 → 周围 N 格自动划入领土
- 领土边界用对应势力颜色染色瓦片叠加层
- 两个势力领土接壤 → 触发外交判断

### 3.4 战争系统

#### 战争触发条件

```
自动宣战触发（任一满足）：
  - 领土接壤超过 5 格且关系为"中立"超过 3~5 游戏年（随机）
  - 资源严重不足（食物 < 10）且邻近势力有充足资源
  - 种族天性（兽人与任何势力接壤后 1~3 游戏年内随机宣战）
  - 上帝工具主动触发（玩家可强制两势力开战）
```

#### 攻城流程

```
1. 宣战 → 双方进入"战争"外交状态
2. 集结 → 兵营训练士兵，在集结点等待
3. 行军 → 士兵分队前往边境
4. 攻城 → 攻击城墙（城墙有 HP）
5. 破城 → 城墙 HP = 0，进入城内
6. 巷战 → 与守城士兵逐一战斗
7. 占领 → 敌方小人全灭或逃跑 → 插旗，领土变色
8. 驻守 → 留下驻守部队，其余回城
```

#### 战斗结算

```
单次攻击伤害 = 基础攻击力 × 武器系数 × 随机(0.8~1.2)

防御计算 = 目标HP扣减 = 伤害 - 防御力（最少扣1）

武器系数：
  徒手：0.5
  石斧：1.0
  铁剑：1.8
  弓箭：1.2（远程，射程 4 格）
  攻城锤：3.0（只对建筑有效）

建筑HP：
  木屋：50    石屋：150
  木城墙：200  石城墙：500
  城堡：1000
```

#### 战败处理

```
领土被全部占领 → 势力灭亡
  - 小人死亡或被奴役（由征服者种族决定）
  - 建筑被摧毁或保留（征服者可以选择）
  - 国王逃跑机制：国王 HP < 10% 时，向己方领土纵深最深处逃跑
    （计算方式：找到距离敌方最远的己方格子作为逃跑目标，而非地图边缘）
```

#### 种族/单位召唤

| 工具 | 效果 | 神力消耗 |
|---|---|---|
| 👥 召唤小人 | 放置指定种族小人 | 10/个 |
| 👑 召唤国王 | 直接创建一个势力 | 50 |
| 🐉 召唤龙 | 放置龙，随机攻击 | 100 |
| 🧟 僵尸病毒 | 指定小人变僵尸 | 30 |
| ☄️ 陨石 | 砸毁一片区域建筑和小人 | 80 |
| 🌪️ 龙卷风 | 移动龙卷风，席卷一切 | 60 |

#### 神力工具（干预）

| 工具 | 效果 | 神力消耗 |
|---|---|---|
| ⚡ 闪电 | 秒杀单个目标 | 20 |
| 🌧️ 降雨 | 提升指定区域食物产量 | 15 |
| ☀️ 干旱 | 降低指定区域食物产量 | 15 |
| 💊 治愈 | 恢复目标全部HP | 10 |
| 💀 瘟疫 | 区域内小人持续掉血 | 40 |
| 🤝 强制和平 | 两势力停战 | 50 |
| ⚔️ 强制宣战 | 两势力立即开战 | 50 |
| 🎁 资源礼包 | 给指定势力投放资源 | 30 |
| 🧨 核弹 | 毁灭大范围区域 | 200 |

#### 神力值获取

- **按现实时间恢复**（不随游戏加速倍率提升）：每现实分钟 +3 神力
  - 这样加速观察文明演化和积累神力之间存在真实取舍
- 文明事件发生（升级、战争结束）：+奖励神力
- 成就完成：+神力奖励

### 3.7 时间系统

```
时间流速：
  正常速度：1 游戏年 = 60 现实秒
  加速 2x：1 游戏年 = 30 现实秒
  加速 4x：1 游戏年 = 15 现实秒
  暂停：时间静止，可操作地形

昼夜循环：
  白天：小人正常工作
  夜晚：小人回家休息，视野减半，怪物活跃度提升

季节系统（初版可选）：
  春：食物产量 ×1.2
  夏：正常
  秋：食物产量 ×1.5（收获季）
  冬：食物产量 ×0.3，小人能量消耗 ×1.5
```

## 四、游戏流程设计

### 典型游戏流程

```
1. 世界生成
   玩家选择地图预设 → 程序随机生成地形

2. 播种文明
   玩家在地图上放置 2~4 个种族起始营地

3. 观察成长（早期）
   文明自动发展：采集 → 建村 → 繁殖
   玩家偶尔降雨、赠资源帮助成长

4. 文明冲突（中期）
   各文明扩张 → 领土接壤 → 摩擦增加
   玩家可点火引战，或强制和平

5. 大战时代（后期）
   多个王国并立 → 大规模战争 → 弱者灭亡
   玩家可扶持弱者、打压强者

6. 里程碑事件（不是「结束」）
   某文明统一全图 → 触发「世界统一」里程碑
     - 事件日志记录、短暂庆典动画
     - 游戏继续运行，世界不会暂停或重置
     - 玩家可以召唤怪物/灾难破坏帝国，观察它如何应对

   玩家主动毁灭 → 触发「神之毁灭」里程碑（核弹/龙/陨石清空全图）
     - 同样继续运行，残余小人在废墟中重建
```

> **核心设计原则：游戏没有强制结局，世界永远在运转。**  
> 里程碑只是一个叙事节点，不触发游戏结束。玩家随时可以重置世界（主菜单新游戏），  
> 但世界本身永远不会「自动结束」——这是 God Game 与策略游戏最根本的区别。

### 事件系统（随机叙事）

游戏内随机触发文字事件，增加世界感：

```
示例事件：
  "人类王国的国王 Arthur 在攻城战中阵亡，其子 Henry 继承王位。"
  "精灵村庄发现了一处古老神树，生产力大幅提升。"
  "一场瘟疫在兽人部落蔓延，人口减少了 30%。"
  "矮人王国与人类王国达成贸易协定。"
```

## 五、技术架构规划

### 技术栈

| 层级 | 技术选型 | 理由 |
|---|---|---|
| 渲染引擎 | **Phaser 3** | 原生 Tilemap 支持，WebGL 批渲染，插件生态好 |
| 语言 | **TypeScript** | 类型安全，与 monorepo 一致 |
| 构建工具 | **Vite** | 与项目现有工具链一致 |
| AI 计算 | **Web Worker** | 寻路和 FSM 计算不阻塞渲染 |
| 状态管理 | **Zustand** | 轻量，适合游戏全局状态 |
| 噪声生成 | **simplex-noise** | 地形生成 |
| 桌面打包 | **Electron**（可选，后期按需）|
