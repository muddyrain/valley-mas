# 🔒 设计一致性闸门

任何功能变更、玩法插入、数值调整或阶段改动前，必须先做一致性检查。

## 必查问题

1. 这个功能是否已经在 `scratch-legend-design.md` 中定义？
2. 它在核心循环中的位置是什么？
3. 它是否会改变当前阶段的目标？
4. 它是否引入了新的成本、奖励、概率、惩罚或解锁条件？
5. 它是否会破坏“洗盘子是门票，刮刮卡是主菜”的主循环？

## 处理规则

* 如果设计文档已定义，任务文档只允许拆解执行步骤。
* 如果设计文档未定义，必须先更新设计文档，再更新任务文档。
* 如果设计文档与任务文档冲突，以设计文档为准，并先修正任务文档。
* 如果只是实现细节变化，但不改变玩法和数值，可以直接写入当前阶段任务。
* 如果会新增玩法、改变节奏或改变奖励池，不允许直接实现。

## 玩法插入规则

中途想加入新玩法时，必须先在设计文档中说明：

* 它插入核心循环的哪个位置。
* 它替代、补充还是延后现有阶段。
* 它的成本、奖励、风险和解锁条件。
* 它对前 10 分钟节奏的影响。
* 它是否属于当前阶段。

只有完成上述设计对齐后，才能把它加入本任务清单。

# 🧩 阶段 1：洗盘子启动金

⚠️ 当前禁止开发，先完成阶段 0。

## 🎯 目标

让玩家通过洗盘子获得第一张低级刮刮卡的启动金，同时避免游戏变成点击器。

## ✅ 核心机制

### 1. 动作锁

```ts
actionDuration = 400~800ms
```

### 2. 收益逻辑

```ts
base = random(1, 3)
critChance = 0.05
critMultiplier = 5
```

### 3. 收益结算

```ts
if (isCrit) {
  gold += base * critMultiplier
} else {
  gold += base
}
```

## ✅ 任务

* [ ] 左侧展示“日常工作”卡片。
* [ ] 日常工作卡片展示 `$1~$3` 奖励、`等级 0` 和进度条。
* [ ] 参考 `v1-gameplay-overview.png` 的 `1.3 工作面板`，展示中奖率、基础收益和升级按钮；升级按钮当前可以展示但不实现升级效果。
* [ ] 点击“日常工作”后，在桌面生成一个脏盘子小物件。
* [ ] 点击桌面脏盘子后，进入放大清洁视图。
* [ ] 参考 `v1-gameplay-overview.png` 的 `1.1 刮盘子界面`，放大清洁视图展示大盘子、污渍层、海绵、奖励金额和清洁进度。
* [ ] 玩家必须拖动海绵擦除污渍。
* [ ] 根据污渍遮罩擦除比例计算清洁进度。
* [ ] 清洁完成前禁止领取奖励。
* [ ] 参考 `v1-gameplay-overview.png` 的 `1.2 结算界面`，清洁完成后显示可领取金额。
* [ ] 领取后结算金币，并回到桌面状态。
* [ ] 同一时间只允许存在一个日常工作盘子。
* [ ] 连续点击工作卡或盘子只增强视觉反馈，不增加收益。
* [ ] 接近 `basicCardCost = 100` 时只提示可以买卡，不自动跳转。
* [ ] 强化“我在赚第一张卡门票”的反馈。

## ✅ 验收标准

* 点击“日常工作”不会直接获得金币。
* 点击桌面脏盘子不会直接获得金币。
* 玩家必须进入放大视图并拖动海绵清洁。
* 清洁进度达到完成阈值后才能领取奖励。
* 金币只在领取奖励时结算一次。
* 30~60 秒可以买得起第一张低级刮刮卡。
* 玩家能理解洗盘子只是启动金来源。

# 🧩 阶段 2：低级刮刮卡 MVP

⚠️ 当前禁止开发，先完成阶段 1。

## 🎯 目标

实现主玩法入口：购买低级卡、拖动刮开、渐进揭露、自动结算。

## ✅ 核心机制

```ts
basicCardCost = 100
```

结算条件：

```ts
if (allRequiredCellsRevealed || erasedPercent >= 70%) {
  settleCard()
}
```

## ✅ 任务

* [ ] 实现低级刮刮卡购买。
* [ ] 购买时预生成卡片结果。
* [ ] 使用 Canvas 遮罩实现拖动擦除。
* [ ] 禁止点击直接开奖。
* [ ] 实现三格匹配基础规则。
* [ ] 实现基础奖励池。
* [ ] 结算后更新金币、刮卡次数和连败次数。

## ✅ 基础奖励池

| 类型 | 概率 | 结果 |
| --- | --- | --- |
| 普通亏损 | 55% | 0 |
| 小奖 | 25% | 120 |
| 大奖 | 12% | 500 |
| 惩罚 | 5% | -50 或扣除一张卡 |
| 传说 | 3% | 2000 |

## ✅ 验收标准

* 玩家必须拖动刮开。
* 刮开过程有遮罩渐进消失。
* 结果在购买时确定，不在最后一刻随机。
* 买卡、刮卡、结算、继续买卡的循环成立。
