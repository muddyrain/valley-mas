# 刮出传说（Scratch Legend）- 核心玩法设计

---

# 一、核心定位

本游戏对标《刮个爽》的核心骨架：

```txt
洗盘子赚启动金
→ 买第一张低级刮刮卡
→ 拖动刮卡
→ 中奖 / 亏损 / 惩罚
→ 买更多不同卡
→ 升级刮卡能力与幸运
→ 解锁自动刮卡
→ 高风险卡 / 终局卡
→ Prestige 永久成长
```

游戏目标不是“公平抽奖”，而是让玩家不断产生：

👉 **“再刮一张就能翻盘 / 解锁 / 爆奖”的冲动**

# 三、核心循环

## 3.1 前 30 秒循环

1. 玩家洗盘子赚启动金。
2. 获得第一张低级刮刮卡的购买资金。
3. 购买低级刮刮卡。
4. 拖动刮开遮罩。
5. 获得小奖、亏损或惩罚。
6. 继续洗盘子或继续买卡。

## 3.2 中期循环

1. 玩家通过刮卡累积金币。
2. 解锁更贵、规则不同、风险更高的刮刮卡。
3. 用金币升级刮卡能力、幸运、倍率和自动化。
4. 通过更高等级卡追求爆奖。

## 3.3 后期循环

1. 自动刮卡开始承担基础收益。
2. 玩家挑战高风险卡和终局卡。
3. 本轮收益到达瓶颈后进行 Prestige。
4. 重置本轮进度，换取永久加成。
5. 下一轮更快进入高阶刮卡。

# 四、设计原则

## 4.1 刮刮卡是主玩法

刮刮卡是游戏的核心爽点。

洗盘子只是启动金来源，不是长期主玩法。

## 4.2 拖动刮开，不点击开奖

刮卡必须是：

```txt
购买卡片 → 生成结果 → 拖动擦除遮罩 → 渐进揭露 → 结算
```

禁止把刮刮卡做成点击按钮开奖。

## 4.3 节奏由系统控制

必须避免：

* 高速点击刷钱
* 手速决定收益
* 瞬间刮完整张卡
* 自动跳过核心揭露过程

正确体验是：

* 洗盘子是在准备启动资金
* 刮卡才是核心决策和爽点
* 升级和自动化逐步放大刮卡收益

# 五、阶段一：洗盘子启动金

## 5.1 定位

洗盘子用于让玩家获得第一笔资金。

它的目标是：

* 让玩家进入第一张刮刮卡
* 控制前期节奏
* 避免游戏变成纯点击器

## 5.2 入口与交互流程

洗盘子必须表现为左侧工作任务，而不是直接显示一个收益按钮。

基础流程：

```txt
左侧出现“日常工作”
→ 玩家点击日常工作
→ 桌子上生成一个脏盘子小物件
→ 玩家点击桌子上的脏盘子
→ 弹出 / 放大脏盘子的完整清洁视图
→ 玩家拖动海绵擦除污渍
→ 清洁进度达到 100%
→ 显示可领取金额
→ 玩家领取金币
→ 回到桌面，继续处理剩余脏盘子或等待下一次日常工作
```

## 5.3 日常工作卡片

左侧必须展示：

* 工作名称：日常工作
* 盘子费用：`$1`
* 等级：`等级 0`
* 等级进度条
* 工作图标：盘子 / 海绵

点击该卡片只负责开始工作和生成盘子，不直接结算金币。

阶段一日常工作等级规则：

* 最高等级：`等级 10`。
* 每个等级升到下一级需要的清洁盘子数，统一由 `apps/scratch-legend/lib/game-config.ts` 中的 `work.level.platesRequiredByLevel` 配置。
* 当前默认配置下，每一级先都需要清洁 `10` 个盘子；后续可按等级分别调整。
* 等级进度条只展示当前等级内进度，不展示“等级进度”文字。
* 阶段一等级收益由配置表约束，默认：
  * `等级 0/1`：每个盘子净收益 `$2`。
  * `等级 2`：每个盘子净收益 `$3`。
  * `等级 3`：每个盘子净收益 `$4`。
  * 后续等级继续由同一配置表扩展，不在组件内散写数值。

## 5.4 桌面脏盘子

日常工作开始后，桌面中央或可见区域出现一个小脏盘子。

规则：

* 盘子是可点击物件。
* 每生成一个脏盘子固定花费 `$1`，任何等级都不随机调整盘子价格。
* 游戏开局金币为 `$1`，保证玩家能生成第一个脏盘子。
* 短点击盘子后立即进入放大清洁视图。
* 长按盘子进入可拖动状态，并在移动后开始拖动；拖动应跟手、不卡顿。
* 桌面盘子悬浮时需要有轻微晃动反馈，长按进入拖动时需要有拿起阴影反馈。
* 未点击盘子前不能清洁。
* 玩家可以连续点击日常工作生成多个脏盘子；多个盘子可以在桌面上重叠，每个盘子独立携带本次工作奖励。
* 新脏盘子从桌子顶部飞入桌面，最终落点在桌面可操作区域内随机生成，不固定在中心点。

## 5.5 放大清洁视图

放大视图中必须包含：

* 大盘子
* 污渍层
* 海绵 / 清洁工具
* 日常工作说明浮层
* 奖励金额

玩家需要拖动海绵擦除污渍。

清洁完成前不允许领取奖励。

清洁视图中，鼠标悬浮到盘子污渍区域时显示为海绵 cursor；按住鼠标拖动才执行擦除。

清洁说明浮层只展示日常工作说明和中奖 / 碎盘概率，不展示清洁进度；玩家通过盘面污渍消失直接感知清洁程度。

## 5.6 清洁判定

推荐使用 Canvas 或等价遮罩：

```ts
ctx.globalCompositeOperation = 'destination-out'
```

清洁进度由污渍遮罩擦除比例决定，内部使用 `0~1` 的比例值：

```ts
cleanCompleteThreshold = 0.95

if (cleanRatio >= cleanCompleteThreshold) {
  canClaimReward = true
}
```

显示给玩家时仍然表现为 `100%` 完成。

## 5.7 行为规则

一次洗盘子动作必须有动作锁：

```ts
actionDuration = 400~800ms
```

收益在整只盘子清洁完成并领取时结算。等级 `0` 时不会出现碎盘：

```ts
if (workLevel < 1) {
  reward = +2
}
```

日常工作达到 `等级 1` 后引入轻惩罚，并通过电话告知玩家：

```ts
if (random < 0.9) {
  reward = +configuredLevelReward
} else if (gold >= 3) {
  reward = -3
  plateBroken = true
} else {
  reward = +configuredLevelReward
}
```

注意：

* 动作锁用于限制清洁采样和结算节奏。
* 收益必须在整只盘子清洁完成后结算。
* 不允许每次点击或每次擦拭都直接给钱。
* 如果金币不足以承担 `$3` 扣除后继续购买 `$1` 盘子，则本次不能生成碎盘惩罚，避免玩家无法继续。
* 碎盘结果需要有明确的盘子破碎反馈，并在领取时扣除金币。

## 5.8 电话与辅助道具提示

阶段一包含三类电话 / 解锁提示：

1. 垃圾桶解锁：
   * 辅助道具页默认只显示“未解锁”占位，不在桌面展示垃圾桶。
   * 当玩家累计清洁第 `3` 个盘子后，垃圾桶自动解锁，不消耗金币。
   * 解锁时在桌面中央播放“已解锁 垃圾桶”发光提示。
   * 解锁后垃圾桶显示在桌面右下角。
   * 脏盘子拖入垃圾桶区域时，垃圾桶打开；在垃圾桶上松开鼠标后，该脏盘子直接消失，垃圾桶自动关闭。

2. 碎盘风险电话：
   * 当日常工作达到 `等级 1` 后，电话响铃并弹出风险说明。
   * 风险说明浮层参考清洁说明卡片样式，展示 `90% +收益` 和 `10% -$3`。
   * 玩家点击“我知道了”后关闭该电话消息。

3. 刮刮乐解锁电话：
   * 左上角金币进度改为展示“累计赚到的金币 / 当前累计里程碑”。
   * 累计赚到 `$10` 后，电话响铃并提示已可解锁刮刮乐模式。
   * 同时播放一次刮刮乐解锁发光提示。
   * 阶段一只展示解锁提示，不实现刮刮乐玩法；刮刮乐玩法在阶段二构建。
   * 配置文件中需要支持继续声明后续累计里程碑，例如累计 `$50` 的下一功能占位，但当前阶段不提前实现对应玩法。

阶段一已落地数值统一收敛到：

```text
apps/scratch-legend/lib/game-config.ts
```

要求：

* 当前阶段已经实现的金币、奖励、等级、电话提示、垃圾桶解锁和交互节奏，统一以该配置文件为单一来源。
* 累计金币解锁也必须从该配置文件读取，不能在组件内写死 `10`、`50` 这类里程碑阈值。
* `game-config.ts` 只负责静态规则，不负责保存玩家当前进度。
* 如果后续调整阶段一数值，先更新本文档，再同步该配置文件和任务文档。

阶段一运行态存档结构统一收敛到：

```text
apps/scratch-legend/lib/game-save.ts
```

要求：

* 持久化时必须单独保存当前金币、累计赚到的金币、当前已解锁状态、电话消息已读状态、桌面盘子列表和当前工作阶段。
* 存档结构与静态配置分离，避免把“默认规则”和“当前进度”混在同一份对象里。
* 后续接本地存档、云存档或版本迁移时，统一以该存档结构为入口。

阶段一前端状态管理统一收敛到：

```text
apps/scratch-legend/lib/game-store.ts
```

要求：

* 前端运行态使用 `zustand` 管理。
* 本地持久化默认使用浏览器 `localStorage`。
* `game-store.ts` 负责状态读写和持久化接线，`game-save.ts` 负责存档结构与默认值，二者职责分离。

## 5.9 强制规则

* 动作未结束不能再次结算收益。
* 连续点击只能增强视觉反馈，不能增加结算次数。
* 收益不能与手速线性绑定。
* 玩家接近第一张卡价格时，只提示可购买，不自动跳转。
* 点击日常工作不结算。
* 点击脏盘子不结算。
* 只有清洁完成并领取时才增加金币。

## 5.10 节奏目标

```txt
30~60 秒 → 累计赚到 `$10` 并收到刮刮乐模式解锁提示
```

# 六、阶段二：低级刮刮卡

## 6.1 定位

第一张低级刮刮卡是玩家真正进入游戏主循环的入口。

它必须让玩家理解：

* 刮卡需要先购买。
* 奖励不稳定。
* 亏损是常态。
* 爆奖会改变下一步选择。

## 6.2 基础成本

```ts
basicCardUnlockGold = 10
```

## 6.3 结构

每张刮刮卡由两层组成：

### 底层：真实内容

* 奖励符号
* 惩罚符号
* 金额或倍率
* 结算结果

### 上层：刮层

* 灰白遮罩
* 厚涂层质感
* 可被擦除
* 显示刮痕和粉末反馈

## 6.4 交互

玩家操作：

* 鼠标按下 / 触摸按下：开始刮
* 移动：擦除遮罩
* 松开：停止刮

禁止点击直接开奖。

## 6.5 技术实现

推荐使用 Canvas：

```ts
ctx.globalCompositeOperation = 'destination-out'
```

擦除方式：

* 圆形 brush：20~40px
* 连续擦除
* 带轻微随机偏移
* 根据升级改变 brush 范围和擦除效率

## 6.6 判定

基础卡满足以下条件之一时结算：

```ts
if (allRequiredCellsRevealed || erasedPercent >= 70%) {
  settleCard()
}
```

注意：

* 结果在购买卡片时已经生成。
* 刮开过程只负责揭露。
* 不允许刮到最后一刻才随机结果。

# 七、刮刮卡规则

## 7.1 MVP 推荐：三格匹配

基础卡使用三格匹配：

```txt
[ ? ] [ ? ] [ ? ]
```

规则：

```txt
3 个相同符号 → 中奖
否则 → 亏损或无奖
```

## 7.2 基础奖励池

| 类型 | 概率 | 结果 |
| --- | --- | --- |
| 普通亏损 | 55% | 0 |
| 小奖 | 25% | 120 |
| 大奖 | 12% | 500 |
| 惩罚 | 5% | -50 或扣除一张卡 |
| 传说 | 3% | 2000 |

## 7.3 近中奖

必须高频出现“差一点”的形态：

```txt
金币 金币 空
星星 星星 惩罚
宝箱 空 宝箱
```

表现要求：

* 前两个格子让玩家产生期待。
* 最后一个格子揭露前停顿 0.3 秒。
* 未中奖时给出明确“差一点”的反馈。

## 7.4 惩罚符号

惩罚是《刮个爽》式风险感的重要来源。

基础惩罚可以是：

* 扣除少量金币。
* 本张卡收益归零。
* 下一张卡成本增加一次。

MVP 阶段惩罚必须轻，不允许让玩家直接崩盘。

## 7.5 弃卡与风险判断

中后期卡片可以允许玩家在完全揭露前弃卡。

目标是让玩家产生：

```txt
我继续刮可能中大奖，也可能刮出惩罚。
```

MVP 初期可以不做弃卡，但文档和架构必须为该能力预留。

# 八、卡片进阶系统

## 8.1 卡片等级

不同卡片必须拥有不同成本、规则和风险。

| 卡片 | 成本 | 定位 | 特点 |
| --- | --- | --- | --- |
| 低级卡 | 100 | 入门循环 | 三格匹配，低风险 |
| 银色卡 | 500 | 中期收益 | 更多格子，更高小奖 |
| 金色卡 | 2000 | 爆点追求 | 高奖池，高亏损 |
| 诅咒卡 | 5000 | 高风险 | 奖励极高，惩罚明显 |
| 终局卡 | 20000 | 本轮目标 | 稀有大奖，推动 Prestige |

## 8.2 解锁方式

卡片解锁由金币、累计刮卡次数和升级等级共同决定。

示例：

```ts
silverCardUnlocked = totalCardsScratched >= 10
goldCardUnlocked = totalCardsScratched >= 30 && maxSingleWin >= 500
cursedCardUnlocked = prestigeCount >= 1 || totalCardsScratched >= 80
```

## 8.3 设计要求

* 新卡必须改变玩家选择，而不是只提高数字。
* 高级卡要让玩家感到“贵，但可能翻盘”。
* 高风险卡必须同时提供爆奖和惩罚。

# 九、升级系统

## 9.1 定位

升级系统让玩家从“手动刮卡”逐步成长为“更快、更幸运、更自动化”。

## 9.2 升级项

| 升级 | 效果 |
| --- | --- |
| 刮刀宽度 | brush 半径 +2 |
| 刮刀力度 | 每次移动擦除更多遮罩 |
| 幸运值 | 小奖及以上概率提升 |
| 奖金倍率 | 正收益奖励增加 |
| 风险感知 | 提前显示部分惩罚线索 |
| 折扣券 | 刮刮卡购买成本降低 |
| 自动刮卡 | 解锁自动化队列 |

## 9.3 升级原则

* 前期升级主要改善手感。
* 中期升级提升收益和幸运。
* 后期升级解锁自动化和风险控制。
* 不允许出现无限正反馈导致金币失控。

# 十、自动刮卡

## 10.1 解锁条件

自动刮卡不能前期开放。

建议条件：

```ts
autoScratchUnlocked = totalCardsScratched >= 50 && ownedUpgrade("自动刮卡")
```

## 10.2 行为

自动刮卡必须模拟真实刮卡节奏：

* 自动购买指定卡片。
* 自动按路径擦除遮罩。
* 自动结算结果。
* 保留动画和时间消耗。

## 10.3 节奏

```ts
autoScratchInterval = 2~5 秒 / 张
```

## 10.4 强制规则

* 不允许瞬间批量开奖。
* 自动化收益不能完全替代玩家决策。
* 高风险卡默认不自动刮，除非玩家手动开启。

---

# 十一、高风险卡与终局卡

## 11.1 高风险卡

高风险卡用于制造后期刺激。

特征：

* 成本高。
* 大奖高。
* 惩罚明显。
* 允许弃卡或风险判断。

## 11.2 终局卡

终局卡是每轮 Prestige 前的追求目标。

它的目标不是稳定收益，而是：

* 制造本轮高潮。
* 触发大额奖励。
* 推动玩家进入下一轮永久成长。

---

# 十二、Prestige 永久成长

## 12.1 定位

Prestige 是后期核心循环。

玩家在本轮进度变慢后，可以重置：

* 当前金币
* 已解锁卡片
* 本轮升级
* 自动化队列

换取永久点数。

## 12.2 永久点数

永久点数可以提升：

| 永久升级 | 效果 |
| --- | --- |
| 起始金币 | 新轮次更快买第一张卡 |
| 永久幸运 | 全局提高稀有奖励概率 |
| 永久倍率 | 全局奖励倍率 |
| 起始刮刀 | 新轮次更快刮开 |
| 自动化保留 | 新轮次更早解锁自动刮卡 |

## 12.3 节奏目标

```txt
第一次 Prestige：10~20 分钟
后续 Prestige：更快回到中高阶卡片
```

---

# 十三、前 10 分钟体验目标

## 0~1 分钟

* 玩家学会洗盘子。
* 玩家买到第一张低级刮刮卡。

## 1~3 分钟

* 玩家完成多张低级卡。
* 出现至少一次小奖或近中奖。

## 3~5 分钟

* 玩家开始升级刮卡能力或幸运。
* 玩家看到下一等级卡片目标。

## 5~10 分钟

* 玩家解锁第二种卡片。
* 自动化目标开始出现。
* 玩家形成“买卡 - 刮卡 - 升级 - 买更贵卡”的循环。

---

# 十四、防崩机制

必须避免：

* 玩家无限赚钱。
* 玩家连续亏损到无法继续。
* 惩罚过重导致早期挫败。
* 自动化绕过刮卡过程。
* 高级卡只是在数值上更贵。

## 14.1 连败补偿

```ts
if (loseStreak >= 3) {
  smallWinRate += 0.03
}
```

## 14.2 保底机制

```ts
if (loseStreak >= 6) {
  result >= smallWin
}
```

## 14.3 破产保护

如果玩家金币低于低级卡成本，必须始终能通过洗盘子重新获得启动金。

---

# 十五、MVP 最低标准

MVP 必须具备：

* 洗盘子赚启动金。
* 购买低级刮刮卡。
* 拖动擦除遮罩。
* 渐进揭露结果。
* 三格匹配或等价刮卡规则。
* 中奖、亏损和轻惩罚。
* 升级刮刀或幸运。
* 至少两种卡片目标。

缺少“拖动刮卡”和“买卡继续刮”的循环，这个游戏就不成立。

---

# 十六、开发优先级

1. 洗盘子启动金。
2. 低级刮刮卡购买与拖动刮开。
3. 三格匹配与基础奖池。
4. 近中奖和轻惩罚。
5. 刮刀 / 幸运升级。
6. 第二种卡片。
7. 自动刮卡。
8. 高风险卡。
9. Prestige。

---

👉 核心认知：

**洗盘子是门票，刮刮卡是主菜，升级和 Prestige 是让玩家一直刮下去的理由。**
