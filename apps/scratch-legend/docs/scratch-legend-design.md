# 刮出传说（Scratch Legend）- 核心玩法设计

---

# 一、核心定位

本游戏对标《刮个爽》的核心骨架：

```txt
洗盘子赚启动金
→ 买第一张低级刮刮卡
→ 拖动刮卡
→ 中奖 / 亏损 / 惩罚
→ 买更多不同卡
→ 升级刮卡能力与幸运
→ 解锁自动刮卡
→ 高风险卡 / 终局卡
→ Prestige 永久成长
```

游戏目标不是“公平抽奖”，而是让玩家不断产生：

👉 **“再刮一张就能翻盘 / 解锁 / 爆奖”的冲动**

* 中央木桌视觉可参考当前蓝图图 1 的更厚桌沿、像素切角、自然木纹、轻微露出的短桌腿和右侧投影关系，但不要求逐像素复刻。

## 2.1 UI 贴图风格规范

Scratch Legend 的 UI 贴图使用统一的“清晰像素 + 轻量手绘质感”方向，目标是让桌面物件、按钮、特效和状态栏像同一套游戏资产，而不是 CSS 临时块。

通用风格：

* 贴图保持像素边缘、块面高光和少量颗粒，不使用写实照片、复杂渐变或高透明玻璃质感。
* 主色维持暖木桌、蓝白瓷盘、金色收益、红色风险、暗色面板的对比关系；避免整屏只有单一色系。
* 资产需要适合小尺寸显示，图标轮廓必须比细节优先，文字不应依赖贴图本身承载。
* UI 文字必须保持可读，按钮文案优先使用短词；当中文过长时，应先缩短文案或扩大按钮内容区。

贴图类别约定：

* 桌面与物件：木桌、桌沿、桌腿、电话、垃圾桶、桌面刮刮卡和盘子使用完整物件贴图，CSS 只负责定位、投影和交互状态。
* 刮刮卡：卡面、桌面小卡、顶部风景、结果格纸纹、图案符号和银色覆盖层使用贴图；遮罩范围只覆盖真实可刮区域。
* 洗盘子：瓷盘使用纯瓷盘主体贴图，污渍层使用 canvas 叠加菜渍、面条、残渣 sprite，海绵 cursor 使用普通态和按压态两张贴图。
* HUD 与进度：金币、盘子、刮卡次数等状态图标使用小像素图标；进度条使用 track / fill 贴图；所有等级进度只展示进度条，不显示进度数字、分母或 `MAX`。
* 按钮与气泡：主要按钮使用金色像素按钮贴图，禁用态使用低亮度或灰化状态；金币增减气泡分别使用金色和红色贴图，并靠右避开当前金额。
* 特效：中奖闪光、刮出图案闪烁、盘子成功闪光、失败冲击和破盘裂纹使用短促 sprite / 贴图叠层；特效只表达已经揭露或已经结算的结果，不提前暗示未揭露结果。

资产维护约定：

* 项目贴图统一放在 `apps/scratch-legend/public/ui-textures/`。
* 预览图可以保留在同目录，命名使用 `preview-*`，用于人工检查风格一致性。
* 新贴图应优先复用当前像素密度、描边厚度、暖色高光和暗色投影规则。
* 后续新增 UI 贴图时，应先确认是否属于已定义玩法阶段；只改变视觉表现、不改变玩法数值或结算规则时，可以作为当前阶段视觉优化处理。

# 三、核心循环

## 3.1 前 30 秒循环

1. 玩家洗盘子赚启动金。
2. 获得第一张低级刮刮卡的购买资金。
3. 购买低级刮刮卡。
4. 拖动刮开遮罩。
5. 获得小奖、亏损或惩罚。
6. 继续洗盘子或继续买卡。

## 3.2 中期循环

1. 玩家通过刮卡累积金币。
2. 解锁更贵、规则不同、风险更高的刮刮卡。
3. 用金币升级刮卡能力、幸运、倍率和自动化。
4. 通过更高等级卡追求爆奖。

## 3.3 后期循环

1. 自动刮卡开始承担基础收益。
2. 玩家挑战高风险卡和终局卡。
3. 本轮收益到达瓶颈后进行 Prestige。
4. 重置本轮进度，换取永久加成。
5. 下一轮更快进入高阶刮卡。

# 四、设计原则

## 4.1 刮刮卡是主玩法

刮刮卡是游戏的核心爽点。

洗盘子只是启动金来源，不是长期主玩法。

## 4.2 拖动刮开，不点击开奖

刮卡必须是：

```txt
购买卡片 → 生成结果 → 拖动擦除遮罩 → 渐进揭露 → 结算
```

禁止把刮刮卡做成点击按钮开奖。

## 4.3 节奏由系统控制

必须避免：

* 高速点击刷钱
* 手速决定收益
* 瞬间刮完整张卡
* 自动跳过核心揭露过程

正确体验是：

* 洗盘子是在准备启动资金
* 刮卡才是核心决策和爽点
* 升级和自动化逐步放大刮卡收益

# 五、阶段一：洗盘子启动金

## 5.1 定位

洗盘子用于让玩家获得第一笔资金。

它的目标是：

* 让玩家进入第一张刮刮卡
* 控制前期节奏
* 避免游戏变成纯点击器

## 5.2 入口与交互流程

洗盘子必须表现为左侧工作任务，而不是直接显示一个收益按钮。

基础流程：

```txt
左侧出现“日常工作”
→ 玩家点击日常工作
→ 桌子上生成一个脏盘子小物件
→ 玩家点击桌子上的脏盘子
→ 弹出 / 放大脏盘子的完整清洁视图
→ 玩家拖动海绵擦除污渍
→ 清洁进度达到 100%
→ 显示可领取金额
→ 玩家领取金币
→ 回到桌面，继续处理剩余脏盘子或等待下一次日常工作
```

## 5.3 日常工作卡片

左侧必须展示：

* 工作名称：日常工作
* 盘子费用：`$1`
* 等级：`等级 0`
* 等级进度条
* 工作图标：盘子 / 海绵

点击该卡片只负责开始工作和生成盘子，不直接结算金币。

阶段一日常工作等级规则：

* 最高等级：`等级 10`。
* 每个等级升到下一级需要的清洁盘子数，统一由 `apps/scratch-legend/lib/game-config.ts` 中的 `work.level.platesRequiredByLevel` 配置。
* 当前默认配置下，每一级先都需要清洁 `10` 个盘子；后续可按等级分别调整。
* 等级进度条只展示当前等级内进度，不展示“等级进度”文字。
* 阶段一等级收益由配置表约束，默认：
  * `等级 0/1`：每个盘子净收益 `$2`。
  * `等级 2`：每个盘子净收益 `$3`。
  * `等级 3`：每个盘子净收益 `$4`。
  * 后续等级继续由同一配置表扩展，不在组件内散写数值。

## 5.4 桌面脏盘子

日常工作开始后，桌面中央或可见区域出现一个小脏盘子。

规则：

* 盘子是可点击物件。
* 每生成一个脏盘子固定花费 `$1`，任何等级都不随机调整盘子价格。
* 游戏开局金币为 `$1`，保证玩家能生成第一个脏盘子。
* 短点击盘子后立即进入放大清洁视图。
* 长按盘子进入可拖动状态，并在移动后开始拖动；拖动应跟手、不卡顿。
* 桌面盘子悬浮时需要有轻微晃动反馈，长按进入拖动时需要有拿起阴影反馈。
* 未点击盘子前不能清洁。
* 玩家可以连续点击日常工作生成多个脏盘子；多个盘子可以在桌面上重叠，每个盘子独立携带本次工作奖励。
* 只要玩家金币足够，即使桌面上已经有未处理刮刮卡，也允许继续生成新的脏盘子；洗盘子始终是可回到的启动金来源。
* 新脏盘子从桌子顶部飞入桌面，最终落点在桌面可操作区域内随机生成，不固定在中心点。

## 5.5 放大清洁视图

放大视图中必须包含：

* 大盘子
* 污渍层
* 海绵 / 清洁工具
* 日常工作说明浮层
* 奖励金额

玩家需要拖动海绵擦除污渍。

清洁完成前不允许领取奖励。

清洁视图中，鼠标悬浮到盘子污渍区域时显示为海绵 cursor；按住鼠标拖动才执行擦除。

清洁说明浮层只展示日常工作说明和中奖 / 碎盘概率，不展示清洁进度；玩家通过盘面污渍消失直接感知清洁程度。

每个桌面脏盘子需要独立记录已经擦除过的清洁点位。玩家清洁一半后关闭放大视图，再次打开同一只盘子时，污渍层必须恢复到上次已清洁的状态，不能要求重新清洁。

清洁完成前，界面不能提前展示本次盘子的实际结算金额，也不能通过领取按钮或概率说明暴露本次预生成结果。只有清洁达到可领取状态后，才显示本次实际获得或扣除的金额。

## 5.6 清洁判定

推荐使用 Canvas 或等价遮罩：

```ts
ctx.globalCompositeOperation = 'destination-out'
```

清洁进度由污渍遮罩擦除比例决定，内部使用 `0~1` 的比例值：

```ts
cleanCompleteThreshold = 0.95

if (cleanRatio >= cleanCompleteThreshold) {
  canClaimReward = true
}
```

显示给玩家时仍然表现为 `100%` 完成。

## 5.7 行为规则

一次洗盘子动作必须有动作锁：

```ts
actionDuration = 400~800ms
```

收益在整只盘子清洁完成并领取时结算。等级 `0` 时不会出现碎盘：

```ts
if (workLevel < 1) {
  reward = +2
}
```

日常工作达到 `等级 1` 后引入轻惩罚，清洁说明开始展示 `90%` 正收益与 `10%` 碎盘风险：

```ts
if (random < 0.9) {
  reward = +configuredLevelReward
} else if (gold - configuredLevelReward >= 1) {
  reward = -configuredLevelReward
  plateBroken = true
} else {
  reward = +configuredLevelReward
}
```

注意：

* 动作锁用于限制清洁采样和结算节奏。
* 收益必须在整只盘子清洁完成后结算。
* 不允许每次点击或每次擦拭都直接给钱。
* 碎盘扣款金额与当前工作等级本应获得的正收益一致，不再使用固定扣款。
* 如果金币不足以承担“当前等级正收益”等额扣除后继续购买 `$1` 盘子，则本次不能生成碎盘惩罚，避免玩家无法继续。
* 碎盘结果需要有明确的盘子破碎反馈，并在领取时扣除金币。
* 碎盘风险电话不在升到 `等级 1` 时提前出现；玩家第一次真实遇到碎盘并结算后，电话才响铃提示风险。

## 5.8 电话与辅助道具提示

阶段一包含三类电话 / 解锁提示：

1. 垃圾桶解锁：
   * 辅助道具页默认只显示“未解锁”占位，不在桌面展示垃圾桶。
   * 左上角进度统一展示“当前段熟练度 / 当前段目标”，开局为 `0/10`。
   * 当玩家刷满第一段 `10` 熟练度后，进度进入下一目标并显示 `0/50`，垃圾桶购买资格自动解锁。
   * 解锁时在桌面中央播放“已解锁 垃圾桶”发光提示。
   * 垃圾桶价格为 `$2`，玩家需要在辅助道具页购买后才会显示在桌面右下角。
   * 已购买垃圾桶后，脏盘子或未打开的刮刮卡拖入垃圾桶区域时，垃圾桶打开；在垃圾桶上松开鼠标后，该对象直接消失，垃圾桶自动关闭。

2. 碎盘风险电话：
   * 当玩家第一次真实遇到碎盘并完成结算后，电话响铃并提示风险。
   * 电话话术方向为：“没有人能一直顺风顺水。刚才那只盘子碎了，但你还在桌边。从等级 1 开始，日常工作会有 10% 概率碎盘。”
   * 风险说明浮层参考清洁说明卡片样式，展示 `90% +当前等级收益` 和 `10% -当前等级收益`。
   * 玩家点击“我知道了”后关闭该电话消息。

3. 刮刮乐解锁电话：
   * 第二段 `50` 熟练度刷满后，电话响铃并分三句提示：
     * “你对自己的日常工作感到厌烦吗？”
     * “我这里有个好东西，非常适合你......”
     * “刮刮卡！”
   * 三句电话结束后，桌面中央播放“刮刮卡！”解锁发光提示。
   * 提示完成前，左侧不展示刮刮卡购买入口；提示完成后才出现阶段二第一张刮刮卡。
   * 后续熟练度目标继续按当前段数值展示，例如 `19/300`、`25/500`、`125/1000`、`500/3000`。

阶段一已落地数值统一收敛到：

```text
apps/scratch-legend/lib/game-config.ts
```

要求：

* 当前阶段已经实现的金币、奖励、等级、电话提示、垃圾桶解锁和交互节奏，统一以该配置文件为单一来源。
* 熟练度解锁也必须从该配置文件读取，不能在组件内写死 `3`、`10`、`50`、`100`、`250`、`1000` 这类分段目标值。
* `game-config.ts` 只负责静态规则，不负责保存玩家当前进度。
* 如果后续调整阶段一数值，先更新本文档，再同步该配置文件和任务文档。

阶段一运行态存档结构统一收敛到：

```text
apps/scratch-legend/lib/game-save.ts
```

要求：

* 持久化时必须单独保存当前金币、累计赚到的金币、当前已解锁状态、电话消息已读状态、桌面盘子列表和当前工作阶段。
* 存档结构与静态配置分离，避免把“默认规则”和“当前进度”混在同一份对象里。
* 后续接本地存档、云存档或版本迁移时，统一以该存档结构为入口。

阶段一前端状态管理统一收敛到：

```text
apps/scratch-legend/lib/game-store.ts
```

要求：

* 前端运行态使用 `zustand` 管理。
* 本地持久化默认使用浏览器 `localStorage`。
* `game-store.ts` 负责状态读写和持久化接线，`game-save.ts` 负责存档结构与默认值，二者职责分离。

## 5.9 强制规则

* 动作未结束不能再次结算收益。
* 连续点击只能增强视觉反馈，不能增加结算次数。
* 收益不能与手速线性绑定。
* 玩家接近第一张卡价格时，只提示可购买，不自动跳转。
* 点击日常工作不结算。
* 点击脏盘子不结算。
* 只有清洁完成并领取时才增加金币。

## 5.10 节奏目标

```txt
30~60 秒 → 推进熟练度到第二段 `10` 并完成刮刮卡电话解锁
```

# 六、阶段二：第一张低级安全卡

## 6.1 定位

阶段二的任务不是“把所有刮卡花样一次做完”，而是先建立《刮个爽》最核心的主循环入口：

```txt
买卡
→ 拖动刮开
→ 看见结果
→ 领奖或亏损
→ 决定继续买卡还是回去洗盘子
```

第一张卡必须承担教学职责，让玩家立刻理解三件事：

* 刮刮卡需要先买。
* 结果在买下那一刻就已经决定。
* 拖动揭露本身就是主玩法的一部分。

## 6.2 解锁位置

阶段二入口继续沿用阶段一节奏：

```ts
proficiencyMilestones = [10, 50, 300, 500, 1000, 3000, 5000, 10000]
```

这里的数值都是“当前段需要刷满的本地熟练度”，不是存档里的累计熟练度，也不宣称等于《刮个爽》原始数值。玩家先完成 `0/10` 解锁垃圾桶，再进入 `0/50`；第二段刷满并完成电话提示后，才显示刮刮卡入口。

熟练度采用分段展示，对齐玩家直觉中的“每次解锁后从 0 重新追下一段”：

```txt
0/10 → 达成后进入 0/50
0/50 → 达成后进入 0/300
0/300 → 达成后进入 0/500
0/500 → 达成后进入 0/1000
0/1000 → 达成后进入 0/3000
0/3000 → 达成后进入 0/5000
0/5000 → 达成后进入 0/10000
```

后期熟练度路线按“先自动化，再风险管理，再高风险挑战”承接，避免 `1000` 过早成为临时终点：

| 当前段目标 | 累计真实阈值 | 解锁 / 作用 |
| ---: | ---: | --- |
| `10` | `10` | 垃圾桶，清理桌面与前期效率工具。 |
| `50` | `60` | 刮刮卡入口，开启核心随机收益。 |
| `300` | `360` | 升级工具，开启刮除范围、刮卡运气和预留成长。 |
| `500` | `860` | 三连胜出，开启更高赔率但仍偏普通的刮刮卡。 |
| `1000` | `1860` | 自动刮刮机资格，开始从手动刷卡过渡到自动化。 |
| `3000` | `4860` | 自动机稳定期，承接容量、速度和可处理票池等自动机升级。 |
| `5000` | `9860` | 高风险预热，承接风险管理、保险、止损或自动机强化。 |
| `10000` | `19860` | 步步加码，解锁真正的高风险逐层挑战票。 |

`1000` 不是最终目标，也不解锁“步步加码”；它只代表自动化入口。`3000` 和 `5000` 是后期路线承接节点，后续实现可以逐步补充具体玩法，但玩家可见 UI 不应写“设计中”“后续目标”这类开发状态文案。

每个目标达成后只切换下一目标，不清空金币、卡片等级、工作等级、贷款或桌面物件；存档里的总熟练度继续累计，用于计算真实解锁阈值。

如果一次收益跨过当前目标，溢出的熟练度带入下一段。例如当前为 `2/3`，本次获得 `$2` 时，应解锁下一段并显示 `1/10`。

电话提示和熟练度推进必须解耦：熟练度达到当前目标后应立刻进入下一段进度；电话只作为待领取的解锁提示。玩家不点击电话时，对应功能仍暂不展示，但玩家可以继续洗盘子或处理当前已开放玩法来积攒后续熟练度。

当最后一个已配置熟练度段也刷满后，左上角不继续显示固定的 `10000/10000` 造成卡死错觉；进度条保持满格，并用中性的已完成文案表达当前路线已达成，不能暗示一个尚未实现的“最终挑战”已经存在。

第一张低级安全卡本地名称和价格：

```ts
basicSafeCardName = '成双入对'
basicSafeCardPrice = 10
```

玩家必须同时满足“第二段熟练度达到 `10`、刮刮卡电话提示完成、当前金币不少于 `$10`”，才可以购买第一张低级安全卡。购买类金额展示使用白色，不使用红色，避免和亏损 / 惩罚语义混淆。点击购买后，刮刮卡像盘子一样飞入桌面；未打开的桌面卡片可长按拖动，点击桌面上的卡片后，才进入放大刮卡视图。

所有已解锁刮刮卡必须遵守同一套桌面规则：

* 只要金币足够，玩家可以连续点击同一张卡或不同卡的购买入口。
* 每次购买都独立扣费、独立预生成结果，并生成一张新的桌面小卡。
* 新买的卡不能覆盖旧卡，桌面可以同时存在多张待处理刮刮卡。
* 所有未打开的桌面刮刮卡都可以自由拖动。
* 所有桌面刮刮卡都可以打开、关闭、结算，或在垃圾桶已购买后拖入垃圾桶丢弃。
* 每张桌面刮刮卡需要独立记录已经刮开的点位。玩家刮到一半后关闭放大视图，再次打开同一张卡时，遮罩必须恢复到上次已刮开的状态，不能重新盖回未刮状态。

桌面刮刮卡图标是小尺寸物件，不应占据盘子同等视觉重量；推荐约为桌面盘子的 `3/4` 大小，并保留迷你标题、图片窗口和刮层条，让玩家一眼知道它是刮刮卡而不是大面板。

## 6.3 通用刮刮卡等级规则

所有刮刮卡都拥有独立等级和等级进度，默认从 `等级 0` 开始。

通用规则：

* 每种卡独立记录自己的等级和等级进度。
* 每结算 1 张某类型刮刮卡，该类型卡获得 1 点等级进度。
* 丢弃未结算卡片不增加等级进度。
* 等级进度只影响该类型卡的配置化成长，不改变卡片价格，也不改变核心中奖规则。
* 购买时必须按当时的卡片等级预生成结果；卡片已购买后，该卡不会因为后续升级 retroactively 改写 payout。
* 左侧目录入口展示每张已解锁卡的当前等级和当前段进度条。
* 放大刮卡信息面板展示本张卡购买时使用的等级、奖池和规则。
* 所有可见 UI 只允许展示当前等级和等级进度条，不允许显示等级进度数字、分母或 `MAX` 文案。

默认等级命名采用 0 起算：`等级 0` 为基础效果，升一级后显示 `等级 1`。后续所有新卡都必须沿用该规则，禁止出现某张卡从等级 1 起步、另一张卡从等级 0 起步的混用。

## 6.4 卡片结构

每张刮刮卡都由两层组成：

### 底层：预生成结果层

* 奖励符号
* 空白 / 亏损结果
* 后续阶段才会接入的惩罚符号
* 本张卡的结算信息
* 同一中奖 / 未中奖档位的符号槽位必须在购卡预生成时随机排列，避免连续卡片出现固定图案位置。

### 上层：可擦除刮层

* 灰白遮罩
* 可被拖动擦除
* 有连续刮痕反馈，笔刷边缘应带有不规则刮屑 / 破碎感，避免像普通橡皮擦抹除
* 不能通过单击跳过
* 已达到有效刮开比例后，剩余遮罩必须让位，底层结果需要完整可读
* 单个结果格达到揭露阈值后，该格残留遮罩也必须自动清空，避免图案已经判定揭露但仍被银色碎片挡住
* 达到可结算状态前，界面不能通过结算按钮、侧边说明或其他 UI 提前展示本张卡的实际 payout；未揭晓时只允许展示奖池、概率、规则和占位金额。
* 刮层笔刷大小必须由 `game-config.ts` 配置控制；阶段二默认使用较小笔刷，避免玩家几下就刮完整张卡。

## 6.5 交互原则

玩家操作固定为：

* 按下：开始刮
* 拖动：持续擦除
* 松开：停止擦除

强制规则：

* 禁止点击直接开奖。
* 禁止在最后一刻才随机结果。
* 刮开过程只负责揭露，不负责重新抽奖。
* 打开的刮刮卡可以像清洁盘子一样，点击卡片以外的桌面区域关闭回桌面。
* 已经刮到可结算状态的卡刷新页面后，不允许重新显示未刮遮罩。

## 6.6 当前推荐的首张卡规则

按高置信骨架，第一张卡优先采用“低风险教学卡”而不是重惩罚卡。

当前推荐规则：

```txt
3 格结果区
必须刮出任意一对才中奖
没有任意一对就是 $0
```

这样更贴近《刮个爽》早期“先教玩家刮和领奖，再逐步引入风险”的节奏。

## 6.7 阶段二本地数值原则

阶段二只锁定数值原则，不锁定《刮个爽》原作精确概率：

* 未成对应允许出现，并结算 `$0`。
* 成对后按符号类型给钱。
* 偶尔出现明显高于卡价的成对大奖，提供翻盘期待。
* 第一张卡不承担重惩罚职责，重惩罚放到阶段三。

阶段二本地 MVP 奖池固定为：

| 类型 | 真实概率 | 展示概率 | 结算金额 |
| --- | --- | --- | --- |
| 未成对 | 72% | 不展示 | `$0` |
| 火焰成对 | 20% | 50% | `$10` |
| 纸钞成对 | 6% | 40% | `$25` |
| 钱袋成对 | 2% | 10% | `$50` |

注意：

* 卡片侧栏按视觉参考展示成对奖项：火焰 `50% / $10`、纸钞 `40% / $25`、钱袋 `10% / $50`。
* 侧栏展示概率是给玩家看的票面概率，不等于真实抽奖概率。
* 真实概率和展示概率都必须从 `game-config.ts` 读取，组件内不散写。
* 没有一对时直接结算 `$0`，不是额外惩罚。
* 阶段二不启用惩罚符号、弃卡和风险卡，但数据结构可以预留扩展位。

结算判定可采用“关键区域揭露完成”或“有效刮开比例达标”的本地实现，但这属于 Scratch Legend 的实现细节，不视为原作规则声明。

## 6.8 成双入对等级

第一张低级安全卡“成双入对”本身也有等级与等级进度，用于让玩家在反复刮同一张卡时获得成长反馈。

阶段二先实现该卡的本地等级规则：

* “成双入对”默认 `等级 0`，中奖金额沿用阶段二基础奖池：火焰 `$10`、纸钞 `$25`、钱袋 `$50`。
* 累计结算 3 张“成双入对”后升到 `等级 1`，中奖金额按 `1.3` 倍向下取整：火焰 `$13`、纸钞 `$32`、钱袋 `$65`。
* `等级 1` 之后继续使用配置表预留等级门槛和倍率，但阶段二 UI 只需要明确展示当前等级、当前段进度和当前等级奖池。

等级进度配置统一收敛到：

```text
apps/scratch-legend/lib/game-config.ts
```

要求：

* 组件内不得散写 `等级 1` 的金额。
* 购买时必须按当时的卡片等级预生成结果，后续升级不会 retroactively 改写桌面上已购买卡片的 payout。
* 结算时才增加“成双入对”的等级进度，丢弃未结算卡片不增加等级进度。

## 6.9 电话展示规则

阶段二起，电话提醒不再自动弹出完整消息。电话只负责在桌面上响铃提示；玩家必须点击电话后，才展示当前电话消息。

规则：

* 有待处理电话时，电话图标响铃。
* 点击电话才展开电话消息。
* 没有待处理电话时点击电话不弹消息。
* 电话消息关闭或完成后，电话停止展示对应消息。

## 6.10 破产贷款兜底

阶段二允许加入一次“无法继续游戏”的破产兜底电话，避免玩家因为买不起盘子、桌面也没有任何可处理对象而卡死。

触发条件：

* 当前金币为 `$0`。
* 桌面没有脏盘子。
* 桌面或放大视图中没有未结算刮刮卡。
* 即使已经有尚未偿还的贷款，只要再次无法继续，也允许电话追加下一笔贷款。

电话文案方向：

* 系统“大发慈悲”给玩家贷款。
* 玩家只要签字即可，签字交互表现为一个指纹按钮。
* 电话必须提示：“温馨提示，我们的贷款利率是 6000%”。
* 电话还需要说明：为了防止玩家不还，右上角会出现贷款按钮，可以查看当前贷款。

贷款数值：

```ts
loanPrincipal = 5
loanInterestRateLabel = '6000%'
loanRepaymentAmount = 300
```

点击指纹签字后：

* 玩家立刻获得 `$5`。
* 生成一条待偿还贷款。
* 右上角显示贷款按钮。
* 点击贷款按钮打开贷款列表。
* 玩家偿还贷款时，需要有明确的“债务解除”反馈：金币扣款飘字、对应贷款盖章并滑出、惩罚解除提示。
* 如果偿还的是最后一笔贷款，贷款账本自动收起，右上角贷款按钮在消失前做一次短暂弹跳。

贷款列表按模板循环生成，当前阶段先预留三条：

| 贷款 | 描述 | 偿还金额 |
| --- | --- | --- |
| 贷款 #1 | 每 20 张卡中会有 1 张不是你要的卡 | `$300` |
| 贷款 #2 | 你的刮除范围降低了 1 | `$300` |
| 贷款 #3 | 刮刮机器人的速度降低了 30% | `$300` |

阶段二只实现贷款列表、签字到账和偿还勾销，不启用这些贷款描述对应的真实惩罚效果。贷款模板需要在 `apps/scratch-legend/lib/game-config.ts` 中带结构化 `penalty` 配置，当前三类分别为：

* `wrong-card-every-n`：每隔固定购卡次数替换一次购卡结果，当前配置为 `everyCards: 20`，用于让第 20 张购入的刮刮卡强制成为错版未中结果。
* `scratch-brush-radius-delta`：影响刮除范围，当前配置为 `delta: -1`，用于让刮卡笔刷半径在升级工具加成后再减少 1。
* `automation-speed-multiplier`：影响自动化速度，当前配置为 `multiplier: 0.7`。

其中 `wrong-card-every-n` 和 `scratch-brush-radius-delta` 当前已启用；其他惩罚配置保持 `enabled: false`，只作为后续启用真实惩罚效果的配置入口。后续如果有新的贷款点子，优先继续追加新的 `penalty.type` 和模板配置，再接入对应玩法计算。

偿还反馈按惩罚类型展示短提示：

* `wrong-card-every-n`：显示“错卡惩罚解除”。
* `scratch-brush-radius-delta`：显示“刮除范围恢复”。
* 未启用或未接入的惩罚类型：显示“债务已偿清”。

## 6.11 熟练度 300：升级工具解锁

当玩家刷满升级工具段 `300` 熟练度后，左上角熟练度进入下一目标并展示 `0/500`，同时触发新的电话响铃。电话仍然遵守阶段二起的规则：只响铃，不自动弹出消息；玩家点击电话后才进入对话。

升级工具电话按以下顺序播放：

1. “刮起来是不是有点费力？”
2. “别担心，我这正好有你要的东西——升级！”
3. “升级到刮工具，刮起卡来轻轻松松！”
4. “听我的，现在不升级，以后那些卡的涂层你根本刮不动......”
5. “而且你还能升级运气呢，多稀罕呀！”

电话结束后，桌面中央播放“已解锁 升级工具”发光提示，右侧预留区域切换为升级工具面板。

阶段二后续先配置以下基础升级工具：

| 工具 | 初始价格 | 初始等级 / 力量 | 当前阶段作用 |
| --- | --- | --- | --- |
| 刮卡运气 | `$200` | 等级 0 | 提升基础刮刮卡和高赔率票的真实中奖权重。 |
| 刮除范围 | `$100` | 等级 0 | 提升刮层与盘子清洁笔刷半径，改善手动操作效率。 |
| 铜币 | `$500` | 力量 1 | 预留奖励放大 / 特殊资源入口，当前先作为可配置工具展示。 |

升级工具配置要求：

* 工具列表、价格、等级上限、每级效果都必须由 `game-config.ts` 配置。
* 工具购买价格必须按当前等级递增，默认倍率表为 `1 / 1.5 / 2.25 / 3.4 / 5.1 / 7.6 / 11.4 / 17 / 25.5 / 38`；超过倍率表时沿用最后一档。
* “刮除范围”同时影响刮刮卡刮层和洗盘子污渍层，但洗盘子的基础清洁范围应独立配置并明显大于刮刮卡基础刮除范围。
* “刮除范围”只能提升手感，不能让玩家一键跳过刮开过程，也不能让洗盘子重新变成长期主玩法。
* “刮卡运气”影响真实奖池概率，但必须同时保留真实概率和票面展示概率的区分。
* “刮卡运气”每级从未中奖权重中转移 `3%` 到中奖档位，中奖档位按原本中奖概率比例分配；未中奖总概率最低保留 `45%`。
* “刮卡运气”当前只影响“成双入对”和“三连胜出”，不影响“险中求财”的危险符号和止损玩法。
* 当前阶段只把升级工具作为中期目标接入，不引入自动刮卡。

## 6.12 熟练度 500：三连胜出刮刮卡

当玩家刷满三连胜出段 `500` 熟练度后，左上角熟练度进入下一目标并展示 `0/1000`，同时触发新的电话响铃。玩家点击电话后，按以下顺序播放：

1. “哇哦，我这正好有新的刮刮卡！”
2. “这张的奖励更多......”

电话结束后，桌面中央播放“已解锁 三连胜出”发光提示，左侧刮刮卡目录新增第二张卡“三连胜出”。该卡不替代“成双入对”，玩家可以在两张卡之间按金币和风险偏好选择。

“三连胜出”本地 MVP 规则：

中奖规则：刮出 3 个相同图案才给钱

票面展示奖池参考卡面右侧说明：

真实奖池仍然必须单独配置，不能直接使用票面展示概率。阶段二后续先采用以下本地真实概率：

| 类型 | 真实概率 | 展示概率 | 结算金额 |

交互规则：

* 点击左侧“三连胜出”后，如果金币足够，扣除 `$100` 并预生成本张卡结果；金币足够时允许连续购买多张。
* 卡片像盘子和“成双入对”一样飞入桌面，可自由拖动、打开、关闭、结算或丢弃。
* 打开放大视图后展示蓝色横向卡面：左侧是 5 个圆形结果位，右侧是独立刮层视觉块，卡面外右侧展示中奖概率说明；真实可刮遮罩只覆盖左侧 5 个圆形结果位区域。
* 刮层仍然必须拖动擦除，禁止点击开奖。
* 结算时只有出现 3 个相同图标才按对应符号给钱，否则结算 `$0`。
* 该卡拥有独立等级和等级进度，默认 `等级 0`；阶段二后续只要求接入通用等级结构，具体倍率可在实施前继续按配置补齐。

## 6.13 阶段 2.6：金币变动与刮卡揭露特效

阶段 2.6 是阶段二后、阶段三风险卡前的体验补充，不新增卡片规则、概率、奖励或惩罚，只强化玩家对“金币变化”和“图案被刮出”的即时反馈。

整体原则：

* 特效必须短促、清晰，不阻挡继续操作。
* 特效只能表达已经发生的结算、购买、扣款或揭露结果，不能提前暗示未揭露的中奖信息。
* 特效实现优先使用 CSS animation 和少量事件状态，不引入持续 canvas 粒子循环。
* 同一时间的浮字和粒子数量必须有限制，避免重新引入刮盘子或刮卡卡顿。
* 必须兼容 `prefers-reduced-motion`：减少动画用户只保留颜色闪烁、金额浮层或静态强调。

金币增加特效：

* 触发场景：洗盘子领取奖励、刮刮卡中奖结算、贷款到账。
* 金币栏短暂放大并闪金光，表示总金额增加。
* 金币栏数字应从旧值短暂滚动到新值，避免数值瞬跳。
* 触发位置或金币栏附近显示 `+$X` 浮字，向金币栏方向或向上漂浮后淡出。
* 大额收入可以追加少量金币粒子，但持续时间应控制在 `500ms~700ms`，且同屏粒子上限建议不超过 `12` 个。
* 贷款到账也属于金币增加，但视觉语义应弱于中奖，避免贷款被误解为奖励。

金币减少特效：

* 触发场景：生成脏盘子、购买刮刮卡、购买升级工具、偿还贷款、碎盘扣钱。
* 金币栏短暂下沉或轻抖，并使用红色 / 橙红色脉冲表示扣款。
* 金币栏附近显示 `-$X` 浮字，向下漂浮后淡出。
* 购买类扣款应保持轻量，表示正常消费；碎盘和还贷可以更重，表示惩罚或债务压力。
* 已有贷款偿还盖章动画可以继续保留，金币扣款浮字作为补充反馈。

刮刮卡图案揭露闪烁：

* 每个结果格在被刮开到可识别程度后触发一次揭露闪烁。
* 单个结果格必须刮开到至少 `95%` 才能触发揭露闪烁，刚蹭到角不算。
* 每个结果格每张卡只触发一次，继续刮同一格不能重复刷闪烁。
* 普通图案使用 `pop + glow`：图案轻微放大、边框扫光、短暂发亮，建议时长约 `360ms`。
* `未中` 或空白图案使用更弱的灰白闪烁，避免把未中奖反馈做得像奖励。
* 刮完整张卡后，如果存在中奖组合，中奖图案再同步闪烁 `2` 次；非中奖图案弱化。
* “三连胜出”中，三个相同中奖图案应同步闪烁，另外两个非中奖位保持弱化。
* 闪烁判定应基于结果格区域的揭露进度或中心点揭露状态，而不是整张卡总进度，避免最后一次性全部闪。

建议实现结构：

* `EffectLayer`：统一承接金币增加、金币减少、粒子和浮字事件。
* `goldEffectEvents`：记录金币变动事件，包含金额、方向、来源和触发位置。
* `revealedSymbolSlots`：记录当前刮刮卡已经触发过揭露闪烁的格子。
* `settlementHighlight`：刮完后标记中奖组合，用于最终同步闪烁。

验收标准：

* 金币增加时，玩家能立刻看到金色反馈和 `+$X` 浮字。
* 金币减少时，玩家能立刻看到扣款反馈和 `-$X` 浮字，且不会误认为奖励。
* 连续买卡、连续洗盘子、快速刮盘子和快速刮卡时不出现明显卡顿。
* 每个刮刮卡图案第一次露出时会闪烁一次。
* 已闪烁过的格子继续刮不会重复闪烁。
* 中奖组合在刮完后有额外强调。
* “三连胜出”的 5 个结果位揭露闪烁与真实遮罩覆盖区域对齐。

# 七、风险卡与 Peek 机制

## 7.1 定位

《刮个爽》真正拉开层次的，不只是“中奖了没”，而是：

```txt
我继续刮，可能爆奖
我继续刮，也可能把惩罚完全揭开
```

因此风险卡不是阶段二主菜，而是阶段三及之后的重要扩展。

## 7.2 惩罚符号与信息结构

高置信骨架下，风险卡必须同时满足“可爆奖”和“可判断”：

* 惩罚与具体危险符号绑定。
* 危险符号必须真实存在于可刮开的票面区域中，不能在结算瞬间凭空生成。
* 玩家必须能通过局部揭露获得部分信息，而不是只能等到最后翻完才知道生死。
* 危险结果默认在完全揭露危险符号后才触发；阶段三先不用“看一眼就立刻爆炸”的重惩罚。

第一张风险卡建议采用以下信息密度：

* `4~6` 个刮开区。
* `1~2` 个潜在危险位。
* 玩家在 `1~3` 次局部 Peek 后，通常能形成“继续 / 放弃”的初步判断。
* 不要求玩家靠像素狙击或记忆游戏才能做对选择。

阶段三第一张风险卡采用以下本地 MVP 规则：

```txt
名称：险中求财
价格：$150
结果区：6 格
危险位：危险结果票面预埋 1 个真实危险符号
危险触发：危险格刮开到 95% 后触发
弃卡成本：本张卡价格的 30%，并保留至少 $1 的前中期保护
轻惩罚：危险触发后本张收益归零，不额外扣金
自动化：默认手动处理，不进入基础自动化目标
```

阶段三本地真实奖池：

| 类型 | 真实概率 | 展示概率 | 结算金额 |
| --- | --- | --- | --- |
| 危险符号 | 32% | 不展示 | `$0` |
| 铜币保底 | 38% | 45% | `$180` |
| 钱袋翻倍 | 22% | 35% | `$260` |
| 纸钞爆发 | 8% | 20% | `$420` |

说明：

* 只有抽到危险结果时，票面才会预埋危险符号；安全结果票面不凭空生成危险。
* 危险符号真实存在于可刮区中，结算时不能临时随机惩罚。
* 安全结果的票面主奖符号必须和本张卡结算档位一一对应：铜币给 `$180`，钱袋给 `$260`，纸钞给 `$420`；其余安全格使用未中占位，不展示其他奖励档位符号。
* 安全结果不按相同图案数量叠加兑奖，避免和“三连胜出”的数量规则混淆。
* 玩家可以通过只刮单格边角或局部区域进行 Peek；如果判断有风险，可以在完全揭露前弃卡。
* 弃卡只扣处理费，已经露出的部分结果不兑奖，也不会推进卡片等级进度。

阶段三允许的轻惩罚方向：

* 本张卡收益归零。
* 扣除少量金币。
* 额外支付一次弃卡 / 处理成本。

前中期不允许用重惩罚直接打断主循环。

## 7.3 Peek、止损与弃卡

中后期卡片要为以下能力提供明确规则：

* 玩家先刮开边角、单格或局部区域。
* 通过局部信息判断是否存在危险符号或低价值信号。
* 在完全揭露前决定继续、止损或放弃当前卡。

为了让“放弃”成为真实决策，而不是白嫖侦查，弃卡必须带成本。阶段三建议：

* 弃卡时损失本张卡价格的 `20%~40%`，或支付一笔固定处理费。
* 已经看见的部分结果不会在弃卡时额外兑奖。
* 玩家不能把终局票、最终挑战票或明确标记为“手动决策”的卡交给默认自动化处理。

这类“半揭露决策”是《刮个爽》区别于纯开奖小游戏的重要特征。

## 7.4 风险层级

为了让中后期设计更接近《刮个爽》的味道，后续卡片至少要分成四种风险职责：

| 票型 | 主要职责 | 是否适合自动化 |
| --- | --- | --- |
| 安全票 | 稳定回本、喂养中期经济 | 是 |
| 风险票 | 通过 Peek 判断继续或止损 | 否，默认手动 |
| 高风险票 | 冲爆奖、吃重惩罚 | 否，默认手动 |
| 终局票 / Final Chance 票 | 结束本轮、换取 Prestige 资源 | 否，强制手动 |

## 7.5 当前取舍

* 阶段二先不强制实现弃卡。
* 阶段三开始正式引入“轻惩罚 + 局部判断 + 弃卡成本”。
* 文档和数据结构从阶段二起就要为惩罚符号、弃卡、票型标签和风险层级预留扩展位。

# 八、卡册与卡片进阶系统

## 8.1 定位

后续卡片的进阶方向，不应该只是“更贵、数值更大”，而必须像《刮个爽》那样形成一组卡册 / 目录：

* 同一目录中同时存在稳定票、手动风险票、爆奖票和终局票。
* 玩家不是“只买当前最贵的票”，而是在不同票之间分配金币、时间和注意力。
* 目录推进既依赖挣钱，也依赖玩家是否敢处理本轮最后的关键票。

## 8.2 高置信进阶梯度

当前按高置信骨架，把卡片梯度拆成四类：

| 卡类 | 定位 | 体验目标 |
| --- | --- | --- |
| 安全卡 | 教学 / 启动循环 | 学会买卡、刮卡、领奖 |
| 风险卡 | 中期主力 | 小心惩罚，开始判断要不要继续刮 |
| 高风险卡 | 后期爆点 | 高奖池、高惩罚、明显翻盘感 |
| 终局卡 | 本轮高潮 | 为 Prestige 前的最后冲刺服务 |

## 8.3 卡册推进原则

为了对齐《刮个爽》的中后期结构，游戏后续应按“卡册 / 目录”推进，而不是无限线性堆票：

* 每个卡册至少包含 `3~5` 张核心票和 `1` 张终局票。
* 当前卡册内必须同时存在：
  * `1` 张自动化友好的稳定票；
  * `1` 张需要手动 Peek 的风险票；
  * `1` 张明显更贵、用于冲波动收益的高赔率票；
  * `1` 张带强止盈 / 继续决策的高风险票；
  * `1` 张不能自动化、会结束本轮的终局票。
* 下一卡册的开放不只看金币，还应要求玩家到达当前卡册的终局阶段。

## 8.4 解锁原则

卡类与卡册解锁应由以下维度共同推动，而不是只看当前金币：

* 当前卡册内的累计刮卡次数。
* 历史总收益。
* 已拥有的关键升级。
* 是否已经结算过上一册的终局票。
* 是否已进入更高 Prestige 轮次。

## 8.5 设计要求

* 新卡必须改变玩家行为，而不是只提高赔率或倍率。
* 每一册都必须同时服务“稳定刷钱”“手动判断”“冲爆奖”“结束本轮”四种职责。
* 至少一部分卡片必须让玩家感到“贵，但值得搏一把”。
* 终局卡必须服务于“本轮高潮”和“进入下一轮成长”，不能只是另一张更贵的刷钱票。

## 8.6 阶段四第一本卡册结构

阶段四先把当前已实现卡片归入第一本卡册，不新增第二本具体玩法。

第一本卡册：

```txt
名称：街角好运
定位：承接当前前中期玩法，从稳定回本过渡到手动风险判断。
```

卡册职责位：

| 职责 | 卡片 | 当前状态 |
| --- | --- | --- |
| 稳定票 | 成双入对 | 已实现 |
| 风险票 | 险中求财 | 已实现 |
| 高赔率票 | 三连胜出 | 已实现 |
| 高风险票 | 步步加码 | 阶段 6A 接入 |
| 终局票 | 终局票 | 仅保留锁定位，后续阶段再定义规则 |

阶段四第一步只改变目录组织方式：

* 左侧刮刮卡入口只展示已经解锁、已经可玩的票。
* 每张已实现卡显示自己的职责标签。
* 终局票规则开放前不在玩家 UI 中展示。
* 第二本卡册只保留为内部结构，不在玩家 UI 中展示“设计中 / 锁定”入口。
* 现有卡片购买、刮开、结算和等级进度规则保持不变。

阶段四需要补齐“赚到钱后追什么”的目标链展示，但不提前实现阶段五自动化：

* 自动刮刮机通过熟练度里程碑获得购买资格，玩家达到该段后通过电话或解锁效果看到新增入口。
* 自动刮刮机入口只在已获得购买资格或已购买后展示，不提前显示锁定位、下一目标或后续升级线。
* 金币只决定是否能支付 `$1000`，不再额外要求本册结算次数。
* 左上角始终保持纯数字分段熟练度，例如 `125/1000`，不切换成“自动机 N/10”。

# 九、升级系统

## 9.1 定位

升级系统负责把玩家从“手动刮单张卡”推进到“更快、更稳、更贪、更自动化”。

## 9.2 高置信升级方向

当前优先采用以下四类升级：

| 升级类 | 作用 | 默认影响域 |
| --- | --- | --- |
| 幸运 | 提高中正收益和稀有收益的能力 | 真实概率权重 |
| 刮擦效率 | 提升刮开面积、力度或完成速度 | 操作手感 |
| 奖励放大 | 提升正收益的回报能力 | 结算倍率 |
| 自动化控制 | 解锁自动刮卡、队列和过滤规则 | 自动化范围与稳定性 |

## 9.3 设计原则

* 每个升级项都必须明确标注自己影响的是“手感 / 真实概率 / 结算倍率 / 自动化行为”的哪一类，避免隐藏影响。
* 前期优先改善手感与稳定性。
* 中期开始放大奖励与幸运。
* 自动化必须作为中期转折点，而不是开局就给。
* 升级应存在上限或递减收益，避免单一项目成为永远最优解。
* 不允许出现升级后完全跳过刮开过程的情况。

## 9.4 当前推荐的中期升级树

为了更贴近《刮个爽》的中期体验，阶段 2.5 之后的运行内升级优先级建议为：

1. `刮除范围 / 刮擦强度`：先改善手感和节奏。
2. `刮卡运气`：让玩家感觉“好票开始多了”。
3. `奖励放大`：把安全票和中风险票变成真正的滚雪球工具。
4. `自动化控制`：先解锁基础队列，再解锁风险过滤和更快处理速度。

## 9.5 当前已实装升级规则

阶段 4 当前先实装两个可购买升级项：

| 升级项 | 初始价格 | 价格成长 | 当前效果 |
| --- | --- | --- | --- |
| 刮除范围 | `$100` | 按当前等级倍率递增 | 提升刮刮卡刮层与洗盘子污渍层的手动刮除半径。 |
| 刮卡运气 | `$200` | 按当前等级倍率递增 | 降低安全票 / 高赔率票的未中奖真实权重，并把减少的概率分配给中奖档位。 |

价格成长统一由工具配置的 `priceMultiplierByLevel` 控制，当前默认倍率为 `1 / 1.5 / 2.25 / 3.4 / 5.1 / 7.6 / 11.4 / 17 / 25.5 / 38`。购买某一级时使用当前等级对应价格，例如“刮除范围”从等级 0 升级需要 `$100`，等级 1 升级需要 `$150`，等级 2 升级需要 `$225`；“刮卡运气”从等级 0 升级需要 `$200`，等级 1 升级需要 `$300`。

幸运升级必须保持温和：每级只从未中奖概率中转移 `3%`，且未中奖总概率最低保留 `45%`，避免中期经济直接失控。风险卡当前不吃幸运加成，危险符号、放弃成本和轻惩罚仍由风险卡自己的规则控制。

# 十、自动化系统

## 10.1 定位

自动化是中期后的明确转折点，作用是接管重复劳动，而不是替代所有决策。

## 10.2 核心能力

自动化至少需要覆盖这些方向中的一部分：

* 自动购买低风险卡。
* 自动刮开低风险卡。
* 自动结算结果。
* 自动处理低价值重复操作。
* 按票型和风险标签过滤可处理对象。

## 10.3 自动化的最小系统维度

后续自动化不能只是一句“自动刮卡”，至少要有这些可调维度：

* `队列容量`：能同时排多少张卡。
* `处理速度`：每张卡需要多少时间。
* `可处理票池`：默认只处理安全票和低风险票。
* `风险过滤`：是否跳过带危险符号、终局标签或手动标记的票。
* `人工接管`：玩家能否把某张票强制改回手动处理。

## 10.4 强制规则

* 不允许瞬间批量开奖。
* 自动化必须保留可见的时间消耗。
* 高风险卡默认不自动处理，或需要玩家显式开启。
* 终局票 / Final Chance 票必须强制手动处理。
* 自动化上线后，玩家仍应保留对高风险决策的控制权。
* 自动化的价值应体现在“减轻重复劳动”，而不是“替玩家完成最刺激的判断”。

## 10.5 自动刮刮机定位

自动刮刮机是阶段五的中期经济发动机，不是一个“自动开奖”按钮。它的职责是接管稳定票的重复劳动，让玩家把注意力转向风险票、升级路线、终局票和后续卡册目标。

自动刮刮机必须形成以下循环：

```txt
手动刮卡积累资金
-> 购买自动刮刮机
-> 配置可自动处理的票种
-> 机器按队列和时间处理稳定票
-> 玩家用稳定收入挑战风险票、购买升级或追终局票
```

它解决的是“重复买安全票、重复刮、重复结算”的疲劳，不解决“风险判断”和“终局挑战”。

## 10.6 解锁与可见目标

自动刮刮机在阶段四先通过熟练度里程碑开放购买资格，阶段五再接入真实运行逻辑。

推荐基础条件：

| 条件 | 推荐值 | 目的 |
| --- | --- | --- |
| 购买价格 | `$1000` | 成为玩家第一个中期大额目标。 |
| 熟练度里程碑 | 自动刮刮机段刷满 `250` | 与前期垃圾桶、刮刮卡、升级工具、三连胜出的解锁方式统一。 |

阶段四展示规则：

* 自动刮刮机未获得购买资格前不在玩家 UI 中展示。
* 达到熟练度里程碑后，电话或解锁特效提示新增工具，辅助道具页出现购买入口。
* 入口展示价格、当前差额和一句职责说明。
* 示例文案：`自动处理稳定票，风险票仍需手动。`
* 如果金币不足，展示 `还差 $X`。
* 电话提示不阻塞熟练度继续累计；熟练度满段后立刻进入下一段数值。
* 阶段四不允许自动购买、自动刮开或自动结算，只开放购买资格和入口。

## 10.7 运行状态与队列

阶段五开始，自动刮刮机采用队列式处理，而不是瞬间批量结算。

基础运行状态：

| 状态 | 含义 |
| --- | --- |
| `locked` | 未解锁，只显示目标。 |
| `idle` | 已解锁但没有待处理卡。 |
| `buying` | 正在尝试自动补票。 |
| `processing` | 正在处理队列中的一张票。 |
| `paused` | 玩家手动暂停。 |
| `blocked` | 因金币不足、队列满、没有可处理票或保留金币限制而暂停。 |

基础参数建议：

| 参数 | 初始值 | 说明 |
| --- | --- | --- |
| 队列容量 | `2` 张 | 控制机器不能无限吞票。 |
| 处理时间 | `8s / 张` | 保留可见时间消耗。 |
| 默认票种 | 成双入对 | 只处理稳定票。 |
| 自动购买 | 默认开启 | 金币足够且队列未满时补票。 |
| 自动结算 | 默认开启 | 处理完成后结算金币、卡片等级和本册进度。 |

机器购买卡片时必须立即扣费并预生成本张卡结果，后续处理只负责按时间揭露和结算，不允许在处理结束时临时抽奖。

## 10.8 自动化票种权限

自动化权限按票型职责控制，不能只按价格控制。

| 卡片 | 自动化权限 | 原因 |
| --- | --- | --- |
| 成双入对 | 默认可自动 | 稳定票，规则简单，适合接管重复劳动。 |
| 三连胜出 | 需要机器力量升级 | 高赔率票但无中途止损判断，适合中后期自动化扩展。 |
| 险中求财 | 默认不可自动 | 依赖 Peek、危险位和止损判断。 |
| 终局票 | 永远不可自动 | 本册高潮，必须手动决策。 |

后续若允许机器处理更高阶票，必须通过“机器力量”或“风险过滤”明确解锁，不能默认全部自动。

## 10.9 机器升级线

自动刮刮机至少包含以下升级方向：

| 升级项 | 初始价格 | 作用 |
| --- | --- | --- |
| 自动刮刮机 | `$1000` | 解锁机器本体，允许自动处理稳定票。 |
| 刮卡机容量 | `$2500` | 提高队列上限，例如 `2 -> 3 -> 5 -> 8`。 |
| 刮卡机力量 | `$2000` | 提高可处理票池，后续可解锁“三连胜出”等非风险高赔率票。 |
| 刮卡机速度 | `$5000` | 缩短单张处理时间，例如 `8s -> 6.5s -> 5s -> 4s`。 |

升级价格可以复用当前工具的 `priceMultiplierByLevel` 曲线，但必须从配置读取，不允许散写在组件里。

## 10.10 过滤与资金保护

自动刮刮机需要最小过滤规则，避免机器把玩家所有金币吞掉，导致玩家失去手动选择空间。

第一版推荐配置：

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| 自动购买 | 开 | 允许机器自动补稳定票。 |
| 允许票种 | 成双入对 | 第一版只开放稳定票。 |
| 最低保留金币 | `$0`，后续可升级或选择 `$100 / $500` | 保留玩家手动决策资金。 |
| 风险票过滤 | 开 | 跳过风险票、终局票和手动决策票。 |

机器在以下情况必须进入 `blocked` 或 `paused`：

* 当前金币不足以购买允许票种。
* 队列已满。
* 当前配置没有任何可处理票种。
* 购买后会低于最低保留金币。
* 当前票种被标记为风险票、终局票或强制手动票。

## 10.11 交互与反馈

自动刮刮机必须有桌面存在感，不能只是后台数字。

推荐表现：

* 桌面一角展示机器本体。
* 队列中展示小卡片槽位。
* 正在处理的卡显示刮开进度或机器工作动画。
* 完成时展示金币浮字、卡片弹出或结算反馈。
* 面板展示状态、队列、当前票、处理进度和启动 / 暂停按钮。

自动化过程中不能提前展示本张票的真实 payout 或完整结果。玩家能看到机器在处理，但结果只能在处理完成后结算。

# 十一、高风险卡与终局卡

## 11.1 高风险卡

高风险卡是后期的刺激来源，应同时满足：

* 成本明显更高。
* 爆奖回报明显更高。
* 惩罚更强。
* 更依赖风险判断、Peek 或弃卡决策。
* 默认不适合基础自动化。

后期至少需要两类不同语法的高风险票：

* `局部排雷型`：玩家通过局部揭露判断是否存在致命危险位。
* `继续加码型`：玩家可以在已经看到部分收益后，决定是否继续刮更多区域追更大奖。

## 11.1.1 阶段 6A 架构接入原则

阶段 6A 开始，新增票种不能继续把类型判断散写到各处。实现前必须先收口一层“卡片规则 / 能力描述”，让后续高风险票、终局票和 Prestige 能兼容同一条购买、刮开、结算和存档链路。

新增票种接入时至少要集中定义：

* `cardType`：卡片类型标识，例如 `push-luck`。
* `role`：卡片职责，当前可用 `stable`、`risk`、`high-odds`、`high-risk`、`finale`。
* `progressKey`：该票种的等级 / 结算进度存档 key。
* `automationPolicy`：自动化权限，当前可用 `allowed`、`requires-machine-power`、`manual-only`、`never`。
* `decisionMode`：是否存在手动决策，当前可用 `none`、`discard`、`cash-out-or-press`、`finale-choice`。
* `settlementMode`：结算方式，当前可用 `fixed-payout`、`penalty-aware`、`staged-cash-out`、`prestige-trigger`。
* `scratchLayout`：刮层布局和每个可揭露区的边界，避免在 Canvas 组件里为每张票重复写硬编码分支。
* `display`：目录名、票面标题、桌面缩写、规则文案和结算文案。

架构约束：

* 新票默认不能进入自动刮刮机，除非 `automationPolicy` 明确为 `allowed`。
* 高风险票与终局票不能被基础自动刮刮机处理；终局票永远强制手动。
* 刮卡运气默认不影响高风险票和终局票，除非设计文档单独声明。
* 存档新增票种进度时必须做默认值合并，旧存档缺字段时自动补齐，不清空桌面、金币、贷款、升级或自动机队列。
* 单张卡购买时必须预生成完整路径；后续“继续 / 收手 / 结算”只消费这条预生成路径，不能在玩家继续时临时抽新结果。
* 高风险票如果需要中途决策，不应复用 `claimable` 表达所有状态；实现时需要显式区分“可收手”“已爆雷”“可最终结算”等运行态，避免和普通刮完结算混淆。
* UI 可以继续在单页内实现，但新增票种的票面、规则说明、奖池行和结算按钮应优先复用通用卡片渲染能力，避免在主组件内复制整块 JSX。

## 11.1.2 阶段 6A 第一张高风险卡：步步加码

阶段 6A 先实现第一张 `继续加码型` 高风险票，不同时实现终局票。该票用于承接阶段 5 自动化后的稳定收益，让玩家把自动机产出的金币拿去做明显更贪、更危险的手动挑战。

基础配置：

| 配置 | 数值 | 说明 |
| --- | --- | --- |
| 卡名 | 步步加码 | 玩家可见高风险票名称。 |
| 类型标识 | `push-luck` | 代码内卡片类型。 |
| 职责 | 高风险票 | 当前卡册的强波动挑战位。 |
| 解锁条件 | 自动刮刮机已购买，且 `10000` 熟练度段达成 | 确保玩家已经经历自动机稳定期和高风险预热，不在前中期过早看到。 |
| 价格 | `$800` | 明显高于“险中求财”和“三连胜出”，但低于自动机本体，便于反复挑战。 |
| 自动化 | `manual-only` | 基础自动机不可处理，后续机器力量也不直接开放。 |
| 幸运加成 | 不生效 | 避免幸运升级把高风险决策变成稳定收益。 |
| 层数 | `4` 层 | 每层刮开后都可能出现“收手 / 继续”的决策。 |

视觉识别：

* “步步加码”不能只复用普通刮刮卡的山景票面换色；桌面小卡和放大票面都应呈现“逐层上升 / 高风险阶梯”的视觉。
* 桌面小卡使用更窄、更竖向的高风险票轮廓，内含 4 个阶梯节点和爆雷标记，让玩家在桌面上一眼区分它不是普通开奖卡。
* 放大票面顶部使用四层推进路径替代普通风景图；结果区保留 4 个真实可刮层位，并与 Canvas 遮罩坐标保持一致。

票面流程：

```txt
购买时预生成 4 层路径
→ 玩家刮开第 1 层
→ 如果第 1 层安全，显示当前可收手金额
→ 玩家选择“见好就收”或“继续加码”
→ 继续后解锁下一层刮区
→ 任一后续层爆雷，则本票当前收益归零并追加本层手续费
→ 玩家收手或第 4 层安全后，进入结算
```

每层安全金额：

| 层级 | 安全后可收手金额 | 体验定位 |
| --- | --- | --- |
| 第 1 层 | `$260` | 给玩家第一次止损机会，但低于票价。 |
| 第 2 层 | `$760` | 接近回本，让玩家纠结是否继续。 |
| 第 3 层 | `$1800` | 明显盈利，开始进入高风险甜点。 |
| 第 4 层 | `$5200` | 高风险票的阶段性爆点。 |

爆雷规则：

| 爆雷层 | 本票 payout | 追加手续费 | 说明 |
| --- | --- | --- | --- |
| 第 2 层 | `$0` | `$120` | 玩家已经错过第 1 层止损机会。 |
| 第 3 层 | `$0` | `$240` | 惩罚更明显，但不直接摧毁前中期经济。 |
| 第 4 层 | `$0` | `$400` | 最高层失败成本最高，强化终局前贪念。 |

手续费扣除必须遵守破产保护：扣除后至少保留 `$1`，如果金币不足则只扣到可承受范围。爆雷不会追加贷款，也不会清空自动机队列、升级或卡片进度。

预生成路径奖池：

| 首个爆雷层 | 概率 | 说明 |
| --- | --- | --- |
| 第 2 层 | `45%` | 最常见失败，教玩家第 1 层可止损。 |
| 第 3 层 | `30%` | 玩家已经接近回本后继续加码才会遇到。 |
| 第 4 层 | `20%` | 玩家已经明显盈利后继续贪，临门一脚失败。 |
| 无爆雷 | `5%` | 四层全安全，高风险票核心爆点。 |

注意：路径概率用于购买时预生成“首个爆雷层”。如果玩家在爆雷层之前见好就收，本票按当前安全层金额结算；如果玩家继续刮到首个爆雷层，本票立刻进入爆雷结算。真实路径不直接展示给玩家，票面只展示“越往后奖励越高，爆雷手续费越高”的风险说明。

结算规则：

* 玩家点击“见好就收”时，按当前安全层金额结算，并移除桌面卡片。
* 玩家刮到爆雷层时，不能再继续加码，只能结算本票 `$0` 和本层手续费。
* 玩家安全刮完第 4 层时，按 `$5200` 结算。
* 结算后推进“步步加码”自己的卡片等级进度；丢弃或爆雷前弃卡不推进。
* “步步加码”的等级结构先接入通用进度，阶段 6A 不启用 payout 倍率成长，避免高风险经济过早膨胀。
* 自动刮刮机面板可以展示“步步加码仍需手动”，但不能提供加入队列的开关。

## 11.2 终局票 / Final Chance 票

终局票不是稳定刷钱票，而是每轮接近 Prestige 前的关键选择，职责更接近《刮个爽》里的 Final Chance：

* 它是当前卡册的明确终点目标。
* 处理终局票后，本轮应进入结算 / Prestige 判定，而不是继续无限刷。
* 它必须强制手动处理，不能交给默认自动化。

终局票必须承担以下职责：

* 给本轮提供最后一次显著高潮。
* 让玩家决定是继续贪，还是接受本轮结果进入下一轮成长。
* 作为进入 Prestige 的情绪和经济踏板。

## 11.3 终局票奖励与失败规则

为了让终局票真正服务循环闭环，建议：

* 处理终局票后，无论成败，都至少给本轮 Prestige 资源一个基础结算。
* 终局票成功时，额外给更高的 Prestige 资源、目录推进奖励或稀有解锁。
* 终局票失败时，不应让玩家继续用同一套运行内资源无限补救；失败本身就是进入下一轮的信号。

# 十二、Prestige 永久成长

## 12.1 定位

Prestige 是完整骨架不可缺少的一环。

为了更接近《刮个爽》的后期结构，Prestige 不应只是“觉得赚慢了就手动重置”，而应主要由终局票 / 关键挑战票结算触发。

当本轮到达终局阶段后，玩家应能重置以下内容：

* 当前金币
* 当前已解锁卡类
* 本轮升级
* 本轮自动化进度
* 当前自动化队列
* 当前贷款与本轮临时负面状态

换取永久点数或永久成长。

## 12.2 Prestige 资源来源

Prestige 资源建议命名为 `荣耀点`、`传奇点` 或等价的永久成长货币。

来源优先级建议为：

* `主要来源`：终局票 / Final Chance 票结算。
* `次要来源`：本轮达成的超级大奖、超高连胜或目录深度奖励。
* `保底规则`：第一次 Prestige 必须保证至少拿到 `1` 点永久资源，避免首次重置像白玩。

## 12.3 永久成长方向

永久成长优先覆盖以下高置信方向：

| 永久成长 | 作用 |
| --- | --- |
| 起始资金 | 更快回到第一张卡 |
| 永久幸运 | 全局提高优质结果出现率 |
| 永久收益放大 | 提升轮回后的滚雪球速度 |
| 永久刮擦基础 | 提高开局操作效率 |
| 自动化提早接入 | 更快回到中期节奏 |
| 卡册起步优势 | 更快回到上一轮曾到达的中期阶段 |

## 12.4 节奏要求

Prestige 必须让玩家感受到：

* 这轮结束得值
* 下一轮明显更快
* 虽然重置了，但没有白玩
* 第一次 Prestige 后，玩家应明显缩短回到中期卡册的时间

# 十三、前 10 分钟体验目标

## 0~1 分钟

* 玩家学会洗盘子。
* 玩家理解洗盘子只是门票。

## 1~3 分钟

* 玩家买到并完成第一张安全卡。
* 玩家建立“买卡 -> 刮卡 -> 领奖 / 亏损”的基础循环。

## 3~5 分钟

* 玩家完成多张低级卡。
* 玩家开始感到“再来一张可能翻盘”。

## 5~10 分钟

* 玩家看到升级目标和下一类卡片目标。
* 风险卡或自动化目标开始出现在界面上。
* 玩家明确意识到后续主菜是刮卡，不是洗盘子。

---

# 十四、防崩机制

必须避免：

* 洗盘子重新变回长期主玩法
* 玩家连续亏损后完全无法继续
* 惩罚过重导致前中期直接放弃
* 自动化绕过核心揭露过程
* 高级卡只有更大数字，没有新决策

## 14.1 破产保护

如果玩家短期亏空，必须始终能通过洗盘子重新获得门票资金。

阶段二新增破产兜底：如果玩家金币为 `$0`、桌面没有脏盘子且没有未结算刮刮卡，电话会响并提供一笔 `$5` 贷款。该贷款利率展示为 `6000%`，偿还金额为 `$300`。如果玩家背着旧贷款再次破产，会继续追加下一笔贷款，贷款列表用于集中查看和偿还。贷款只负责恢复继续游戏的最低启动金，不直接提供额外收益。

## 14.2 风险保护

风险卡与惩罚卡必须遵守：

* 先轻后重
* 先教会判断，再放大奖惩
* 不允许前几分钟出现一次错误就直接崩盘

## 14.3 自动化保护

自动化不应替代全部手动内容，而应把玩家从重复劳动里解放出来，把注意力推向高风险决策。

---

# 十五、MVP 最低标准

当前高置信骨架下，MVP 必须至少具备：

* 洗盘子赚启动金
* 解锁并购买第一张低级安全卡
* 购买时预生成结果
* 拖动擦除遮罩
* 渐进揭露结果
* 成对中奖与未成对 `$0`
* 为阶段三预留惩罚符号和弃卡扩展位

如果缺少“买卡 -> 拖动刮开 -> 结算 -> 继续买卡”的循环，这个项目就还没有真正站到《刮个爽》的主玩法上。

---

# 十六、开发优先级

1. 洗盘子启动金。
2. 第一张低级安全卡与拖动刮开。
3. 风险卡、轻惩罚与 Peek。
4. 更多卡类与升级系统。
5. 自动化。
6. 高风险卡与终局卡。
7. Prestige。

---

👉 核心认知：

**洗盘子是门票，安全卡是入场教学，风险卡才开始长出《刮个爽》的味道，升级、自动化和 Prestige 负责把这个循环拉长。**
