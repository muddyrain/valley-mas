# 刮出传说（Scratch Legend）- 核心玩法设计（修正版）

---

# 一、核心目标（必须牢记）

本游戏目标不是“公平”，而是：

👉 让玩家不断产生
👉 **「再来一把就能翻盘」的冲动**

---

# 二、核心循环（精确版）

一个完整循环：

1. 玩家进行启动金行为（刮盘子前置积累）

2. 获得约 80~150 金币

3. 进行 1 次刮盘子

4. 结果：

   * 亏（多数情况）
   * 小赚（常见）
   * 爆赚（极少）

5. 玩家心理：
   👉 “刚刚差一点，再来一把”

---

## 2.1 节奏校准（关键）

当前玩法必须避免：

❌ 高速点击器
❌ 手速决定收益
❌ 自动跳阶段

正确体验应是：

* 玩家在“准备刮一次盘子”
* 刮盘子才是核心爽点
* 节奏由系统控制，而不是玩家手速

---

# 三、第一阶段：启动金系统（弱化）

### 3.1 定位（修正）

👉 第一阶段只是：

👉 **为“刮盘子”提供启动资金**

不是核心玩法

---

### 3.2 行为规则

* 一次点击 ≠ 一次收益
* 必须有动作时间锁（400~800ms）
* 收益在动作结束结算

---

### 3.3 节奏目标

👉 30~60 秒 → 1 次刮盘子

---

# 四、核心玩法：刮盘子系统（重点重构）

---

## 4.1 核心定义

👉 刮盘子 ≠ 点击开奖
👉 刮盘子 = **擦除遮罩 + 渐进揭露 + 自动开奖**

---

## 4.2 结构设计（双层）

盘子必须由两层组成：

### 1️⃣ 底层（真实内容）

* 食物图片
* 奖励结果（提前生成）

---

### 2️⃣ 上层（刮层）

* 灰白遮罩
* 可被擦除

---

## 4.3 交互方式（必须）

玩家操作：

* 鼠标按下 → 开始刮
* 鼠标移动 → 擦除
* 鼠标抬起 → 停止

👉 禁止点击触发

---

## 4.4 技术实现（推荐）

使用 Canvas：

```ts
ctx.globalCompositeOperation = 'destination-out'
```

擦除方式：

* 圆形 brush（20~40px）
* 连续擦除
* 带轻微随机感

---

## 4.5 擦除进度判定（核心）

```ts
if (erasedPercent >= 40%) {
  triggerResult()
}
```

实现方式：

* 像素采样
* 或网格采样（推荐）

---

## 4.6 视觉过程（必须一致）

刮盘子必须有完整过程：

1. 完全覆盖
2. 擦出局部内容
3. 内容逐渐清晰
4. 接近结果
5. 自动开奖

---

👉 玩家必须看到：

👉 “越来越接近结果”

---

## 4.7 开奖逻辑

注意：

👉 奖励在开始就确定
👉 不是刮的时候算

开奖时：

* 显示金额（如 $3）
* 播放动画（scale / glow）
* 锁定状态

---

## 4.8 奖励池（基础）

| 类型 | 概率  | 奖励   |
| -- | --- | ---- |
| 普通 | 60% | 50   |
| 小奖 | 25% | 120  |
| 大奖 | 10% | 500  |
| 传说 | 5%  | 2000 |

---

## 4.9 收益期望

👉 单次期望 ≈ 85 金币

👉 玩家长期略亏
👉 但被爆点驱动

---

## 4.10 动态概率（必须）

### 连败补偿

```ts
if (loseStreak >= 3) {
  bigWinRate += 0.02
}
```

---

### 保底机制

```ts
if (loseStreak >= 6) {
  result >= 小奖
}
```

---

## 4.11 近中奖设计（关键上瘾点）

表现：

* 快刮出大奖形状
* 最后一点没中

👉 强化“再来一把”

---

## 4.12 反馈设计（必须）

| 类型 | 表现   |
| -- | ---- |
| 普通 | 普通弹出 |
| 小奖 | 小动画  |
| 大奖 | 强动画  |
| 传说 | 全屏特效 |

---

## 4.13 节奏控制（极关键）

必须满足：

* 刮完一个盘子：1.5 ~ 3 秒
* 不允许瞬间完成
* 不允许一次擦完

---

# 五、阶段三：刮刮乐系统（🔥核心补充）

---

## 5.1 定位（极重要）

👉 刮盘子 = 过程爽
👉 刮刮乐 = **爆点爽 + 上瘾核心**

---

## 5.2 基本玩法

玩家花费金币购买刮刮乐卡：

```ts
scratchCardCost = 100
```

---

## 5.3 结构（多区域刮开）

刮刮乐不是整张擦完，而是：

👉 多个“刮区”

### 示例：

```
[ ? ] [ ? ] [ ? ]
```

---

## 5.4 玩法规则（关键）

### 模式一：匹配型（推荐）

```txt
刮出 3 个相同 → 中奖
```

示例：

| 图案     | 结果    |
| ------ | ----- |
| 💰💰💰 | +200  |
| 💰💰❌  | 无     |
| ⭐⭐⭐    | +1000 |

---

---

### 模式二：金额叠加

```txt
刮出金额 → 累加
```

例：

```
$2 + $1 + $5 = $8
```

---

👉 MVP建议先做：

👉 **匹配型（三连）**

---

## 5.5 技术实现（与刮盘子不同）

### 刮盘子：

👉 一整张 canvas

### 刮刮乐：

👉 多个小 canvas（或 mask）

---

### 每个格子：

* 独立遮罩
* 独立擦除
* 可点击或滑动刮

---

## 5.6 交互方式

支持两种：

### 方式 A（推荐）

👉 拖动连续刮多个格子

### 方式 B

👉 点击格子自动刮开（弱一点）

---

## 5.7 判定逻辑

```ts
if (allCellsRevealed || revealCount >= 3) {
  calculateResult()
}
```

---

## 5.8 奖励池（关键）

| 类型 | 概率  | 奖励   |
| -- | --- | ---- |
| 普通 | 60% | 0    |
| 小奖 | 25% | 120  |
| 大奖 | 10% | 500  |
| 传说 | 5%  | 2000 |

---

👉 重点：

👉 大部分刮刮乐是亏的

---

## 5.9 近中奖设计（最关键）

必须实现：

👉 “差一点三连”

例：

```
💰 💰 ❌
⭐⭐ ❌ ⭐
```

---

👉 玩家心理：

👉 “刚刚就差一个！！”

---

## 5.10 动画设计（必须）

| 状态   | 表现        |
| ---- | --------- |
| 刮开   | 粉末/刮痕     |
| 匹配成功 | 闪光        |
| 大奖   | 放大 + 屏幕震动 |
| 差一点  | 停顿 0.3s   |

---

## 5.11 节奏控制

👉 一张刮刮乐：

```
2 ~ 4 秒
```

👉 不允许瞬间结束

---

## 5.12 成瘾点设计

必须同时具备：

* 不确定性
* 近中奖
* 爆点稀有
* 可连续购买

---

## 5.13 UI关键点

* 每个格子必须有厚遮罩感
* 刮开要有“粉末”
* 图案必须夸张（金币 / 星星 / 宝箱）

必须实现：
