# 刮出传说（Scratch Legend）- 核心玩法设计

---

# 一、核心定位

本游戏对标《刮个爽》的核心骨架：

```txt
洗盘子赚启动金
→ 买第一张低级刮刮卡
→ 拖动刮卡
→ 中奖 / 亏损 / 惩罚
→ 买更多不同卡
→ 升级刮卡能力与幸运
→ 解锁自动刮卡
→ 高风险卡 / 终局卡
→ Prestige 永久成长
```

游戏目标不是“公平抽奖”，而是让玩家不断产生：

👉 **“再刮一张就能翻盘 / 解锁 / 爆奖”的冲动**

# 三、核心循环

## 3.1 前 30 秒循环

1. 玩家洗盘子赚启动金。
2. 获得第一张低级刮刮卡的购买资金。
3. 购买低级刮刮卡。
4. 拖动刮开遮罩。
5. 获得小奖、亏损或惩罚。
6. 继续洗盘子或继续买卡。

## 3.2 中期循环

1. 玩家通过刮卡累积金币。
2. 解锁更贵、规则不同、风险更高的刮刮卡。
3. 用金币升级刮卡能力、幸运、倍率和自动化。
4. 通过更高等级卡追求爆奖。

## 3.3 后期循环

1. 自动刮卡开始承担基础收益。
2. 玩家挑战高风险卡和终局卡。
3. 本轮收益到达瓶颈后进行 Prestige。
4. 重置本轮进度，换取永久加成。
5. 下一轮更快进入高阶刮卡。

# 四、设计原则

## 4.1 刮刮卡是主玩法

刮刮卡是游戏的核心爽点。

洗盘子只是启动金来源，不是长期主玩法。

## 4.2 拖动刮开，不点击开奖

刮卡必须是：

```txt
购买卡片 → 生成结果 → 拖动擦除遮罩 → 渐进揭露 → 结算
```

禁止把刮刮卡做成点击按钮开奖。

## 4.3 节奏由系统控制

必须避免：

* 高速点击刷钱
* 手速决定收益
* 瞬间刮完整张卡
* 自动跳过核心揭露过程

正确体验是：

* 洗盘子是在准备启动资金
* 刮卡才是核心决策和爽点
* 升级和自动化逐步放大刮卡收益

# 五、阶段一：洗盘子启动金

## 5.1 定位

洗盘子用于让玩家获得第一笔资金。

它的目标是：

* 让玩家进入第一张刮刮卡
* 控制前期节奏
* 避免游戏变成纯点击器

## 5.2 入口与交互流程

洗盘子必须表现为左侧工作任务，而不是直接显示一个收益按钮。

基础流程：

```txt
左侧出现“日常工作”
→ 玩家点击日常工作
→ 桌子上生成一个脏盘子小物件
→ 玩家点击桌子上的脏盘子
→ 弹出 / 放大脏盘子的完整清洁视图
→ 玩家拖动海绵擦除污渍
→ 清洁进度达到 100%
→ 显示可领取金额
→ 玩家领取金币
→ 回到桌面，继续处理剩余脏盘子或等待下一次日常工作
```

## 5.3 日常工作卡片

左侧必须展示：

* 工作名称：日常工作
* 盘子费用：`$1`
* 等级：`等级 0`
* 等级进度条
* 工作图标：盘子 / 海绵

点击该卡片只负责开始工作和生成盘子，不直接结算金币。

阶段一日常工作等级规则：

* 最高等级：`等级 10`。
* 每个等级升到下一级需要的清洁盘子数，统一由 `apps/scratch-legend/lib/game-config.ts` 中的 `work.level.platesRequiredByLevel` 配置。
* 当前默认配置下，每一级先都需要清洁 `10` 个盘子；后续可按等级分别调整。
* 等级进度条只展示当前等级内进度，不展示“等级进度”文字。
* 阶段一等级收益由配置表约束，默认：
  * `等级 0/1`：每个盘子净收益 `$2`。
  * `等级 2`：每个盘子净收益 `$3`。
  * `等级 3`：每个盘子净收益 `$4`。
  * 后续等级继续由同一配置表扩展，不在组件内散写数值。

## 5.4 桌面脏盘子

日常工作开始后，桌面中央或可见区域出现一个小脏盘子。

规则：

* 盘子是可点击物件。
* 每生成一个脏盘子固定花费 `$1`，任何等级都不随机调整盘子价格。
* 游戏开局金币为 `$1`，保证玩家能生成第一个脏盘子。
* 短点击盘子后立即进入放大清洁视图。
* 长按盘子进入可拖动状态，并在移动后开始拖动；拖动应跟手、不卡顿。
* 桌面盘子悬浮时需要有轻微晃动反馈，长按进入拖动时需要有拿起阴影反馈。
* 未点击盘子前不能清洁。
* 玩家可以连续点击日常工作生成多个脏盘子；多个盘子可以在桌面上重叠，每个盘子独立携带本次工作奖励。
* 新脏盘子从桌子顶部飞入桌面，最终落点在桌面可操作区域内随机生成，不固定在中心点。

## 5.5 放大清洁视图

放大视图中必须包含：

* 大盘子
* 污渍层
* 海绵 / 清洁工具
* 日常工作说明浮层
* 奖励金额

玩家需要拖动海绵擦除污渍。

清洁完成前不允许领取奖励。

清洁视图中，鼠标悬浮到盘子污渍区域时显示为海绵 cursor；按住鼠标拖动才执行擦除。

清洁说明浮层只展示日常工作说明和中奖 / 碎盘概率，不展示清洁进度；玩家通过盘面污渍消失直接感知清洁程度。

## 5.6 清洁判定

推荐使用 Canvas 或等价遮罩：

```ts
ctx.globalCompositeOperation = 'destination-out'
```

清洁进度由污渍遮罩擦除比例决定，内部使用 `0~1` 的比例值：

```ts
cleanCompleteThreshold = 0.95

if (cleanRatio >= cleanCompleteThreshold) {
  canClaimReward = true
}
```

显示给玩家时仍然表现为 `100%` 完成。

## 5.7 行为规则

一次洗盘子动作必须有动作锁：

```ts
actionDuration = 400~800ms
```

收益在整只盘子清洁完成并领取时结算。等级 `0` 时不会出现碎盘：

```ts
if (workLevel < 1) {
  reward = +2
}
```

日常工作达到 `等级 1` 后引入轻惩罚，并通过电话告知玩家：

```ts
if (random < 0.9) {
  reward = +configuredLevelReward
} else if (gold >= 3) {
  reward = -3
  plateBroken = true
} else {
  reward = +configuredLevelReward
}
```

注意：

* 动作锁用于限制清洁采样和结算节奏。
* 收益必须在整只盘子清洁完成后结算。
* 不允许每次点击或每次擦拭都直接给钱。
* 如果金币不足以承担 `$3` 扣除后继续购买 `$1` 盘子，则本次不能生成碎盘惩罚，避免玩家无法继续。
* 碎盘结果需要有明确的盘子破碎反馈，并在领取时扣除金币。

## 5.8 电话与辅助道具提示

阶段一包含三类电话 / 解锁提示：

1. 垃圾桶解锁：
   * 辅助道具页默认只显示“未解锁”占位，不在桌面展示垃圾桶。
   * 当玩家累计清洁第 `3` 个盘子后，垃圾桶自动解锁，不消耗金币。
   * 解锁时在桌面中央播放“已解锁 垃圾桶”发光提示。
   * 解锁后垃圾桶显示在桌面右下角。
   * 脏盘子拖入垃圾桶区域时，垃圾桶打开；在垃圾桶上松开鼠标后，该脏盘子直接消失，垃圾桶自动关闭。

2. 碎盘风险电话：
   * 当日常工作达到 `等级 1` 后，电话响铃并弹出风险说明。
   * 风险说明浮层参考清洁说明卡片样式，展示 `90% +收益` 和 `10% -$3`。
   * 玩家点击“我知道了”后关闭该电话消息。

3. 刮刮乐解锁电话：
   * 左上角金币进度改为展示“累计赚到的金币 / 当前累计里程碑”。
   * 累计赚到 `$10` 后，电话响铃并提示已可解锁刮刮乐模式。
   * 同时播放一次刮刮乐解锁发光提示。
   * 阶段一只展示解锁提示，不实现刮刮乐玩法；刮刮乐玩法在阶段二构建。
   * 配置文件中需要支持继续声明后续累计里程碑，例如累计 `$50` 的下一功能占位，但当前阶段不提前实现对应玩法。

阶段一已落地数值统一收敛到：

```text
apps/scratch-legend/lib/game-config.ts
```

要求：

* 当前阶段已经实现的金币、奖励、等级、电话提示、垃圾桶解锁和交互节奏，统一以该配置文件为单一来源。
* 累计金币解锁也必须从该配置文件读取，不能在组件内写死 `10`、`50` 这类里程碑阈值。
* `game-config.ts` 只负责静态规则，不负责保存玩家当前进度。
* 如果后续调整阶段一数值，先更新本文档，再同步该配置文件和任务文档。

阶段一运行态存档结构统一收敛到：

```text
apps/scratch-legend/lib/game-save.ts
```

要求：

* 持久化时必须单独保存当前金币、累计赚到的金币、当前已解锁状态、电话消息已读状态、桌面盘子列表和当前工作阶段。
* 存档结构与静态配置分离，避免把“默认规则”和“当前进度”混在同一份对象里。
* 后续接本地存档、云存档或版本迁移时，统一以该存档结构为入口。

阶段一前端状态管理统一收敛到：

```text
apps/scratch-legend/lib/game-store.ts
```

要求：

* 前端运行态使用 `zustand` 管理。
* 本地持久化默认使用浏览器 `localStorage`。
* `game-store.ts` 负责状态读写和持久化接线，`game-save.ts` 负责存档结构与默认值，二者职责分离。

## 5.9 强制规则

* 动作未结束不能再次结算收益。
* 连续点击只能增强视觉反馈，不能增加结算次数。
* 收益不能与手速线性绑定。
* 玩家接近第一张卡价格时，只提示可购买，不自动跳转。
* 点击日常工作不结算。
* 点击脏盘子不结算。
* 只有清洁完成并领取时才增加金币。

## 5.10 节奏目标

```txt
30~60 秒 → 累计赚到 `$10` 并收到刮刮乐模式解锁提示
```

# 六、阶段二：第一张低级安全卡

## 6.1 定位

阶段二的任务不是“把所有刮卡花样一次做完”，而是先建立《刮个爽》最核心的主循环入口：

```txt
买卡
→ 拖动刮开
→ 看见结果
→ 领奖或亏损
→ 决定继续买卡还是回去洗盘子
```

第一张卡必须承担教学职责，让玩家立刻理解三件事：

* 刮刮卡需要先买。
* 结果在买下那一刻就已经决定。
* 拖动揭露本身就是主玩法的一部分。

## 6.2 解锁位置

阶段二入口继续沿用阶段一节奏：

```ts
basicCardUnlockGold = 10
```

这里的 `$10` 是当前 Scratch Legend 的本地阶段切换阈值，不宣称等于《刮个爽》原始数值。

## 6.3 卡片结构

每张刮刮卡都由两层组成：

### 底层：预生成结果层

* 奖励符号
* 空白 / 亏损结果
* 后续阶段才会接入的惩罚符号
* 本张卡的结算信息

### 上层：可擦除刮层

* 灰白遮罩
* 可被拖动擦除
* 有连续刮痕反馈
* 不能通过单击跳过

## 6.4 交互原则

玩家操作固定为：

* 按下：开始刮
* 拖动：持续擦除
* 松开：停止擦除

强制规则：

* 禁止点击直接开奖。
* 禁止在最后一刻才随机结果。
* 刮开过程只负责揭露，不负责重新抽奖。

## 6.5 当前推荐的首张卡规则

按高置信骨架，第一张卡优先采用“低风险教学卡”而不是重惩罚卡。

当前推荐规则：

```txt
3 格结果区
任意 2 格相同即中奖
否则为无奖或普通亏损
```

这样更贴近《刮个爽》早期“先教玩家刮和领奖，再逐步引入风险”的节奏。

## 6.6 阶段二本地数值原则

阶段二只锁定数值原则，不锁定《刮个爽》原作精确概率：

* 普通亏损应占大头。
* 小奖必须稳定出现，保证玩家能继续买卡。
* 偶尔出现明显高于卡价的大奖，提供翻盘期待。
* 第一张卡不承担重惩罚职责，重惩罚放到阶段三。

结算判定可采用“关键区域揭露完成”或“有效刮开比例达标”的本地实现，但这属于 Scratch Legend 的实现细节，不视为原作规则声明。

# 七、风险卡与 Peek 机制

## 7.1 定位

《刮个爽》真正拉开层次的，不只是“中奖了没”，而是：

```txt
我继续刮，可能爆奖
我继续刮，也可能把惩罚完全揭开
```

因此风险卡不是阶段二主菜，而是阶段三及之后的重要扩展。

## 7.2 惩罚符号原则

高置信骨架下，惩罚应遵守以下方向：

* 惩罚与具体危险符号绑定。
* 惩罚不是“翻牌瞬间凭空判定”，而是和揭露行为相关。
* 危险结果通常应在“完全揭露”后才真正触发。

可用的轻惩罚方向：

* 本张卡收益归零
* 扣除少量金币
* 扣除一次卡片购买资格或附加一次成本

前中期不允许用重惩罚直接打断主循环。

## 7.3 Peek 与止损

中后期卡片要为以下能力预留：

* 玩家先刮开边角或局部区域
* 通过局部信息判断是否存在危险符号
* 在完全揭露前决定继续还是放弃

这类“半揭露决策”是《刮个爽》区别于纯开奖小游戏的重要特征。

## 7.4 当前取舍

* 阶段二先不强制实现弃卡。
* 阶段三开始正式引入“轻惩罚 + 局部判断”。
* 文档和数据结构从阶段二起就要为惩罚符号、弃卡和风险卡预留扩展位。

# 八、卡片进阶系统

## 8.1 定位

后续卡片的进阶方向，不应该只是“更贵、数值更大”，而必须让玩家面对不同类型的决策。

## 8.2 高置信进阶梯度

当前按高置信骨架，把卡片梯度拆成四类：

| 卡类 | 定位 | 体验目标 |
| --- | --- | --- |
| 安全卡 | 教学 / 启动循环 | 学会买卡、刮卡、领奖 |
| 风险卡 | 中期主力 | 小心惩罚，开始判断要不要继续刮 |
| 高风险卡 | 后期爆点 | 高奖池、高惩罚、明显翻盘感 |
| 终局卡 | 本轮高潮 | 为 Prestige 前的最后冲刺服务 |

## 8.3 解锁原则

卡类解锁应由以下维度推动，而不是只看当前金币：

* 累计刮卡次数
* 历史总收益
* 已拥有的升级
* 是否已进入 Prestige 后轮次

## 8.4 设计要求

* 新卡必须改变玩家行为，而不是只提高赔率或倍率。
* 至少一部分卡片必须让玩家感到“贵，但值得搏一把”。
* 终局卡必须服务于“本轮高潮”和“进入下一轮成长”。

# 九、升级系统

## 9.1 定位

升级系统负责把玩家从“手动刮单张卡”推进到“更快、更稳、更贪、更自动化”。

## 9.2 高置信升级方向

当前优先采用以下四类升级：

| 升级类 | 作用 |
| --- | --- |
| 幸运 | 提高中正收益和稀有收益的能力 |
| 刮擦效率 | 提升刮开面积、力度或完成速度 |
| 奖励放大 | 提升正收益的回报能力 |
| 自动化 / 功能解锁 | 解锁自动刮卡和中后期辅助处理 |

## 9.3 设计原则

* 前期优先改善手感与稳定性。
* 中期开始放大奖励与幸运。
* 自动化必须作为中期转折点，而不是开局就给。
* 不允许出现升级后完全跳过刮开过程的情况。

---

# 十、自动化系统

## 10.1 定位

自动化是中期后的明确转折点，作用是接管重复劳动，而不是替代所有决策。

## 10.2 核心能力

自动化至少需要覆盖这些方向中的一部分：

* 自动购买低风险卡
* 自动刮开低风险卡
* 自动结算结果
* 自动处理低价值重复操作

## 10.3 强制规则

* 不允许瞬间批量开奖。
* 自动化必须保留时间消耗。
* 高风险卡默认不自动处理，或需要玩家显式开启。
* 自动化上线后，玩家仍应保留对高风险决策的控制权。

---

# 十一、高风险卡与终局卡

## 11.1 高风险卡

高风险卡是后期的刺激来源，应同时满足：

* 成本明显更高
* 爆奖回报明显更高
* 惩罚更强
* 更依赖风险判断或弃卡决策

## 11.2 终局卡

终局卡不是稳定刷钱卡，而是每轮接近 Prestige 前的目标卡。

它的职责是：

* 给本轮提供最后一次显著高潮
* 让玩家决定是继续贪还是准备重置
* 作为进入 Prestige 的情绪和经济踏板

---

# 十二、Prestige 永久成长

## 12.1 定位

Prestige 是完整骨架不可缺少的一环。

当本轮收益增长明显放缓后，玩家应能主动重置以下内容：

* 当前金币
* 当前已解锁卡类
* 本轮升级
* 本轮自动化进度

换取永久点数或永久成长。

## 12.2 永久成长方向

永久成长优先覆盖以下高置信方向：

| 永久成长 | 作用 |
| --- | --- |
| 起始资金 | 更快回到第一张卡 |
| 永久幸运 | 全局提高优质结果出现率 |
| 永久收益放大 | 提升轮回后的滚雪球速度 |
| 永久刮擦基础 | 提高开局操作效率 |
| 自动化提早接入 | 更快回到中期节奏 |

## 12.3 节奏要求

Prestige 必须让玩家感受到：

* 这轮结束得值
* 下一轮明显更快
* 虽然重置了，但没有白玩

---

# 十三、前 10 分钟体验目标

## 0~1 分钟

* 玩家学会洗盘子。
* 玩家理解洗盘子只是门票。

## 1~3 分钟

* 玩家买到并完成第一张安全卡。
* 玩家建立“买卡 -> 刮卡 -> 领奖 / 亏损”的基础循环。

## 3~5 分钟

* 玩家完成多张低级卡。
* 玩家开始感到“再来一张可能翻盘”。

## 5~10 分钟

* 玩家看到升级目标和下一类卡片目标。
* 风险卡或自动化目标开始出现在界面上。
* 玩家明确意识到后续主菜是刮卡，不是洗盘子。

---

# 十四、防崩机制

必须避免：

* 洗盘子重新变回长期主玩法
* 玩家连续亏损后完全无法继续
* 惩罚过重导致前中期直接放弃
* 自动化绕过核心揭露过程
* 高级卡只有更大数字，没有新决策

## 14.1 破产保护

如果玩家短期亏空，必须始终能通过洗盘子重新获得门票资金。

## 14.2 风险保护

风险卡与惩罚卡必须遵守：

* 先轻后重
* 先教会判断，再放大奖惩
* 不允许前几分钟出现一次错误就直接崩盘

## 14.3 自动化保护

自动化不应替代全部手动内容，而应把玩家从重复劳动里解放出来，把注意力推向高风险决策。

---

# 十五、MVP 最低标准

当前高置信骨架下，MVP 必须至少具备：

* 洗盘子赚启动金
* 解锁并购买第一张低级安全卡
* 购买时预生成结果
* 拖动擦除遮罩
* 渐进揭露结果
* 中奖与普通亏损
* 为阶段三预留惩罚符号和弃卡扩展位

如果缺少“买卡 -> 拖动刮开 -> 结算 -> 继续买卡”的循环，这个项目就还没有真正站到《刮个爽》的主玩法上。

---

# 十六、开发优先级

1. 洗盘子启动金。
2. 第一张低级安全卡与拖动刮开。
3. 风险卡、轻惩罚与 Peek。
4. 更多卡类与升级系统。
5. 自动化。
6. 高风险卡与终局卡。
7. Prestige。

---

👉 核心认知：

**洗盘子是门票，安全卡是入场教学，风险卡才开始长出《刮个爽》的味道，升级、自动化和 Prestige 负责把这个循环拉长。**
