# 玩具风平台重制任务清单

> 目标：参考《Climber Animals Together / 攀爬动物在一起》的多人攀爬乐趣、区域递进和机关密度，重做为 Valley MAS 自有的玩具世界美术与玩法资产。  
> 原则：只复用玩法类型和关卡节奏方法，不复制原作模型、关卡布局、角色、UI、音效或可识别造型。

---

## 资料依据与边界

### 已确认的公开信息

- Steam 商店页显示该游戏是多人合作攀爬/平台玩法，支持 1-8 人，强调从谷仓顶部一路爬到顶峰，并包含动态障碍。
- Steam 商店页示例障碍包含旋转锤、拳击手套和旋转杆等，会在攀爬过程中制造掉落风险。
- Steam 成就页能看到高度进度节点（如 100m、250m、500m、800m）、城堡段和 Olympos 等进度/区域提示。
- SteamDB 信息页可作为版本、标签、类型与基础元数据参考。

### 不能做的事

- 不照抄原作的模型比例、贴图、材质、区域布局、镜头构图、UI 或音效。
- 不把公开视频逐帧复刻成相同跳点路线。
- 不使用未经授权的原作模型、截图切片、音频或反编译数据。

### 可以做的事

- 提炼“平台功能类型”：静态落脚、动态预判、陷阱干扰、垂直辅助、区域主题递进。
- 做“玩具风等价物”：积木、软胶、木质玩具、磁吸零件、弹簧、轨道、云朵、发条机械。
- 按相同体验目标设计原创路线：新手热身、节奏平台、动态机关、低容错冲顶。

## 总体路线

### Phase 1：资产与机制字典

- [x] P1-01 建立 `ToyPlatformKind` 类型，覆盖静态、动态、功能、攀爬辅助、主题组合平台。
- [~] P1-02 建立平台视觉生成器目录：`src/prototype/platformVisuals/*`，把当前 `toyPlatformVisuals.ts` 拆分成可组合模块。
- [x] P1-03 建立平台机制映射表：视觉种类、碰撞形状、物理参数、音效、粒子、难度标签。
- [x] P1-04 更新 `docs/ASSET_GUIDE.md`，把程序化平台资产纳入资源登记。
- [x] P1-05 设计关卡 DSL 字段：`kind`、`themeZone`、`difficultyTier`、`mechanicTags`、`visualVariant`。
- [x] P1-06 将成熟的程序化平台结构导出或重制为可复用 GLB 资产文件。（S1-S3 首批 GLB 已生成并接入主地图加载器）

### Phase 2：静态基础平台资产

- [x] S1 方形平台：玩具积木板、拼接地砖、木质积木盒、软胶垫，作为主路径和缓冲区。（已有程序化 Mesh 组合 + `toy_square_plate_s1.glb`）
- [x] S2 圆形平台：纽扣圆盘、瓶盖圆台、发条齿轮圆盘，用于精准落点。（已有程序化 Mesh 组合 + `toy_round_disc_s2.glb`）
- [x] S3 长条踏板：雪糕棒、铅笔桥、磁力条、塑料轨道，用于连续节奏跳。（已有程序化 Mesh 组合 + `toy_narrow_plank_s3.glb`）
- [ ] S4 不规则碎片平台：破碎拼图块、泡沫碎片、积木残片，中心安全、边缘视觉危险。
- [ ] S5 堆叠平台组：积木堆、玩具箱、桶状积木、软木块阶梯，用于开局教学。
- [~] S6 为所有静态平台生成 2-3 个颜色/边框/凸点变体，避免大地图重复感。（已扩展谷仓/城堡 15 个物件化变体）

### Phase 3：动态机关平台资产与机制

- [~] D1 摇晃平台：跷跷板积木、吊挂木板、弹簧支撑板，支持周期摇摆和踩踏加剧。（已接入 `toy_barn_seesaw_board`，复用不稳定平台状态机）
- [ ] D2 移动平台：电动玩具轨道车、滑轨托盘、升降积木板，复用现有 `moving`。
- [ ] D3 旋转平台：齿轮圆盘、十字风车、旋转积木叶片，强化中心安全/边缘危险的视觉提示。
- [~] D4 伸缩平台：抽屉积木桥、伸缩尺踏板、弹出式玩具桥，新增 `extendable` 机制。（已接入 `toy_castle_extendable_ruler_bridge`）
- [~] D5 倾斜平台：平衡木板、半球底座台、软胶倾斜板，新增 `tilting` 机制。（已接入 `toy_castle_tilt_balance_board`）
- [~] D6 忽隐忽现平台：透明亚克力积木、发光拼图块，复用并升级现有 `blink`。（已接入 `toy_blink_acrylic_panel`）

### Phase 4：特殊功能平台资产与机制

- [ ] F1 蹦床平台：弹簧床、软胶鼓面、大号玩具弹垫，复用现有 `bouncy` 并增加可视化压缩。
- [~] F2 弹跳垫：小型弹力贴片、蘑菇垫、云朵垫，新增弱弹力参数模板。（已接入 `toy_cloud_bounce_pad`）
- [~] F3 传送带平台：积木履带、玩具工厂传送带，复用并调优现有 `conveyor`。（已接入 `toy_conveyor_track_belt`）
- [~] F4 冰块平台：透明蓝色积木、塑料冰片，复用现有 `icy` 并增加滑痕/高光。（已接入 `toy_plastic_ice_block`）
- [~] F5 粘性平台：软糖垫、橡皮泥板、魔术贴平台，新增 `sticky` 低速/低跳机制。（已接入 `toy_gummy_sticky_pad`）
- [~] F6 死亡/崩塌平台：裂纹积木、破碎饼干板、坏掉拼图块，复用并升级现有 `crumble`。（已接入 `toy_cracked_puzzle_crumble`）

### Phase 5：攀爬辅助模块

- [ ] C1 攀爬墙：带凸点的积木墙、玩具攀岩板，新增 `climbWall` 检测与上下移动。
- [ ] C2 绳索/藤蔓：彩绳、跳绳、挂钩绳，新增 `swingRope` 抓取/摆荡/松手。
- [ ] C3 梯子：木质玩具梯、金属玩具梯，新增 `ladder` 区域控制。
- [ ] C4 杆状平衡柱：铅笔柱、积木圆柱、棒棒糖柱，使用圆柱碰撞和窄顶面难度。
- [ ] C5 辅助模块统一调试视图：显示抓取范围、可攀爬面、摆荡半径和落点预测。

### Phase 6：四大主题区域重构

- [ ] Z1 谷仓玩具区（0-25%）：草垛积木、木箱、木桶、低速摇晃板、蹦床。目标是教学和建立节奏。
- [ ] Z2 城堡积木区（25-50%）：石砖积木、大齿轮圆盘、伸缩吊桥、攀爬墙。目标是预判和垂直衔接。
- [ ] Z3 高空岛屿区（50-75%）：金属板、移动圆盘、忽隐忽现碎片、平衡柱。目标是低容错和节奏记忆。
- [ ] Z4 奥林匹斯玩具云巅（75-100%）：大理石玩具台、旋转光环、弹跳云、摆荡绳。目标是综合机制终局挑战。
- [ ] Z5 每个区域至少设置 2 个喘息台，保证掉落惩罚重但不让路线不可读。
- [ ] Z6 每个区域制定颜色、材质、背景剪影、音效和粒子规范。

### Phase 7：地图数据与平衡

- [ ] M1 新建 `src/levels/toyAnimalClimbWorld.ts`，作为玩具风重制主路线。
- [ ] M2 先做 25m 垂直切片，只包含谷仓玩具区，验证基础手感。
- [x] M3 扩展到 60m，加入城堡积木区和 2-3 种动态机关。
- [ ] M4 扩展到 100m，加入高空岛屿区和奥林匹斯玩具云巅。
- [ ] M5 每 10m 做一次落点审查：平台尺寸、边缘距离、顶面高差、相机遮挡、失败回退成本。
- [ ] M6 建立 `ClimberJumpClearanceReport` 的截图/日志验收流程。

### Phase 8：多人友好与可读性

- [ ] U1 大平台保留 2-4 人同时站立空间，避免多人互相遮挡。
- [ ] U2 窄平台只用于短节奏，不连续超过 3 个。
- [ ] U3 高风险机关前加预告模型：轨道、箭头、发光边、倒计时材质。
- [ ] U4 掉落路径尽量可看见上一层结构，让玩家知道自己失去了什么。
- [ ] U5 HUD 增加当前区域名、高度进度和最佳高度，支持掉落后的目标感。

### Phase 9：大场景空间与掉落回收层重排

- [~] W1 设计宽幅空间规则：每个高度段使用横向岛组，XZ 范围逐段扩大（谷仓约 ±18m、城堡约 ±24m、高空约 ±30m、云巅约 ±36m），避免路线像单根竖直楼梯。
- [~] W2 将 0-60m 从单线跳点改为平台群：每段包含主路径、侧路、接落层、诱导平台和风险空洞，让掉落后的结果更丰富。
- [~] W3 设置关键接落层：约 20m 谷仓屋顶/草垛堆，约 35m 城堡外墙/大吊桥，约 50m 塔楼横梁/玩具集装箱。
- [ ] W4 接落层不做存档点：只作为真实可落地空间，玩家仍会损失高度和路线进度，保留掉落惩罚。
- [ ] W5 每个 8-12m 高度段至少包含 1 个主路平台群、1 个侧向接落平台和 1 个明确风险空洞。
- [~] W6 增加横向回爬路线：玩家落到接落层后，可通过 2-4 个平台重新接回主路，而不是只能归零重来。
- [ ] W7 建立大场景可读性提示：高风险路线前放置轨道、边框、箭头、材质变化或背景剪影，避免玩家看不懂下一跳。
- [ ] W8 建立掉落模拟验收：从 30m、45m、60m 三处测试自然掉落，确认部分轨迹会被低层结构接住，同时仍保留掉到底的风险。
- [~] W9 视觉升级：平台模块从单块模型升级为区域场景物件组合；装饰件不参与碰撞，可站部件必须有模型级碰撞。
- [ ] W10 浏览器截图验收：普通视图和 debug collider 视图都覆盖接落层、侧路和主路径，确认没有空气墙和明显穿模。

## 建议实施顺序

1. 先完成 Phase 1 + Phase 2：把“平台资产字典”和静态平台库做好。
2. 再完成 Phase 3 的 D1-D3：摇晃、移动、旋转三种最能提升趣味的动态平台。
3. 做 25m 谷仓玩具区垂直切片，实际玩 5-10 分钟，调跳跃手感。
4. 加 Phase 4 的蹦床、传送带、崩塌平台，形成第一版 60m 路线。
5. 重排 0-60m 大场景空间，把谷仓和城堡段从单线跳点改成宽幅平台群，并加入接落层。
6. 最后再做攀爬墙、绳索和奥林匹斯终局区，因为它们会改输入和物理。
