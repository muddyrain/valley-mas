# 博客 AI MVP 冻结与 UI 接入方案（BAI-2 / BAI-3）

## 1. 冻结结论（BAI-2）

### 1.1 首批上线能力（MVP In）

- `AI 导读`
- `问文章`

### 1.2 暂缓能力（MVP Out）

- `章节速览`（延期到 BAI+1）

### 1.3 冻结理由

- `AI 导读`：
  - 首屏可感知，阅读转化收益高。
  - 无需复杂多段索引，首版实现风险可控。
- `问文章`：
  - 用户感知最强，能直接验证“AI 是否真有帮助”。
  - 可复用“文内检索 + 回答 + 引用锚点”主链路，为后续能力打底。
- `章节速览` 暂缓：
  - 依赖章节级切片质量与目录映射稳定性，首版复杂度高于收益。
  - 避免首发阶段同时维护三条异步链路造成体验不稳定。

## 2. 上线范围约束

- 仅支持博客详情页（`/blog/:id`）和博客列表页（`/blog`）入口展示。
- 仅支持当前文章上下文，不跨文章回答。
- 首版不做个性化记忆，不做自动持续生成。

## 3. UI 接入方案（BAI-3）

## 3.1 博客详情页（`apps/web/src/pages/blog/BlogPost/index.tsx`）

### A. AI 导读入口与反馈

- 入口位置：
  - 详情页首屏“阅读导览”卡片中，新增按钮：`生成 AI 导读`
- 交互状态：
  - 默认态：按钮可点击，展示“还未生成导读”
  - 生成中：按钮 loading，文案 `正在生成导读...`
  - 成功态：展示导读正文 + 3 个阅读重点 + 建议阅读路径
  - 失败态：展示提示 `导读暂时生成失败，请稍后重试。`，保留重试按钮
- 保底：
  - 若失败且有文章摘要，继续显示原摘要
  - 若无摘要，显示“请直接阅读正文”

### B. 问文章入口与反馈

- 入口位置：
  - 正文区与评论区之间（`PostComments` 组件之前）
- 交互结构：
  - 输入框（占位：`问问这篇文章的观点、结论或细节`）
  - 发送按钮（`提问`）
  - 回答区（答案 + 引用片段 + 章节锚点）
- 交互状态：
  - 默认态：输入可编辑
  - 生成中：按钮 loading，保留输入内容
  - 成功态：显示答案与引用片段（最多 2 段）
  - 失败态：`暂时无法回答这个问题，请换个问法或稍后再试。`
- 保底：
  - 失败时不清空输入，支持一键重试

## 3.2 博客列表页（`apps/web/src/pages/blog/BlogList/index.tsx`）

### A. 轻入口策略（不打断主阅读流）

- 入口位置：
  - 顶部搜索卡右侧新增轻入口按钮：`AI 推荐读哪篇`
- 行为：
  - 点击后弹出轻量面板（非整页跳转）
  - 输入“想读主题/问题”，返回 3 篇推荐博客（标题 + 推荐理由）
- 结果跳转：
  - 点击推荐项进入详情页，并带上来源 state：`source: 'blog-ai-recommend'`

### B. 失败兜底

- 面板内提示：`暂时无法生成推荐，请稍后再试。`
- 保底：继续使用现有关键词搜索 + 分组筛选，不影响列表功能

## 4. 统一反馈文案（首版）

- `正在生成导读...`
- `导读暂时生成失败，请稍后重试。`
- `暂时无法回答这个问题，请换个问法或稍后再试。`
- `暂时无法生成推荐，请稍后再试。`

## 5. 验收口径

- BAI-2：
  - 首批能力是否冻结为 2 项，并明确 1 项延期。
- BAI-3：
  - 详情页与列表页入口位置是否明确。
  - 每个入口是否有默认/生成中/成功/失败四态。
  - 失败时是否不破坏现有博客阅读与搜索主链路。
