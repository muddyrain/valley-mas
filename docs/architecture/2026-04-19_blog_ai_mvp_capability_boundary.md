# 博客 AI 能力边界文档（BAI-1 / MVP）

## 1. 目标与范围

- 目标：为博客阅读场景定义首批 AI 能力边界，避免后续 UI 接入时反复改结构。
- 范围：仅覆盖 `apps/web` 博客列表与博客详情页的访客阅读链路。
- 本文不包含：模型选型、服务端实现细节、最终视觉稿。

## 2. MVP 能力清单（冻结候选）

- 能力 A：`AI 导读`
- 能力 B：`章节速览`
- 能力 C：`问文章`

## 3. 能力边界（输入/输出/入口/失败兜底）

### 3.1 AI 导读

- 输入：
  - `postId`
  - 文章标题、摘要（可为空）、正文纯文本（截断后）
  - 当前用户语言（默认中文）
- 输出：
  - 120~220 字导读
  - 3 个阅读重点（短句）
  - 1 个建议阅读路径（如“先看第 2、4 节”）
- 页面入口：
  - 博客详情页头部导览区（首屏可见）
- 触发方式：
  - 手动点击“生成导读”；可缓存最近一次结果
- 失败兜底：
  - 文案：`导读暂时生成失败，请稍后重试。`
  - 保底：展示文章原摘要；无摘要时展示“请直接阅读正文”

### 3.2 章节速览

- 输入：
  - `postId`
  - 目录结构（TOC）
  - 各章节正文片段（按标题分段截断）
- 输出：
  - 按章节返回 1~2 句摘要
  - 每章 2~4 个关键词
  - 可选阅读时长估计（分钟）
- 页面入口：
  - 博客详情页目录区域（桌面侧栏 / 移动端目录面板）
- 触发方式：
  - 手动触发“生成章节速览”
- 失败兜底：
  - 文案：`章节速览暂时不可用。`
  - 保底：回退为原始目录列表，不阻塞目录跳转

### 3.3 问文章

- 输入：
  - `postId`
  - 用户问题（长度限制）
  - 文章正文检索片段（仅当前文章）
- 输出：
  - 直接回答（短段落）
  - 引用片段（最多 2 段）与章节锚点
  - 置信提示（如“基于文内信息回答”）
- 页面入口：
  - 博客详情页正文下方（评论区上方）
- 触发方式：
  - 手动输入问题并发送
- 失败兜底：
  - 文案：`暂时无法回答这个问题，请换个问法或稍后再试。`
  - 保底：保留用户输入，不清空问题框

## 4. 统一约束（MVP 必须遵守）

- 只回答当前文章，不跨文章扩展。
- 不输出内部提示词、系统指令、推理过程原文。
- 中文输出优先，风格为产品文案，不使用“提示词感”措辞。
- 单次响应设定超时与重试上限，避免长时间阻塞页面。
- 统一失败态：轻量提示 + 保底可继续阅读，不打断主链路。

## 5. 非目标（本阶段不做）

- 不做跨文章知识库与全站问答。
- 不做个性化长期记忆。
- 不做自动连续生成（全部能力先以手动触发为准）。

## 6. 验收口径（供 BAI-2 使用）

- 能力范围是否冻结为 2~3 个且边界清晰。
- 每个能力是否具备输入/输出/入口/失败兜底四要素。
- 失败时是否不破坏博客阅读主流程。
