# 名著 CLD-2 导入标准与清洗流程（首版冻结）

## 1. 目标

- 冻结 `epub / txt / html` 三种格式的导入规则，使后续工具链与存储结构可以单向推进，不因格式差异返工。
- 明确章节切分、脚注处理、图片占位三条核心清洗规则。
- 输出给 CLD-3（元数据模型）的字段约束，以及给 CLR-2（阅读器 MVP）的章节数据契约。

---

## 2. 支持格式一览

| 格式 | 优先级 | 适用来源 | 说明 |
| --- | --- | --- | --- |
| `epub` | 高 | Project Gutenberg、Wikisource 可机读版、国内公开古籍库 | 结构最完整；优先使用 |
| `txt` | 中 | 中文维基文库、无结构化历史文本 | 需依赖正则推断章节边界 |
| `html` | 低 | 站点页面抓取、Wikisource 页面 | 需先去噪再解析；最后兜底 |

---

## 3. 章节切分规则

### 3.1 epub 格式

- 以 `<nav>`/`toc.ncx`/`toc.xhtml` 中的 `navPoint` 顺序作为章节列表权威来源。
- 每个 `navPoint` 对应一个逻辑章节，`playOrder` 决定顺序。
- 若 `toc` 不存在或条目 < 3，回退到按 `spine` 文件列表切分。
- 切分单元：`chapter_index`（从 0 起）、`chapter_title`（取 `navPoint.navLabel.text`）、`chapter_html`（取对应 spine 文件内容）。

### 3.2 txt 格式

优先级从高到低，命中即停：

1. **显式章节行**：行文本匹配 `^(第[一二三四五六七八九十百千\d]+[章节回]|Chapter\s+\d+)[^\n]*$`（去首尾空白后）。
2. **数字标题行**：匹配 `^\d+[\.\、。]\s*.+$` 且单行字数 <= 30。
3. **空行分割**：连续 >= 3 空行视为章节边界。
4. **无切分信号**：整体作为单一章节，`chapter_title` 取书名，`chapter_index` 为 0。

### 3.3 html 格式

1. 先用 `Readability`（或等效去噪算法）提取正文节点，丢弃导航栏、页脚、广告区。
2. 在正文节点内按 `<h1>/<h2>/<h3>` 标题层级切分章节；若无标题则整体为单章节。
3. 保留正文内的 `<blockquote>`、`<em>`、`<strong>`、`<sup>`（脚注锚点）。
4. 丢弃 `<script>`、`<style>`、`<form>`、`<iframe>`、注释节点。

---

## 4. 脚注处理规则

| 场景 | 处理方式 |
| --- | --- |
| epub 内置脚注（`epub:type="footnote"` 或 `role="doc-footnote"`） | 提取脚注 `id` 与脚注正文，存入 `footnotes[]` 字段；正文中保留锚点 `[^n]` 标记 |
| html `<sup>` 锚点 + 页内跳转 `#fn-xxx` | 同上；若跳转目标不在同页则视为无效脚注，降级为行内括号注释 |
| txt 行末括号注（形如 `（注：…）` 或 `（1）…`） | 识别后提取到 `footnotes[]`；正文保留 `[注n]` 占位 |
| 译注（含"译者注""译注""注"前缀） | 提取并打上 `type: translator_note` 标签，与作者注区分 |
| 无法解析的脚注 | 保持原样内联，打上 `raw: true` 标记，不影响章节切分 |

---

## 5. 图片占位规则

名著阅读场景以文字为主，图片以插图与封面为主。

| 类型 | 处理方式 |
| --- | --- |
| 封面图（epub `cover-image`，或 html 首张全屏图） | 下载并存储；绑定到 `book.cover_url` 字段 |
| 章节内插图（正文 `<img>` 或 epub `svg` 图片） | 下载并存储；正文中用 `[插图 n]` 占位，`illustrations[]` 字段存储 URL 与位置索引 |
| 数学公式图（alt 含 "formula"/"equation" 或 MathML） | 保留原始 MathML/LaTeX 字符串；不下载位图 |
| 导航/装饰图（宽 <= 32px 或 alt 为空且宽高比极端） | 丢弃，不占位 |
| 外链图（src 为 http/https 且域名不在白名单） | 替换为 `[外链图片，已隐藏]` 占位，不下载 |

---

## 6. 文本清洗通则

以下规则对三种格式均适用：

1. **编码归一**：统一转换为 UTF-8，BOM 去除。
2. **换行规范**：`\r\n` → `\n`，连续 >= 3 个 `\n` 压缩为 2 个。
3. **全/半角标点**：不做强制转换，保留原文风格。
4. **零宽字符**：去除 `\u200b / \u200c / \u200d / \ufeff`。
5. **HTML 实体**：在 txt/html 场景内解码常见实体（`&amp;` → `&`，`&lt;` → `<` 等）。
6. **广告/免责声明**：若段落命中 `"本书由.*整理"/"版权所有"/"转载请注明"/"此电子书"` 等模式，整段标记 `skip: true`，不写入章节正文。

---

## 7. 清洗流程（Pipeline）

```
原始文件（epub/txt/html）
  ↓ 1. 格式探测与编码归一
  ↓ 2. 去噪（html 走 Readability；epub 走 spine；txt 走全文）
  ↓ 3. 章节切分（见第 3 节规则）
  ↓ 4. 脚注抽取（见第 4 节规则）
  ↓ 5. 图片处理（封面下载 / 插图占位 / 外链替换）
  ↓ 6. 文本清洗通则（编码/空行/实体/广告段落）
  ↓ 7. 输出章节结构体（见第 8 节）
  ↓ 8. 质量校验（见第 9 节）
```

## 8. 输出结构（章节数据契约）

提供给 CLR-2（阅读器）与 CLD-3（元数据模型）的标准化结构：

```json
{
  "book_id": "string",
  "chapters": [
    {
      "chapter_index": 0,
      "chapter_title": "string",
      "chapter_html": "string（已清洗的 HTML 正文，可直接渲染）",
      "word_count": 1234,
      "footnotes": [
        {
          "id": "fn-1",
          "type": "author_note | translator_note | unknown",
          "anchor": "[^1]",
          "content": "string",
          "raw": false
        }
      ],
      "illustrations": [
        {
          "index": 0,
          "placeholder": "[插图 1]",
          "url": "string",
          "alt": "string"
        }
      ]
    }
  ],
  "cover_url": "string | null",
  "import_format": "epub | txt | html",
  "import_at": "ISO 8601"
}
```

## 9. 质量校验门槛

每本书导入后必须通过以下校验，否则进入人工复核队列：

| 校验项 | 通过条件 |
| --- | --- |
| 章节数 | >= 1（不允许零章节） |
| 正文总字数 | >= 500 字 |
| 章节标题缺失率 | <= 20%（允许部分章节无标题，但不能全无） |
| 广告段落占比 | <= 5%（`skip: true` 段落字数 / 总字数） |
| 脚注孤立锚点 | 锚点有对应 footnote 条目，否则日志警告但不阻断 |
| 图片 404 率 | 下载图片 404 率 <= 10%，超出则整本书标记 `cover_url: null` 并告警 |

## 10. 与其他任务的接口约定

| 下游任务 | 本文输出什么 | 下游接收什么 |
| --- | --- | --- |
| CLD-3（元数据模型定版） | `import_format`、`chapters[].chapter_title`、脚注类型 | 需在元数据模型中为脚注类型、插图索引、导入格式留出字段 |
| CLR-2（阅读器 MVP） | `chapter_html`（可直接渲染）、`word_count`、`footnotes[]` | 阅读器不再自己解析原始文件，直接消费本结构 |
| CLAI-1（AI 伴读入口） | `chapter_html`（纯文字） | AI 服务接收章节纯文本，不含 HTML 标签 |
