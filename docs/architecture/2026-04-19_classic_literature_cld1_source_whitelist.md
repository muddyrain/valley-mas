# 名著 CLD-1 数据源白名单与版权策略（首版）

## 1. 目标

- 目标：先冻结“可持续导入”的首批来源白名单，避免后续阅读器与导入链路返工。
- 用户导向：以中国用户为主，内容供给策略采用“中文来源优先、海外来源补充”。
- 本文定位：产品与工程执行基线，不替代正式法律意见。

## 2. 白名单准入规则（必须同时满足）

- 资源页明确可商用或明确 Public Domain（公版）状态。
- 资源页可追溯到原文来源、作者、版本信息。
- 允许平台侧缓存、清洗、索引与分发（至少用于在线阅读）。
- 能够保留并展示来源声明、许可证与版本说明。

## 3. 首批白名单（中文优先）

| 来源 | 许可证/权利信号 | 可用范围 | 导入策略 | 风险级别 |
| --- | --- | --- | --- | --- |
| 中文维基文库（逐作品审核） | 站点内容 CC BY-SA；作品页有单独版权说明 | 仅导入页面明确可再利用的中文公版作品 | 优先导入中文经典；保留原页面与版本信息 | 中 |
| 国内公开古籍/文献库（逐项） | 条目页明确 CC0/PD/可商用协议 | 仅导入权利说明清晰的单篇作品 | 人工审核后入库，不做整站批量抓取 | 低到中 |
| Project Gutenberg | 站内标注 Public Domain（美国） | 仅导入条目页明确 PD 的作品；优先英文经典 | 先抓元数据，再抓文本；保留来源 URL | 中 |
| Wikisource | 站点内容 CC BY-SA；具体作品需看页面版权说明 | 仅导入页面明确“公版可再利用”的作品 | 仅用可机读版本；保留原页面与版本信息 | 中 |
| 海外政府/高校开放库（逐项） | 条目页明确 CC0/PD/可商用协议 | 仅导入条目页授权清晰的单篇作品 | 人工审核后入库，不做整站批量抓取 | 低到中 |

## 4. 选源优先级（执行顺序）

1. 中文公版来源（优先覆盖中国用户高频阅读内容）。
2. 中文译本公版来源（能明确译本权利状态时）。
3. 海外原文公版来源（用于补齐中文缺失的经典作品）。
4. 条目授权不清晰的来源一律进入 `deny` 或人工复核，不自动导入。

## 5. 暂不纳入（待后续评估）

- 权利状态混杂、同站点作品授权差异过大的聚合站。
- 仅提供“可阅读”但未明确再分发/缓存许可的站点。
- 无稳定来源字段或无法追溯版本来源的文本源。

## 6. 落库最小字段（CLD-1 输出给 CLD-3 的输入）

- `source_name`：来源名称
- `source_url`：作品原始页面
- `license_type`：PD / CC0 / CC BY / CC BY-SA / 其他
- `license_url`：许可证链接
- `rights_note`：条目页权利说明摘录
- `region_scope`：适用地域（如 US / Global / Unknown）
- `import_policy`：allow / conditional / deny
- `reviewed_at`：审核时间
- `reviewed_by`：审核人

## 6. 执行门槛（导入前）

1. 作品级权利校验通过（不是仅站点级判断）。
2. 元数据字段完整率 >= 95%（标题、作者、来源、许可证必填）。
3. 样本抽检 20 篇，权利字段误判率 < 5%。

## 8. 结论与下一步

- `CLD-1` 已冻结首批白名单策略，可支持后续导入链路设计。
- 已明确中国用户优先策略：默认先消费中文来源，再补充海外来源。
- 下一步进入 `CLD-2`：确定 `epub/txt/html` 的标准化解析与清洗流程。
