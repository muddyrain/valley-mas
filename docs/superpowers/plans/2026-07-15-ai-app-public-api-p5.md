# AI 应用公开 API：P5 接入体验计划

## 当前状态

- P4 的服务端与页面能力已实现：API Key 显式绑定应用、每 Key 每日 100 次配额、JSON/SSE 公开调用、仅元数据审计。
- P4 已完成真实验收并提交：创建 Key、绑定已发布应用、JSON 调用、`stream: true` SSE、未绑定 Key 返回 `403`、撤销失效和第 101 次返回 `429` 均已通过。
- 没有外部集成需求时，用户无需创建或配置 API Key；智能体编辑和在线调试不受影响。

## P5 目标

让已完成授权的公开 API 可以被直接复制并接入外部脚本或网站，降低首次调用成本。

## 范围

1. 在智能体编辑页的公开 API 区域增加可复制调用示例。
2. 提供 `curl` 示例：Bearer Key、普通 JSON 请求与响应结构。
3. 提供 JavaScript `fetch` 示例：默认 JSON 和 `stream: true` 的 SSE 读取方式。
4. 示例只使用占位 Key，不展示或保存用户真实 Key。
5. 保持当前安全边界：只允许已发布版本、Key 显式绑定应用、每日 100 次、审计不保存正文。

## 验收标准

1. 用户能在页面复制 JSON 与 SSE 两类示例。
2. 示例中的接口路径、字段名、鉴权方式与后端实现一致。
3. 真实 Key 仅在创建时显示一次；示例和调用记录中均不泄露 Key 或外部正文。
4. 不影响现有智能体编辑、发布和在线调试流程。

## 不在 P5 范围内

- 调整每日 100 次配额策略。
- 增加团队级 Key、计费、账单或 Webhook。
- 保存外部请求或回复正文。
