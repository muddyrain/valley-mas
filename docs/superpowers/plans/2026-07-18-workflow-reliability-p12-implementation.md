### 2026-07-18 第一段落地记录

- 工作流运行与节点运行新增 `cancelled` 状态。浏览器主动停止会立即把本地会话和运行详情标记为“已取消”；服务端收到请求上下文取消后，会把当前运行持久化为 `cancelled` 并保留 `WORKFLOW_CANCELLED` 安全错误码，而不是写成泛化失败。
- 历史抽屉保留原有版本时间线，并新增“最近运行”：按 owner 读取最近 20 次运行；选择任一次后只读展示节点类型、状态、输入/输出安全摘要、耗时和错误码，不覆盖当前草稿。
- 运行面板与节点详情共用错误建议映射：模型配置、超时、上游失败、解析失败、变量引用和取消分别显示下一步动作，保留稳定错误码供 trace 与 AI 修复链路使用。
- 本段不新增独立运行取消接口或跨请求运行注册表；现有 SSE 请求的 AbortSignal 已能向执行上下文传递取消。服务端 `POST /runs/:runId/cancel`、副作用重试二次确认、断线重连事件序列、版本/状态筛选属于下一段。

### 2026-07-18 第二段落地记录

- 大模型节点现在支持“本节点输入”别名：输入绑定仍保存为可追溯的上游引用，系统提示词和用户提示词可引用 `{{输入名}}`；服务端仅在同一大模型节点的提示词中解析该别名，并拒绝输入自引用。
- 变量编辑器将未知或手动粘贴的 `{{...}}` 保持为可编辑文本；选择候选变量会替换当前引用范围而非追加。候选浮层改用动态锚点，随右侧面板滚动保持在输入位置下方，并在大模型节点中同时展示本节点输入和上游输出。
- 自动保存继续持久化当前工作流草稿，但不再创建历史版本；首次创建保留一份初始快照，此后只有“立即保存”或发布会创建不可变版本快照。读取详情、试运行与恢复版本不再隐式制造历史记录。
- 已取消节点保留稳定 `WORKFLOW_CANCELLED` 错误码供 trace 使用，但客户端详情仅展示中性“已取消”状态，不再作为红色执行错误呈现。

### 2026-07-19 有序事件与 trace 基础落地

- 新增不可变的 `workflow_run_events` 记录：每个事件在单次运行内拥有连续 `sequence`，仅保存与 SSE 相同的安全输入/输出摘要、节点/能力标识、状态、错误码和耗时；旧运行没有事件时仍可继续读取节点摘要。
- 节点状态持久化与事件写入使用同一数据库事务；SSE 为每个已持久化事件同时发送 `id` 和 `sequence`，历史详情接口返回按序事件，前端在已有运行详情下展示只读事件顺序。
- 本切片保持现有启动接口兼容；断线重连由后续独立事件订阅承接，不能仅依赖浏览器重新连接伪造恢复。

### 2026-07-19 服务端取消控制器落地

- 当前服务进程维护活动运行的受控 context；取消接口仅允许 owner 操作仍在运行中的记录，并将运行先标记为 `cancelling`，再由执行器的取消事件写入最终 `cancelled` 状态。
- 编辑器停止按钮不再主动中断浏览器 SSE 或立即伪造“已取消”，而是请求服务端取消并等待终态事件；浏览器断开不会再自动取消服务端执行。
- 该控制器仅覆盖当前服务进程。断线重连和跨进程接管仍需下一切片提供独立事件订阅与运行调度，不将其误报为已完成。

### 2026-07-19 断线续接事件订阅落地

- 新增 owner 私有的运行事件 SSE 订阅接口，按 `Last-Event-ID` 或 `after` 序号只回放缺失事件，并在运行进入终态后关闭连接。
- 初始运行流在已获得 `runId` 后意外断开，前端会从最后一个已处理序号续订；不会重复应用已处理事件，也不会直接显示“连接已关闭”。
- 当前续接能力覆盖同一服务进程中的运行和已持久化事件。跨进程调度、重启恢复与无限重试不属于本切片，仍需运行调度层与保留策略共同设计。

### 2026-07-19 安全重新运行落地

- 历史详情仅对已结束运行提供“重新运行”。服务端使用来源运行的不可变图快照创建新 run，并记录 `sourceRunId`；当前编辑草稿不会被历史 trace 覆盖。
- 运行历史只保留脱敏输入摘要，因此重新运行始终要求重新填写文本和上传文件，不会静默重放旧输入。来源运行关联的应用版本会优先用于执行，缺少旧版本记录时才回退到当前草稿版本。
- 图中包含 `write` 或 `model_and_storage` capability 时，前端先展示副作用确认，服务端同时强制 `X-Workflow-Retry-Confirmed: true` 才会创建新运行；任何客户端绕过弹窗的请求仍会被拒绝。

### 2026-07-19 结构化 LLM 输出第一段落地

- 大模型节点保留默认文本输出；可选切换为结构化 JSON，并声明每个字段的字符串、字符串数组、对象、数字或布尔类型。下游变量选择器、End 输出映射和服务端 GraphValidator 共用该字段契约。
- 结构化模式通过受控提示词要求模型仅返回 JSON 对象；运行时会校验 JSON、字段存在性和字段类型。解析失败返回稳定 `LLM_STRUCTURED_OUTPUT_INVALID` 节点错误，不会悄悄回退为普通文本。
- 文件引用和任意元素数组的 LLM 生成尚未开放：模型节点不能伪造文件引用；现有 Start/Tool/End 的文件类型契约保持不变。

### 2026-07-19 子工作流输出契约落地

- 新建子工作流节点会从锁定的已发布版本读取 Start 输入和 End 的已声明类型输出，不再从当前草稿推断。已声明输出会直接进入画布、变量选择器和 End 映射的字段列表。
- GraphValidator 使用该契约校验父工作流到子工作流的输入绑定，以及子工作流输出到下游节点的类型。保存、运行和重试时服务端会将节点内的契约与已发布版本逐项比对，防止客户端伪造字段或类型。
- 历史子工作流节点若未保存契约仍保持可运行，但不会暴露未知类型字段；重新选择已发布版本即可补齐类型化输入和输出。
- 编辑器顶栏保留“立即保存”并新增可发现的“发布”入口：每次保存后都会刷新草稿与发布版本状态；已发布工作流后续保存只推进草稿版本，不会悄悄替换已发布版本；发布前会先保存当前画布，再显式锁定该草稿版本；没有新草稿时显示“已发布”。历史抽屉中的发布操作复用同一链路。
- 每个已发布版本都记录不可变发布标记；父工作流可继续锁定此前发布的子工作流版本。新增发布不会使既有父流程失效；旧数据按当时发布指针及此前版本回填兼容标记。
- 子工作流节点属性面板显示人可读的 `vN` 锁定状态，并提供已发布版本选择。切换前会检查输入/输出契约；兼容版本可直接更新，存在字段移除或类型变化时先列出受影响映射，再由用户确认更新，避免删除并重新添加节点。

### 2026-07-19 AI 协作取消第一段落地

- 每次 AI 协作规划现在创建 owner 私有的请求记录，并在运行、取消中、完成、失败和已取消之间持久化终态与稳定错误码；提示词和模型回复仍只保留在既有消息记录中。
- 停止按钮改为请求服务端取消。服务端使用独立受控 context 执行规划，浏览器断开不会隐式停止规划；只有 owner 能取消当前进程内仍在运行的请求，取消后前端等待 SSE 的已取消终态。
- 本段尚未实现 AI 协作 SSE 的断线续接或跨进程取消接管；请求终态已可持久化，为下一段查询与续接接口提供基础。

## 4. 可取消、可诊断的 AI 协作

影响范围：`server/internal/handler/ai_workbench_copilot.go`、模型迁移、`apps/web/src/api/workbenchCopilot.ts`、`WorkbenchCopilot.tsx`。

1. 引入协作请求 ID 和可持久化的处理状态；历史会话保留用户消息、助手结果和失败类别。
2. 服务端将读取、规划、校验、完成、失败、取消映射为真实 SSE 阶段；定时心跳不能伪造完成。
3. 实现取消接口并将 context cancellation 传入 ARK 调用和受控 planning loop。
4. 建立错误映射：配置缺失 `503`、上游/解析失败 `502`、无效请求 `400`、草稿冲突 `409`。
5. 失败运行上下文只注入安全摘要；AI 只能给出澄清或 operations，不可直接运行或保存。
6. 前端停止按钮等待终态，失败消息保留重试入口与 request ID；提案状态不因网络断开误标成功。

验收：fake ARK 下所有失败分支、浏览器取消和重连均有确定终态；真实 ARK 至少验证一次超时或取消。
