# P13.1：提示词库设计

## 状态

已完成；本期已收敛为轻量提示词库，并于 2026-07-19 完成页面人工验收。

## 目标

让用户保存和复用常用提示词正文。提示词库是个人词库，不是独立可运行资产，因此不引入草稿、发布、版本、变量契约或试验台。

## 首期范围

1. 在“AI 资源 / 提示词”提供 owner 私有的提示词库。
2. 每条提示词只有名称、简短描述和正文；可以创建、编辑、归档。
3. 大模型节点的“提示词库”入口打开选择弹窗，展示列表和正文预览。
4. 用户确认“插入提示词”后，将正文追加到当前节点系统提示词末尾；插入后是节点内的普通文本，不与词库保持绑定。
5. AI 优化沿用现有“先给建议、用户确认后写入”的模式，只优化提示词正文。

## 关键决策

- 提示词库不声明输入或输出。需要引用工作流变量时，用户在节点的系统提示词或用户提示词中自行插入变量。
- 修改或归档词库条目不会改写任何既有节点，因为节点保存的是插入后的文本。
- 不提供词库内试运行；实际效果通过承载它的大模型节点试运行验证。
- 资源仍按当前登录用户隔离，归档条目不在默认候选中显示。

## 数据与接口边界

`AIPrompt` 只保存 `name`、`description`、`content`、owner 与归档时间。

首期接口：

- `GET/POST /ai/prompts`
- `GET/PATCH/DELETE /ai/prompts/:promptId`

所有读取与写入均按当前登录用户过滤。

## 明确不做

- 草稿、发布、版本、版本差异或工作流锁定引用。
- 输入/输出变量契约、词库专属试验台和词库自动执行。
- 资源共享、团队协作、公开 Prompt 市场和跨 owner 引用。
- 自动更新已插入的节点文本。

## 验收

1. 用户可创建、编辑、归档自己的提示词；其他用户无法读取或修改。
2. 模型节点弹窗可搜索、预览并插入词库正文；插入后正文出现在系统提示词中。
3. 编辑或归档原词库后，已经插入过的节点文本不发生变化。
4. AI 优化只在用户确认后替换词库正文，不自动保存。
