# Web AI 工作台重设计

## 背景与目标

当前 AI 工作台已具备智能体、工作流、知识库、版本、发布和运行调试能力，但编辑器将多个任务并置在缺少层级的卡片与面板中。此次调整提升工作流编排和智能体调试的可读性与操作聚焦度，使其接近成熟 AI 创作工具的效率，同时保持 Valley 已有能力和 shadcn 组件体系。

目标：

- 让工作流画布成为工作流页的明确视觉与操作中心。
- 让节点配置按用户任务分段，减少长表单和无关信息的干扰。
- 让智能体编辑、知识库与工具绑定、在线调试和版本发布各自有稳定入口。
- 让运行状态与结果贴近产生它们的画布或调试区。

非目标：

- 不修改既有 URL、后端接口、数据模型、节点运行策略或 SSE 协议。
- 不引入新的第三方依赖。
- 不仿制 Coze 或 Dify 的品牌、文案、图标或专有交互，只借鉴其信息层级与工作流模式。

## 设计方向

采用“自适应工作台”方向：工作流页以三栏画布为桌面端基线，智能体页以配置主区和在线调试侧栏为基线。侧栏可折叠或在窄屏降级为抽屉式面板，避免画布和配置区互相挤压。

视觉基线使用当前 shadcn 语义 token、默认圆角和边框体系。状态颜色只表达语义：主色用于当前选择与主要操作，成功、警告和失败仅用于运行状态或校验状态。节点类别用低频、低饱和图标底色区分，不使用独立页面色系或装饰性渐变。

## 页面设计

### AI 工作台首页

- 顶部改为轻量工作区头部：页面标题、智能体与工作流数量概览、知识库入口、主要创建操作。
- 智能体和工作流以独立区块展示，区块内优先展示最近编辑、状态和最近运行，而不是大段产品介绍或统计卡片。
- 模板保留为快速启动入口，未开放模板保持明确禁用状态。
- 删除不具备下一步操作价值的固定统计卡片，避免首页成为信息重复的长页面。

### 智能体编辑页

- 顶部固定展示返回、智能体名称、保存状态、版本入口、保存草稿与发布动作。
- 主区改为标签式编辑：`编排`、`知识库`、`工具`、`发布`。编排放置名称、说明、系统提示词和开场白。
- 知识库与工具绑定从编排表单中抽离，独立显示已绑定条目、空状态和跳转管理入口。
- 右侧保留在线调试。输入、流式响应、引用资料、最近运行依照时间与任务关系排序，运行时可停止，非运行时显示最近可查看结果。
- API Key 与公共调用记录移入“发布”标签，避免干扰私有智能体的创建与调试任务。
- 版本历史移到顶部入口触发的抽屉或对话框，不占用编辑页末尾的大块垂直空间。

### 工作流编辑页

- 顶部工具栏包含返回、可编辑工作流名称、保存状态、撤销与重做、版本、资料库、运行记录、试运行、保存和发布。
- 左侧“节点”面板提供搜索和常用节点，其他节点按类别折叠分组。拖拽与点击添加保持当前能力。
- 中央画布保持 React Flow。缩放、适配画布、小地图等控制归入画布悬浮控制区，节点运行状态在节点自身和底部运行抽屉中共同呈现。
- 右侧为选中节点的“配置”面板。配置以节点名称、输入输出、节点专属参数和高级设置的分段或标签承载；未选中节点时展示可执行的快捷提示，而非空白大面板。
- 试运行结果采用可展开的底部面板。面板显示输入、整体状态、节点时间线和最终输出，不遮挡画布默认操作。
- 右侧面板在窄屏收起为抽屉，左侧节点库同样可按需展开。移动端不承诺完整拖拽编辑，提供节点选择与配置的可用回退。

### 节点视觉与属性表单

- 节点使用统一尺寸、边框、标题行、配置摘要和连接点。选择、未配置、运行中、成功、失败与计划中状态以统一语义 token 表达。
- 连接点保持当前 React Flow 句柄模型，条件分支仍使用明确的真/假输出。
- 属性表单使用一致的段落标题、字段说明、校验反馈和可折叠高级设置。现有节点表单只调整容器结构与视觉，不改变字段名称、类型、校验和保存数据结构。
- 各节点表单的加载、禁用、空状态和错误提示复用现有 shadcn 组件和 toast 行为。

## 交互与状态

- 保存、发布、运行、停止、恢复版本、绑定知识库和工具仍调用现有 API 函数。
- 保存状态在顶部常驻反馈，保存失败使用现有 toast 并保留用户编辑内容。
- 节点选中同步更新右侧面板；清空选择后关闭配置内容但保留可发现的面板入口。
- 运行结束后节点状态和底部结果面板更新，运行记录保持可回看。
- 所有动态布局以 150-250ms 的 transform 或 opacity 过渡呈现，并尊重减少动态效果偏好。

## 组件边界

- `WorkflowEditorPage` 负责页面状态、工具栏、画布、右侧配置区和运行抽屉编排。
- `NodePanel` 负责节点检索、分类和添加入口。
- `WorkflowNode` 负责节点视觉、摘要与节点级运行状态。
- `PropertyPanel` 和各 `properties/*PropertyForm` 负责统一配置框架与节点专属字段。
- `AIAppEditor` 拆分为编辑标签区、在线调试区、发布区和版本入口，以避免单文件继续膨胀。

## 验证与验收

- 类型检查通过：`pnpm --filter @valley/web exec tsc --noEmit`。
- 样式与静态规则通过：`pnpm --filter @valley/web check`。
- Harness 检查通过：`pnpm check:harness`。
- 浏览器验证工作台、智能体编辑和工作流编辑的桌面与窄屏视图，检查节点添加、节点选择、属性更新、保存、试运行、停止、版本入口和知识库绑定入口。
- 在浅色与深色主题检查主要文本、按钮、输入控件与运行状态的可读性。

## 风险与迁移

- 最大风险是大幅拆分编辑器组件时影响现有运行会话、快捷键和面板宽度状态。实施时保留现有状态和 API 调用路径，先完成结构抽取再替换视觉。
- 工作流画布属于高交互表面，必须以本地运行时证据验证，不能只依据静态检查。
- 本次不改变长期功能状态、路由、接口、依赖或数据模型，因此不需要同步长期计划文档。
