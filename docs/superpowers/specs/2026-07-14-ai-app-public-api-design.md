# AI 应用公共 API P4 设计

## 目标

让已发布的 AI 应用可被外部系统通过受限 API Key 调用，并提供每日配额和不保存内容的调用观测。

## 范围

- 一个 API Key 仅能调用 owner 显式绑定的已发布应用。
- 每个 Key 每个自然日最多 100 次调用。
- `POST /api/v1/public/ai/apps/:appId/chat` 默认返回 JSON；请求体 `stream:true` 时返回 SSE。
- 公共调用固定运行应用的 `published_version_id`，不读取草稿。
- 工作台可创建 Key、绑定应用、查看今日已用/剩余和调用元数据。

## 不在本期范围

- 不开放写入工具、草稿创建或自动发布。
- 不保存外部请求正文、模型回复正文、工具参数或工具结果。
- 不引入 Redis、计费、按分钟限流或跨 owner 共享应用。

## 数据与安全边界

新增三类持久化数据：

- Key-应用绑定：`api_key_id`、`app_id`，唯一约束保证一对绑定只存在一次。
- 每日计数：`api_key_id`、`usage_date`、`request_count`，唯一约束保证一次原子计数；每个 Key 的上限固定为 100。
- 公共调用元数据：owner、应用、发布版本、Key、状态、耗时、是否流式、错误码、创建时间。

公共调用先校验 `Authorization: Bearer valley_...`，再校验 Key 状态、应用归属绑定和发布状态。验证通过后，在同一数据库事务内递增每日计数；计数已达上限时不执行模型调用并返回 `429`。调用日志只在请求结束时写入元数据，不写入输入或输出。

## 接口契约

### 公共调用

`POST /api/v1/public/ai/apps/:appId/chat`

请求：

```json
{"message":"你好","stream":false}
```

JSON 成功响应返回 `reply`、`model`、`versionId`、`requestId` 和剩余次数。`stream:true` 返回 `text/event-stream`；事件使用 `meta`、`delta`、`tool_call`、`tool_result`、`done`、`error`，且 `done` 仅返回 `requestId`、模型、版本和剩余次数，不返回工具原始结果。

失败响应：

- `400`：消息为空或请求无效。
- `401`：Key 缺失、格式错误、无效或已撤销。
- `403`：Key 未绑定该应用，或应用未发布。
- `429`：当天 100 次配额已用完。
- `502`：ARK 或 Agent 执行失败。

### 工作台管理与观测

保留现有 Key 创建与撤销接口，新增 Key 绑定替换接口、Key 今日用量读取接口，以及按应用或 Key 筛选的元数据列表接口。所有管理与观测接口继续要求 owner 的 JWT。

## 页面行为

AI 应用编辑页增加 API 调用区域：选择 Key 绑定当前已发布应用，展示每日 `已用 / 100` 与剩余次数，并显示最近公共调用的时间、Key、状态、耗时和流式标记。页面不展示外部请求或回复内容。

## 验收

- 未绑定、已撤销或跨 owner 的 Key 无法调用应用。
- 绑定 Key 只能运行当前已发布版本；草稿修改不改变公共调用结果。
- 第 100 次调用成功，第 101 次返回 `429`，并且并发请求不会越过上限。
- JSON 与 SSE 均能返回成功、上游失败和配额失败的明确状态。
- 观测数据不包含请求正文、回复正文或工具原始结果。
