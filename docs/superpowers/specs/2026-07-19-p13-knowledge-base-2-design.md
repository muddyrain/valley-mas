# P13.2：知识库 2.0 设计

## 状态

实施中。现有工作流已经冻结知识库绑定与检索策略，并在运行结果中保存安全引用；文档可观察性与检索试验台已完成，下一步验证版本可见性。

## 目标

让用户能确认每份资料从何而来、处理到了哪个阶段、被切成了哪些可检索片段；预览只能看到自己授权资料的安全文本摘要，不返回原文件、向量或完整检索上下文。

## 本阶段范围

1. 文档列表明确展示处理状态、进度、类型、大小和更新时间。
2. 文档详情展示来源、状态、分段数与可折叠的片段预览。
3. 片段接口按知识库、文档和 owner 三重校验；内容仅返回长度受限的预览文本。
4. 不改变现有上传、嵌入、工作流检索和版本快照语义。

## 已完成：检索试验台

1. owner 可在单个知识库中手动输入查询，复用既有默认检索阈值和 pgvector 检索路径。
2. 结果只返回命中文档名、受限片段摘要、分数和安全片段 ID；不会返回全文、向量或原始文件。
3. PostgreSQL、pgvector、向量模型或索引配置不可用时，显示现有稳定的可诊断错误，不伪造结果。

## 后续阶段

1. 版本可见性：在绑定界面明确显示已冻结的知识库修订与检索策略。
2. 在试验台验收后，才评估重排、混合检索和多模态摄取。

## 验收

1. owner 可打开自己文档的详情，看到来源、状态、时间和片段预览。
2. 非 owner 无法读取文档片段。
3. 失败文档仍展示失败原因，但不提供不存在的片段。
4. 既有工作流检索、引用和知识库上传流程不受影响。
5. owner 可在试验台看到命中文档、受限摘要与分数；配置不可用时看到可诊断提示。
