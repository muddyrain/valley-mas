# Life Trace 本地透明抠图设计

## 背景

P3.1 / P3.2 的 AI 拍照分析、条码识别、历史商品命中、裁剪封面和库存确认流程已完成收口。剩余后置项是“真正透明背景抠图”：把用户真实拍摄的商品图处理成透明背景 PNG，作为 Pantry `thumbnailUrl` 候选，不覆盖真实图片。

用户已确认首版不走云端抠图服务，优先评估本地第三方库。

## 目标

- 用户在库存编辑或拍照确认链路中，可以手动生成透明背景封面。
- 图片不发送到第三方抠图服务；首版优先在浏览器本地处理。
- 透明抠图结果只写入 `thumbnailUrl`，原始 `imageUrl` 始终保留。
- 生成失败、浏览器不支持或性能不足时，不影响现有裁剪封面和原图封面。
- 不自动对每张上传图片抠图，避免耗时、耗电和误覆盖。

## 非目标

- 不做扫码即入库。
- 不接外部商品库。
- 不把透明抠图结果当作真实商品图片。
- 不承诺所有机型都能本地完成高质量抠图。
- 不在首版引入云端抠图 API 或第三方服务密钥。

## 推荐路径

首版做“浏览器端本地抠图技术验证”，推荐优先评估 `Transformers.js`：

- 依赖更偏通用推理框架，许可证相对友好。
- 图片可留在用户设备内处理。
- 可按需懒加载，避免影响首屏和普通拍照入库。
- 模型文件可以后续改为自托管，减少运行时依赖外部 CDN。

备选路径：

- `@imgly/background-removal`：接入简单，效果目标贴近商品图，但 AGPL-3.0 / 商业授权风险需要先确认。
- `rembg`：MIT，效果和模型选择成熟，但需要 Python / Docker / sidecar，部署明显变重，放在浏览器方案失败后再评估。

## 用户流程

1. 用户已有商品实拍图。
2. 在库存编辑或拍照确认页点击“生成透明封面”。
3. 前端懒加载本地抠图库和模型。
4. 处理成功后生成透明 PNG 预览。
5. 用户保存后，把 PNG 通过现有 `uploadLifeTraceImage` 上传，并写入 Pantry `thumbnailUrl`。
6. 处理失败时显示轻量错误，保留现有封面选择，不阻塞入库或编辑保存。

## 架构

### 前端

- 新增本地抠图 adapter，例如 `src/lib/pantryCutout.ts`。
- adapter 对页面只暴露一个函数：输入 `File | Blob`，输出透明 PNG `File`。
- adapter 内部负责：
  - 能力检测。
  - 懒加载依赖和模型。
  - 超时和取消。
  - 把输出统一转成 `image/png`。
- `PhotoItemAnalysisPage` 和 `PantryItemDrawer` 只负责按钮、状态、预览、上传和保存，不直接耦合具体模型。

### 服务端

- 首版不新增抠图接口。
- 复用现有上传接口保存透明 PNG。
- Pantry 创建 / 更新接口继续只保存 `thumbnailUrl`。

### 数据

- 不新增数据库字段。
- `imageUrl` 保留真实图片。
- `thumbnailUrl` 可保存透明 PNG，也可保存现有裁剪封面或 AI 缩略图。

## 失败兜底

- 浏览器不支持：隐藏或禁用透明封面生成入口，保留原图 / 裁剪封面。
- 模型加载失败：提示稍后重试，不影响保存。
- 处理超时：取消本次处理，保留当前封面。
- 抠图质量差：用户可以不保存生成结果，继续用原图 / 裁剪封面。
- 上传失败：透明 PNG 不写入 Pantry，原有保存流程继续可用。

## 性能策略

- 默认不随页面加载模型。
- 只在用户点击生成时加载。
- 处理期间显示明确的本地生成状态。
- 首版限制输入尺寸，先把图片缩到适合移动端处理的尺寸，再做抠图。
- 生成 PNG 后释放对象 URL，避免内存泄漏。

## 验收标准

- 没有点击生成时，现有拍照分析、裁剪封面、入库和编辑保存行为不变化。
- 点击生成透明封面后，支持设备能生成透明 PNG 预览。
- 保存后 Pantry `thumbnailUrl` 使用透明 PNG，`imageUrl` 仍是原始实拍图。
- 不支持或失败时，用户仍能保存库存。
- 模型和依赖只在点击生成时加载。
- 不出现自动上传到第三方抠图服务的行为。

## 测试计划

- 前端单测：
  - adapter 成功返回 PNG File。
  - adapter 失败时页面不覆盖已有 `thumbnailUrl`。
  - 用户手动选择透明封面后保存 payload 带新 `thumbnailUrl`。
  - 未生成透明封面时原有保存流程不变。
- 类型检查：
  - `pnpm --filter @valley/life-trace exec tsc --noEmit`
- 手动验收：
  - Chrome Android 真机生成透明封面。
  - iPhone / Safari 不支持或性能不足时兜底不阻塞。
  - 拍照确认页和库存编辑弹框各走一遍保存。

## 实施切分

1. 做本地抠图 adapter 技术验证，确认依赖、模型、许可证和产物大小。
2. 若验证通过，再接入库存编辑弹框。
3. 再接入拍照确认页的封面选择。
4. 同步 PLAN，把后置项从“评估中”改成“本地透明抠图首版落地中”。
5. 补测试和真机验收说明。
