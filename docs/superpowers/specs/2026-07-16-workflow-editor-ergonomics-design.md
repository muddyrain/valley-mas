# 工作流模板与编辑器体验设计

## 决策

模板与用户工作流是两类不同的资源：模板是平台维护的只读来源，工作流是用户空间内可编辑、可运行、可版本化的实例。两者不能以“打开即创建”的编辑器路由混用。

```text
工作台模板卡片 → 模板详情（只读、无持久化） → 复制到工作台 → 用户工作流编辑器（持久化）
```

模板详情仅使用前端静态模板定义和 Graph v2 图快照，不创建本地草稿、不写服务端、不出现在工作流列表。点击“复制到工作台”才调用现有 `POST /workflows`，成功后以 `replace` 跳转到 `/workbench/edit?id=<id>`。复制请求进行中禁用按钮，失败时停留详情页并允许重试；一次点击最多创建一个实例。

编辑器只编辑带 `id` 的已持久化工作流。历史 `?template=` 链接重定向到模板详情，避免绕过复制边界；空白“新建工作流”仍走现有创建方式，但不承担模板预览。复制后的名称默认等于模板名称，用户可在编辑器中内联改名。

## 模板详情

工作台模板卡片跳转至 `/workbench/templates/:templateId`。详情页显示名称、标签、能力说明、节点数和只读工作流图预览；不提供节点参数编辑、运行、保存、版本或资料库绑定。未开放模板显示不可复制状态。详情页的返回动作回到工作台，刷新和浏览器前进/后退保持 URL 所表达的模板身份。

模板元数据与图快照维持在独立模块中，调用方只能取得克隆，避免预览或编辑污染来源定义。首期不新增服务端模板表或 API：平台模板当前由前端静态定义，现有工作流创建接口足以形成用户实例。

## 已复制工作流的编辑器

名称、节点、边与节点配置在用户停止编辑约 700ms 后自动保存。保存快照按 revision 串行提交；飞行中的请求完成后如有更新，补发最新快照，不能让旧请求覆盖新编辑。运行前先 flush 最新编辑，保存失败则不发起运行。顶部只显示创建、保存、已保存或失败状态；“立即保存”作为刷新和失败恢复入口，不再是模板运行的前置步骤。

节点保持固定紧凑几何。移除双击、右键和节点内展开式折叠；点击节点始终在右侧面板打开“配置”。右侧面板提供“配置 / 本次运行”标签，运行失败自动定位失败节点并切换至运行详情；底部试运行面板继续负责工作流输入与整体结果。

LLM 系统提示词、LLM 提示词和知识库查询改用行内变量胶囊。底层仍保存 `{{node.output.field}}`；选择器只列当前节点在 DAG 上游的 Graph v2 合法输出，无法解析的历史引用保留错误样式并在运行前提示修复。

## 非目标与验收

- 不新增后端模板数据模型、模板市场服务、模板运行、模板编辑或新的节点协议。
- 不新增 HTTP、代码、分支或循环节点。
- 从模板卡片进入详情不产生服务端或本地持久化工作流；只有复制成功的实例出现在工作流列表。
- 复制后无需手动保存即可运行；连续修改自动保存不丢最后一次编辑。
- 节点高度和连接线不受查看运行详情影响；变量只能从拓扑上游选择。
