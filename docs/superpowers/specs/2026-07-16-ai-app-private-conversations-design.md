# AI App 私有会话与受控工具设计（P8）

> 状态：实施中（自动化验证完成，待真实浏览器与 ARK 验收）
> 日期：2026-07-16
> 范围：`apps/web` 工作台、`server` AI App 私有运行时

## 目标

在 AI 工作台的智能体应用中提供 owner 私有、可回看的多轮会话。会话固定到创建时的 AI App 草稿版本；每一轮可继续使用该版本快照的知识库和受控工具，并以安全摘要展示工具状态。

这项能力不改变公开 API：`/public/ai/apps/:appId/chat` 仍保持无状态调用，不读取、创建或暴露私有会话。

## 已有能力与收敛决策

- `AIApp` 已提供不可变版本、私有调试、知识库版本快照、`content.search` 受控只读工具及 `AIAppRun` 运行摘要。
- 旧 `/ai/agents/:agentId/conversations/*` 已有服务端会话模型，但未接入 Web 工作台，且不能复用 AI App 的版本与工具绑定。
- P8 不扩展或迁移旧 `AIAgent` 会话；新入口只建立在 `AIApp(type=agent)` 上，避免两套工作台会话产品和权限模型长期并存。

## 资源与版本边界

```text
AI App（owner）
  └─ AI App Version（不可变）
       ├─ 知识库快照（已有）
       └─ 工具快照（P8 新增）
            └─ 私有会话（创建时锁定该版本）
                 ├─ 用户/助手消息
                 ├─ AI App Run（安全输入输出摘要）
                 └─ 工具轨迹（名称、状态、耗时；无参数和原始结果）
```

- 新会话默认绑定当前草稿版本；后续保存、改绑知识库或工具不会改变既有会话。新建会话才使用新的草稿版本。
- 首版只开放已实现的 `content.search`。目录中列出的写入工具仍不可绑定、不可执行。
- 工具名、执行成功/失败和耗时可以被会话参与者看到；工具参数、原始返回值、系统提示词、文件原文和密钥均不持久化到会话或轨迹。
- 会话、消息、运行和轨迹均以 `user_id + app_id` 做服务端 owner 校验。前端路由和隐藏按钮不承担权限控制。

## API 与流式契约

仅在认证分组增加以下私有 API：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/ai/apps/:appId/conversations` | 列出当前 owner 的会话摘要。 |
| `POST` | `/ai/apps/:appId/conversations` | 以当前草稿版本创建会话。 |
| `GET` | `/ai/apps/:appId/conversations/:conversationId` | 返回会话、消息和安全工具轨迹。 |
| `DELETE` | `/ai/apps/:appId/conversations/:conversationId` | 软删除会话及其消息、运行、轨迹。 |
| `POST` | `/ai/apps/:appId/conversations/:conversationId/chat` | 提交一条消息，支持 JSON 或 SSE。 |

聊天 SSE 延用现有 `delta` / `tool_call` / `tool_result` / `done` / `error` 事件类型，并在 `meta` 中返回会话与版本标识。`tool_call` 和 `tool_result` 只包含工具名、成功状态与耗时。完成事件返回已持久化的用户消息、助手消息、运行摘要与引用摘要。

## 运行与持久化规则

- 取最近 24 条已完成消息组成多轮上下文；系统提示词、RAG 上下文和当前用户消息分别构造，防止用户消息伪造系统身份。
- 先持久化有效的用户消息，再执行 ARK。成功后持久化助手消息；配置缺失、检索失败、上游失败和取消只保留用户消息与失败运行摘要，不伪造助手回答。
- 会话标题由首条用户消息截断生成，用户可在首版重命名；标题生成不再额外调用模型。
- 每次运行使用会话的 `version_id`，而不是当前 `draft_version_id`；知识库检索、工具白名单、运行模型和引用均从该版本快照读取。
- 运行取消沿用请求 context；SSE 断开写入 `cancelled` 运行摘要，不创建助手消息。

## Web 体验

- 在 AI App 编辑页增加“私有会话”入口，进入 `/workbench/apps/:appId/conversations/:conversationId`；编辑配置与会话运行分离，避免会话被未保存草稿静默影响。
- 会话页使用现有 shadcn `Sidebar`、`ScrollArea`、`Textarea`、`Button`、`Skeleton` 和 `Badge`：左侧列表支持新建、切换和删除，右侧展示消息、引用与工具状态。
- 提交时立即显示用户消息和助手占位；SSE 增量更新助手内容，断开、失败和取消应能明确恢复输入状态，不把失败文本写成历史回答。
- 路由刷新、前进和后退以 URL 中的 app/conversation 身份为准；跨会话迟到 SSE 事件不得回写当前页面。

## 非目标

- 不新增公开会话、团队共享、跨 owner 协作、记忆摘要、用户自带模型 Key、任意 HTTP、代码执行、Webhook、循环或写入型工具。
- 不迁移或删除旧 `/ai/agents/*` 兼容接口；它们不出现在 Web 工作台新入口。
- 不在 P8 首版新增模型、第三方依赖或向量检索策略。

## 验收标准

1. 仅 owner 可创建、读取、聊天和删除自己的 AI App 会话；跨用户和非 agent 应用均被拒绝。
2. 会话固定版本：修改应用的知识库或工具绑定后，旧会话仍按旧版本运行；新会话按新草稿版本运行。
3. 已绑定 `content.search` 的会话可在 SSE 和历史中看到安全工具状态，不泄露参数和原始工具结果。
4. 多轮聊天刷新后仍可恢复消息、引用和运行/工具摘要；配置缺失、模型失败、取消均不产生伪造助手消息。
5. Web 新建、切换、删除、流式回答、停止和错误恢复在真实浏览器中验证；静态检查、服务端测试和 Harness 检查通过。
