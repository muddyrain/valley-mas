# 博客摘要模型目录迁移设计

## 状态

已确认，实施中。

## 目标

博客编辑器的“AI 截取摘要”由操作者从模型目录选择一个已启用的文本模型，不再读取 `ARK_TEXT_MODEL`。服务端只接受模型目录 ID，并记录实际 Provider、模型、token 和延迟。

## 决策

- 选择权属于当前博客编辑器操作者；不新增管理员场景策略或默认模型。
- 选择器复用工作流已有的模型选择体验，展示目录中已启用且具备 `text` 能力的模型。
- 服务端以目录 ID 再校验模型已启用且具备 `text` 能力；浏览器传递 Provider 或原始模型名均不生效。
- 本切片仅覆盖博客编辑器的单次摘要生成。批量 Markdown 导入工作流继续使用既有 Legacy 路径，后续单独迁移。

## 接口与运行时

`POST /admin/blog/ai/excerpt` 请求增加必填 `modelId`。响应保留摘要与实际模型，并新增实际 Provider。

服务端解析所选模型的 Provider 凭据，通过 OpenAI-compatible 客户端调用；上游返回空内容、不可用模型、Provider 未配置和调用失败均保持可理解的 4xx/5xx 结果。每次成功或失败调用写入 `blog-ai-excerpt` 审计记录，包含 Provider、模型、token、延迟与调用用户。

## 验收

1. 博客编辑器只展示已启用的文本模型，未选模型时不能请求摘要生成。
2. 请求携带目录模型 ID；停用、未知或非文本模型均被服务端拒绝。
3. SiliconFlow 或 Amux 的真实摘要调用返回内容、实际 Provider/模型，并生成完整用量审计记录。
4. 批量 Markdown 导入工作流和现有封面生成功能行为不变。
