# 工作流循环节点设计规格

## 状态

已确认产品方向，核心链路实施中。目标为对齐 Coze 的复合循环能力：真实循环体画布、数组/次数/无限三种模式、中间变量、循环控制和聚合输出。

## 问题与目标

当前工作流 Graph v4 是单层 DAG，服务端明确拒绝普通图中的回边。循环不能通过解除这一限制实现：任意回边会让拓扑校验、变量可达性、运行事件与取消语义失效。

循环应作为一个复合节点实现：外层仍是 DAG；循环节点在运行时多次执行其受控的循环体子图。画布上，循环节点与循环体同时出现，循环体不是侧栏中的简化编辑器，而是主画布内带入口/出口的容器。

## 交互与画布模型

1. 从节点选择器添加“循环”时，创建一个外层 `loop` 节点和其唯一关联的循环体容器。
2. 循环体有专属左入口、右出口和顶部关联端口；从入口端口拖线或 hover 添加按钮时，节点选择器在循环体内创建节点。
3. 可把普通节点拖入循环体。拖入后其位置转换为容器局部坐标；循环体外与循环体内的普通节点不得直接连线。拖出恢复外层节点属于后续交互补齐项。
4. 循环体内使用独立的 DAG 校验，入口可到达所有内部节点，所有可执行路径都要抵达出口。循环体外仍把 `loop` 当作一个普通单入/单出节点。
5. 选中循环节点显示循环配置；选中循环体或内部节点时，右栏显示循环体上下文、局部变量和节点配置。循环卡摘要显示模式、输入、中间变量、输出；循环体卡显示内部节点数量与校验状态。
6. 连线 hover、插入、拖拽、缩放和 MiniMap 复用现有高性能交互策略。循环体不监听全局 pointermove，不在主画布拖动时重建内部节点数组。

## 图数据合约

外层图继续使用 Graph v4，保证既有工作流不迁移、不失效。`loop` 是新增的外层节点类型，其 `config` 包含 `body` 子图；循环体显示节点和内部节点只从该配置投影到 React Flow，不能作为外层 `Graph.nodes` 重复持久化。

```json
{
  "mode": "array",
  "input": { "name": "items", "type": "string[]", "value": "{{start.output.items}}" },
  "middleVariables": [
    { "name": "previous", "type": "string", "initialValue": "" }
  ],
  "outputs": [
    { "name": "results", "type": "string[]", "source": "{{expand.output.text}}" }
  ],
  "body": {
    "nodes": [],
    "edges": []
  }
}
```

- `mode: array`：输入必须为数组，`item`、`index` 为本轮只读局部变量。
- `mode: count`：次数为固定整数或上游 number，范围为 1–1000；暴露 `index`。
- `mode: infinite`：每轮从 0 开始递增 `index`，只能通过循环体中的“终止循环”结束；服务端仍有硬上限，避免无界消耗。
- `middleVariables`：在循环开始前初始化；循环体可由“设置循环变量”更新，更新值仅对后续轮次生效。
- `outputs`：从循环体节点输出映射，按轮次聚合为数组；被提前终止时仅保留已完成轮次。
- `body`：内部节点沿用既有 `Node` / `Edge` 数据结构，但不能含外层 `start`、`end`；入口和出口由容器隐式提供。

## 循环体内支持的能力

循环体开放当前工作流已经开放的普通能力：LLM、Tool、Condition、Switch、Merge、Variable、Subworkflow、Intent，以及另一个循环节点。嵌套循环允许存在，但最大嵌套深度为 3。

新增仅允许在循环体内创建的控制节点：

- `set_loop_variable`：把本轮可达节点输出或常量写入已声明的中间变量。
- `continue_loop`：结束当前轮，跳过本轮剩余可执行路径。
- `terminate_loop`：结束整个循环；仅无限循环必需，数组/次数循环中也可提前结束。

工具能力、模型调用、子工作流和写入型能力沿用外层的权限、确认、预算和审计策略；循环不绕过任何安全边界。运行前要按“单轮最坏成本 × 最大轮数”检查预算，拒绝无法由运行时预算约束的图。

## 运行语义

1. 解析循环输入、初始化中间变量并校验模式限制。
2. 每轮建立只读局部作用域 `item`、`index` 与中间变量快照；外层可达上游变量在循环体内只读可用。
3. 按内部 DAG 执行，处理条件/选择器的跳过传播；遇到 continue 或 terminate 时按控制语义收束本轮。
4. 轮次成功后提交本轮中间变量更新、提取输出映射并追加聚合结果；失败时默认停止循环，并保留失败轮次、内部节点与错误码。
5. 外层 `loop` 输出聚合结果后，继续外层 DAG。

运行事件新增 `loopIteration`、`loopDepth`、`bodyNodeId` 字段。编辑器将循环节点展示为汇总状态，展开后按轮次查看内部节点状态、输入、输出、中间变量变更和失败位置。

## 校验与限制

- 外层图仍禁止普通回边；循环体只允许通过 `loop` 配置中的 `body` 表达重复。
- 循环体节点 ID 在本循环内唯一；内部引用必须来自同一循环体的上游节点、循环局部变量或外层可达变量。
- 循环体内禁止直接引用外层下游节点，也禁止向外层普通节点拉线。
- 每个循环体最多 30 个节点，循环深度最多 3，数组/次数循环最大 1000 轮；无限循环必须同时配置最大轮次与总体运行时截止限制。
- 中间变量名在当前循环唯一，类型不可变；`set_loop_variable` 的值必须类型匹配。
- 聚合输出源在每一条完成路径上必须已声明且可用；输出类型为元素类型对应的数组类型。
- 循环体中的 `start`、`end`、裸 `merge` 缺失候选、未连接控制节点和跨作用域变量引用一律拒绝。

## 非目标

- 不开放任意回边、用户脚本、HTTP、SQL 或不受控外部凭据。
- 不把循环体拆成独立可发布工作流；需要复用时仍使用现有 Subworkflow。
- 不承诺并行迭代。第一版严格串行，保证中间变量、写入顺序和 trace 可解释；未来可在不含中间变量/写入的数组循环中增加受控并发。

## 验收标准

1. 用户可在主画布创建循环和循环体，拖入/新增内部节点并完成内部连线；跨边界连线得到明确拒绝。
2. 数组、次数、无限三种模式均可运行；无限循环可由终止节点结束，且硬限制可阻止失控执行。
3. `item`、`index`、中间变量、外层上游变量在正确作用域可选；错误引用在保存和运行前被定位。
4. 循环体普通节点、条件分支、循环控制、嵌套循环、写入能力确认、取消、失败恢复和聚合输出均有服务端测试。
5. 运行记录可定位到“第几轮 / 哪个循环体内部节点”；既有非循环 Graph v4 保存、运行、历史和画布性能不回归。
