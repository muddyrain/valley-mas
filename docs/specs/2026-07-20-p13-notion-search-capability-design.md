# P13.3.1：Notion 只读搜索能力设计

## 状态

已实现并由用户在真实 Notion 工作区验收。OAuth 连接管理保持不变；`notion.search` 已注册为 Graph v4 的只读工具能力。

## 背景

用户已可在“工具”页连接自己的 Notion 工作区。下一步需要让工作流在明确、受控的配置下读取该连接可访问的内容，而不是开放任意 HTTP 请求或写入操作。

## 目标

在现有 Graph v4 的“工具”节点中提供白名单能力 `notion.search`：

1. 仅使用当前工作流 owner 的 Notion OAuth 连接。
2. 按用户显式提供的查询词搜索当前连接可访问的 Notion 页面或数据源。
3. 向下游节点输出受限的结果元信息，供条件判断或大模型后续处理。
4. 全程只读，不创建、更新、归档或分享 Notion 内容。

## 能力契约

### 输入

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `query` | string | 必填，去除首尾空白后长度为 1–200。 |
| `limit` | integer | 可选，默认 5，范围 1–10。 |

### 输出

```json
{
  "results": [
    {
      "id": "notion-page-id",
      "title": "页面标题",
      "url": "https://www.notion.so/...",
      "kind": "page",
      "lastEditedAt": "2026-07-20T00:00:00Z"
    }
  ],
  "count": 1
}
```

- 最多返回 `limit` 条；标题、URL、类型和时间均做长度与格式约束。
- 不返回 access token、授权码、用户邮箱、完整页面正文、块内容、数据库行或未受限的提供方原始响应。
- `id` 仅用于同一工作流后续白名单能力的变量引用；首期不提供读取正文的 `notion.readPage` 能力。

## 执行与安全边界

1. 复用现有 `ExternalConnection`、加密凭据和 owner 隔离；不新增 OAuth 配置或数据库迁移。
2. 执行时才解密当前 owner 的凭据；凭据不得进入工作流图、运行输入、输出、SSE、错误消息或日志。
3. 调用 Notion 搜索 API 时设置短超时、结果上限和稳定 API 版本；提供方失败统一映射为可诊断的公开错误类别。
4. 未连接、已断开或凭据失效时，节点失败并提示重新连接；不得回退到其他用户或公共凭据。
5. 记录 owner 私有的调用审计事件，只保存能力名、成功/失败、时间和错误类别，不保存查询词、token 或 Notion 原始响应。
6. 继续复用工作流现有版本快照、工具 capability 校验、节点级事件和运行预算；该能力标记为只读，不计入写入能力预算。

## Web 体验

1. 工具节点的能力选择器显示“搜索 Notion”，并直接由 capability schema 生成查询词和结果数量字段。
2. 未连接时，节点配置与试运行结果给出明确的连接指引；不在工作流页面发起 OAuth 弹窗。
3. 运行详情只显示结果数量和受限结果元信息，不展示凭据或原始提供方响应。

## Notion 访问范围

1. OAuth 连接仅可读取用户在 Notion 中明确授权给 Valley 的页面范围，不默认读取整个工作区。
2. 工具页在已连接状态提供“添加授权页面”指引；用户在目标 Notion 页面的“Connections”中添加 Valley，避免已连接状态重复 OAuth 后直接回调却无法选页的误导。
3. 页面被作为项目根目录授权后，后续创建在其下的子页面继承访问范围；根目录外的独立页面由用户在 Notion 的“Connections”中添加 Valley。
4. 补充授权不改变只读、owner 隔离、凭据不落入工作流运行数据等安全边界。

## 非目标

- 不提供 Notion 页面正文读取、块读取、数据库写入、页面创建、归档、分享或权限变更。
- 不支持任意第三方 URL、任意 HTTP 请求或用户自带 provider 配置。
- 不引入通用连接器市场、凭据共享、多用户协作或发布审批。

## 验收标准

1. [x] 已连接用户可在工具节点中保存并运行 `notion.search`，得到不超过配置上限的受限结果。
2. [x] 未连接、已断开、失效凭据和 Notion 上游失败都有稳定且不泄露敏感数据的错误结果。
3. [x] 其他用户的 Notion 连接不能被读取或用于执行。
4. [x] 工作流保存、版本、运行事件与现有工具能力不回归。
5. [x] 使用真实 Notion 连接人工验收搜索结果和“只读、不产生 Notion 内容变更”。
6. [x] 补充授权项目根目录后，验证其新建子页面可搜索；根目录外页面仍需用户显式添加 Valley。
