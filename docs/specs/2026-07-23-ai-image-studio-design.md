# AI 图片创作工作台设计

> 状态：V2 协议适配、能力验证与画布体验修复已完成
> 日期：2026-07-23

## 产品目标

在 Web AI 工作台中新增独立的图片创作入口。用户可以绘制简笔画、摆放本地参考素材、选择受控提示词模板与输出比例，通过已启用的图片模型生成图片，并查看生成状态、历史结果、下载或保存到私有资源库。

完整创作体验落在独立页面；智能体后续只作为发起生成与跳转继续编辑的入口，不在首期把画布、素材和尺寸控制塞进聊天消息。

## 页面与交互

- 新增 `/workbench/images`，侧栏“创作”分组增加“AI 图片”。
- 桌面端采用工具区、中央画布、生成设置区与结果历史区；窄屏按纵向顺序排列。
- 画布首期支持选择、画笔、橡皮、撤销、清空、素材上传、素材移动、缩放和删除。
- 画笔粗细与素材大小使用共享 Slider；素材以长边为缩放基准并保持原始宽高比，切换画布比例时重新计算尺寸但不拉伸内容。
- 画布按选定比例导出为单张 PNG 参考图；输入参考图只用于当前生成请求，不写入生成历史或审计日志。
- 启用参考画布时，画布是主体数量、轮廓、姿态、取景、空间布局和相对比例的结构真源；模板和文字只补充外观、材质与渲染方式，冲突时以画布为准。
- 画面描述支持调用已有提示词助手：以当前输入为视觉需求，选择文本模型生成扩写结果，预览并确认后才替换当前内容；空输入不触发生成。
- 固定模板由服务端白名单控制，首期提供自由创作、草图成图、文章封面、产品展示、角色头像和毛毡玩具。
- 输出比例支持 `1:1`、`4:3`、`3:4`、`16:9`、`9:16`；清晰度支持 `1K`、`2K`，服务端映射为受控尺寸。
- 生成结果按 owner 私有展示，可下载并保存为私有资源；方形结果默认保存为头像，其他比例默认保存为壁纸。

## 生成状态与动效

服务端生成任务使用 `queued`、`running`、`succeeded`、`failed` 状态，并提供 `preparing`、`generating`、`storing`、`completed` 阶段。前端轮询任务状态，在画布上展示阶段文案、扫描线、颗粒与局部清晰化效果。

表现动效只表达真实任务阶段，不声称展示模型扩散中间帧。所有持续动画必须响应 `prefers-reduced-motion`；任务失败后停止动画并提供可重试信息。

## 服务端契约

- `GET /ai/image-presets`：返回受控模板与比例选项。
- `GET /ai/image-generations`：返回当前用户最近的生成记录。
- `POST /ai/image-generations`：创建异步生成任务，输入模型目录 ID、模板、用户提示词、比例、清晰度和最多三张 data URL 参考图。
- `GET /ai/image-generations/:generationId`：读取单条任务状态。
- `POST /ai/image-generations/:generationId/resource`：把成功结果登记为私有资源。

图片生成继续通过数据库模型目录解析 Provider。所有图片模型要求 `image_generation`；带参考图时额外要求 `reference_image`。“支持参考图”是生图模型的附加输入能力，不再作为独立的“编辑模型”呈现。

服务端通过统一图片生成接口接收提示词、受控尺寸和参考图，由 Provider Adapter 处理具体协议：

- SiliconFlow Adapter 使用兼容 JSON 生图协议和 Provider 对应尺寸字段。
- Amux Adapter 使用 OpenAI Images 生图协议；无参考图走 JSON `/images/generations`，有参考图走 multipart `/images/edits`。
- ARK Adapter 使用 ARK 图片 JSON 协议。
- Adapter 统一解析临时 URL、`b64_json` 等结果格式，调用方不再感知 Provider 字段差异。

模型目录通过 `image_protocol` 保存模型级图片协议，默认 `auto` 按 Provider 匹配，也允许兼容网关显式选择协议。模型目录同时保存管理员声明的能力；连接检测对声明能力执行真实最小调用，并记录未验证、部分验证、已验证或验证失败状态。检测可能产生费用，管理端必须明确提示且不得自动触发。Provider 密钥只从服务端环境变量读取。

## 数据与隐私

- 新增 `ai_image_generations`，保存 owner、模型、受控提示词、比例、状态、结果资源元数据与安全错误摘要。
- 不保存参考图 data URL、模型原始返回体或完整调用日志；AI 用量审计只记录调用元数据。
- 成功图片先转存 TOS，再进入历史；远端临时 URL 不直接作为长期资产。
- 保存到资源库时复用生成图片的 TOS 对象并记录 `resource_id`，不重复上传。

## 失败语义

- `400`：提示词、比例、清晰度、参考图或保存类型不合法。
- `404`：生成记录不存在或不属于当前用户。
- `409`：任务尚未成功或结果已经保存。
- `503`：模型、Provider 或 TOS 未配置。
- 异步上游失败写入任务 `failed`，前端读取安全错误信息；原始上游错误只进入服务端安全摘要与 AI 用量审计。
- 上游 HTTP 成功但结果格式无效属于响应解析失败；URL、Base64 等合法结果必须先由 Adapter 归一化，不能仅因缺少 URL 把成功调用误报为失败。

## 非目标

- 首期不做蒙版局部重绘、图层持久化、多人协作、批量组图、生成任务取消、真正的扩散预览帧或智能体图片附件。
- 首期不新增第三方画布依赖，使用浏览器 Canvas 完成有限编辑能力。
