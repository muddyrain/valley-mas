# P10：上下文副驾驶与对话式工作流生成规格

> 已由 P11 Graph v4 通用工作流内核取代。本文仅保留历史设计背景，不再代表当前工作流协议；当前规格见 `2026-07-18-general-workflow-v4-spec.md`。

## 目标

把 AI 工作台内分散的一次性生成能力收敛为按资产持久化的上下文副驾驶。副驾驶读取当前本地草稿、选中节点和安全运行摘要，经多轮澄清后只返回可审查的变更提案；保存、运行和发布仍由用户单独确认。

## 产品边界

- 工作台首页支持创建智能体或工作流；工作流编辑器右栏一级为“AI 协作 / 节点信息”，节点信息内二级为“配置 / 运行”；一级 Tab 只由用户切换，画布选中节点和运行失败定位不得自动改 Tab。智能体编辑器右栏为“在线调试 / AI 协作”。
- 每个资产可保存多条历史会话，会话按 `user + scope + targetId + sessionId` 隔离；仅向模型发送当前会话最近 20 条消息与最新草稿。
- AI 终态限定为回答、1–3 个澄清问题或完整候选草稿提案。
- 提案显示语义差异、写入风险和明确状态；基础草稿哈希变化后禁止应用旧提案。已应用提案保存基础草稿和候选哈希，只有当前草稿仍等于该候选时才允许撤销。
- AI 不能直接保存、运行、发布、访问任意 HTTP、执行代码或使用外部凭据。

## 工作流协议

- Graph v2 保持最多 8 节点的线性 DAG。
- Graph v3 最多 12 节点、一个条件节点、两个独立分支、最多一个 LLM、一个封面生成节点和一个博客草稿写入节点。
- 条件只支持 `equals`、`notEquals`、`contains`、`isEmpty`、`greaterThan`、`lessThan`，左值必须是开始输入或上游输出引用。
- Graph v3 开放 `content.search`、`condition` 和 `image.generateCover`。封面节点可用受控 `runIf` 绑定上游 boolean；条件为假时记录 `skipped`、不调用生图或上传，并继续执行后续节点。未命中分支同样不能执行模型或写入节点。
- 能力目录由服务端注册表生成，前端和 AI 只消费服务端声明为开放的能力。

## 数据与接口

- `AIWorkbenchCopilotSession`：资产级 owner 私有历史会话，同一资产允许多条。
- `AIWorkbenchCopilotMessage`：用户与助手消息，不复制到 AI usage 日志。
- `AIWorkbenchChangeProposal`：基础哈希、基础草稿、完整候选草稿、候选哈希、服务端语义差异及状态；撤销后为 `reverted`。
- 接口：当前会话查询、历史会话列表、新建会话、消息 SSE、提案接受/拒绝/撤销、工作流能力目录。
- 工作流保存带可选基础哈希；图更新、Workflow 与 AI App 草稿版本同步在同一事务内完成。

## 安全与错误

- 副驾驶使用 `internal/ai/agent`，只注册当前资产、能力目录、安全运行摘要和候选校验四个只读工具。
- 结构化结果允许一次受控 JSON 修复。
- 缺少 ARK 配置映射为 `503` 业务码，上游或结构化失败映射为 `502`，无效上下文映射为 `400`，并发冲突映射为 `409`。
- `AI_WORKBENCH_COPILOT_ENABLED=false` 时保留旧 AI 创建入口。

## 验收标准

- 从空白需求经过澄清生成并应用工作流；明确点名节点优先于当前选择，现有画布可按节点名称或当前节点持续修改。
- 刷新后历史会话和提案仍存在，可点击切换，跨用户、跨资产不串数据。
- 手工修改后旧提案不能覆盖；应用提案后仍不自动运行或发布，且仅在没有后续手工编辑时可一键撤销。
- 分支未命中时节点显示“已跳过”，对应模型和写入执行器调用次数为零。
- Graph v2 历史运行与已发布版本行为保持不变。
