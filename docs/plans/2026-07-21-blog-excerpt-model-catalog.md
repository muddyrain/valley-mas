# 博客摘要模型目录迁移实施计划

> 状态：已实施，待真实 Provider 验收。规格见 [博客摘要模型目录迁移设计](../specs/2026-07-21-blog-excerpt-model-catalog-design.md)。

## 范围

把博客编辑器的单次“AI 截取摘要”从 `ARK_TEXT_MODEL` 迁移到数据库模型目录；不迁移批量 Markdown 导入、封面生成、视觉、生图或向量能力。

## 步骤

1. [x] 将模型选择器抽为非工作流专属组件，供博客编辑器按 `text` 能力加载模型目录。
2. [x] 扩展博客摘要请求与前端 API，要求传递目录模型 ID；未选择时保持按钮不可提交。
3. [x] 服务端校验已启用文本模型，以其 Provider 的兼容客户端生成摘要，并将成功或失败写入 AI 用量审计。
4. [x] 补充服务端定向回归并运行 Go 全量测试、Web 类型检查。
5. [ ] 在真实 Provider 环境做一次博客摘要与审计人工验收；Harness 与计划索引检查受本机 WSL2 虚拟磁盘缺失阻断。

## 完成条件

操作者能在博客编辑器选择模型，服务端不会接受任意模型名或未启用模型；一次实际调用的输出与 AI 调用审计均可追溯到所选 Provider 和模型。
