## 已实施

- 服务端新增会话、消息、提案模型与迁移，完成 owner 隔离的多会话列表/切换、SSE 事件和提案接受/拒绝/安全撤销。
- 副驾驶接入通用 Agent loop 与四个只读工具，保留一次结构化修复；旧工作流生成入口改为读取能力目录。
- 服务端新增能力目录、Graph v3 条件分支、`content.search`、`skipped` 调度与未选分支副作用隔离。
- Graph v3 新增 `image.generateCover`，复用 ARK 生图和 TOS 上传；节点级 `runIf` 只接受上游 boolean，条件不满足时跳过生图并继续后续节点。
- 工作流保存增加基础哈希冲突检查，并把 Workflow 更新和 AI App 草稿快照同步放入同一事务。
- Web 新增共享 `WorkbenchCopilot`，接入工作台、工作流与智能体编辑器；字段级 AI 入口切换到同一协作标签。
- `WorkbenchCopilot` 提供历史会话选择和新建会话；已应用提案在当前草稿仍匹配候选哈希时显示“撤销这次改动”。工作流把节点名称映射作为安全上下文发送给副驾驶，明确点名节点优先于当前选择。
- 移动端共享全高 `MobileCopilotSheet`；工作流画布在窄屏隐藏节点库和桌面右栏，一级“AI 协作 / 节点信息”与节点内二级“配置 / 运行”复用同一移动工作区。
- 前端节点库消费服务端能力清单，支持受控条件、站内内容搜索和跳过状态。
- 灰度开关为 `AI_WORKBENCH_COPILOT_ENABLED`；关闭时工作台回退旧 AI 创建弹窗。

## 自动化覆盖

- Graph v3 选择分支、未选写入节点零调用、嵌套条件拒绝。
- `content.search` 执行器输入与安全结果。
- 可选封面在未勾选时零次调用生成器且下游继续，勾选时只调用一次并输出安全封面对象。
- 副驾驶历史会话 owner 隔离、指定会话读取、提案仅 owner 可处理；撤销必须匹配候选哈希。
- “生成摘要节点后插入条件封面”回归覆盖：即使当前选中 `create-draft`，明确点名仍必须在摘要节点后重连，错误插入被拒绝。
- 工作流基础哈希冲突返回 `409` 业务码。

## 验证结果

- 通过：Web 类型检查、生产构建、AI 工作台 SSE 回归、全量 Go 测试、定向 Biome、编码检查和 `git diff --check`。
