# AI 协作规范

> **给所有 AI 助手的重要提示**

---

## 🤖 工作流程规范

### 1. ❌ 不要主动初始化数据

当需要初始化数据时：
- ✅ **只提示用户手动运行命令**
- ❌ 不要自动执行 `curl http://localhost:8080/init-data`
- ❌ 不要自动执行 `Invoke-RestMethod`

**示例提示**：
```
⚠️ 需要初始化数据！

请运行以下命令：
curl http://localhost:8080/init-data

或者访问：http://localhost:8080/init-data
```

---

### 2. ❌ 不要主动创建总结文档

完成功能后：
- ✅ **先询问用户是否需要总结文档**
- ❌ 不要自动创建 `summary_xxx.md`
- ❌ 不要自动创建 `guide_xxx.md`

**示例询问**：
```
功能已完成！

需要我创建总结文档吗？
- [ ] 功能实现文档
- [ ] 使用指南
- [ ] API 文档
```

---

### 3. ✅ 应该做的事情

#### 代码实现
- ✅ 直接实现功能代码
- ✅ 添加必要的注释
- ✅ 添加 Swagger 文档注释
- ✅ 修复编译错误

#### 简要说明
- ✅ 简短说明做了什么
- ✅ 提示如何测试
- ✅ 提示下一步计划

#### 提交代码
- ✅ 提示用户提交代码
- ✅ 建议 commit message

---

## 📝 文档创建规则

### 何时需要文档？

**只在以下情况创建文档**：
1. 用户明确要求
2. 重大功能完成后询问
3. 复杂配置需要说明

### 文档类型

- **功能文档** - 详细说明功能实现
- **使用指南** - 教用户如何使用
- **API 文档** - Swagger 注释 + README
- **总结文档** - 阶段性总结

---

## 🎯 高效协作原则

1. **少说废话，多写代码**
2. **问题驱动，按需文档**
3. **快速迭代，及时反馈**
4. **保持简洁，避免冗余**
