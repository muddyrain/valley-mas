# 卡片系统一致性

所有卡片相关任务都可以用这个技能来兜底判断。

## 核心原则

该统一的地方统一，该分开的地方分开。

## 规则

1. 博客、图文、资源卡片可以共享骨架，但不能被强行压成同一种预览模型。
2. 只要有真实封面或真实生成图，就优先展示真实内容，而不是模拟预览。
3. 信息层级保持稳定：身份信息、标题、摘要、参数、操作区。
4. 创作空间卡片可以暴露管理按钮，公开卡片不一定需要。
5. 如果抽共享组件，要确认每种变体仍然符合真实产品需求。

## 设计检查

- 预览是否真实
- 参数区位置是否稳定
- 操作按钮是否对齐且大小一致
- 桌面端和移动端是否都还能成立
