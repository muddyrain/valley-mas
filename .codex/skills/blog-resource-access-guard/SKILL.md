# 博客资源访问护栏

只要任务碰到博客或资源访问逻辑，就用这个技能。

## 规则

1. 区分“普通访问权限”和“作者管理访问权限”。
2. 如果作者本人可以访问私密内容，公共详情路由通常也需要 optional auth。
3. 如果公共接口可能拿不到作者自己的私密内容，要有创作者兜底路径。
4. 卡片、详情页、错误页、空状态都要保留带来源的返回行为。
5. “404 + 返回错页”要当产品 bug 看，而不是只当接口 bug。

## 后端检查

- 路由是否接了可选鉴权
- 可见范围判断和状态判断是否分开
- 私密内容作者自访问是否有兜底读取

## 前端检查

- 卡片跳转是否带来源状态
- 详情页返回是否识别入口上下文
- 错误态返回是否一致
- 创作空间里的入口是否特殊处理
