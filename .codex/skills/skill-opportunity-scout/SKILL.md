# 技能机会侦察

当开发过程中出现“这类事情以后还会反复发生”的信号时，使用这个技能。

## 目的

让 AI 不只是被动使用已有 skills，也能主动判断：

- 这次改动里有没有一套稳定规则正在形成
- 这套规则以后是否值得沉淀成一个 skill
- 是不是已经到了“继续靠临时口头约定不够”的程度

## 什么时候应该建议新增 skill

- 同一类判断已经重复出现 2 到 3 次以上
- 某条产品规则开始变得稳定
- 某个页面或模块有了固定的开发习惯
- 某类问题每次都要重复解释一遍
- 某条验证链路已经固定下来
- 某类“不要这样做”的坑已经出现过不止一次
- 某个功能已经形成自己独立的产品心智

## 什么样的内容适合沉淀成 skill

- 产品规则
- 交互约定
- 组件使用边界
- 发布或部署检查
- 编码安全规则
- 文案风格
- 特定模块的开发流程

## 什么样的内容不适合

- 只出现过一次的小修小补
- 还在频繁摇摆、没有稳定结论的方案
- 只是某个临时任务的上下文
- 不足以复用的纯实现细节

## 工作规则

1. 在开发过程中，如果发现某类规则正在稳定形成，要主动提醒用户“这可能值得抽成 skill”。  
2. 建议时要说清楚为什么值得抽，不要只说“可以做个 skill”。  
3. 如果用户同意，再创建 skill；如果用户没同意，先作为建议保留。  
4. 如果已有 skill 已经能覆盖，不要重复造新 skill。  
5. 如果新的模式会影响已有 skill，也要联动检查 [$skill-sync-guard](/D:/my-code/valley-mas/.codex/skills/skill-sync-guard/SKILL.md)。  

## 推荐表达

- “这类图文编辑规则已经连续改了几轮，建议沉淀成一个独立 skill，后续 AI 会稳定很多。”  
- “这不是一次性改动了，已经形成固定的回跳和权限判断，适合抽成 access guard 类的 skill。”  

## 最终说明

如果这次明确发现了新的 skill 机会，最终回复里最好补一句：

- 是否发现新的 skill 机会
- 是建议新增，还是已经落地新增
