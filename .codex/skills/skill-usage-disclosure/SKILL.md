# 技能使用透明化

当任务中使用了一个或多个 skills 时，使用这个技能。

## 目的

让协作过程更透明，避免用户不知道 AI 这次依据了哪些本地 skill 在做判断。

## 工作规则

1. 只要本回合触发了任何 skill，就要明确告诉用户用了哪些 skills。
2. 说明尽量简短，通常一行即可。
3. 最好在开始阶段说明，而不是等到最后才补一句。
4. 说明内容至少包括：
   - 使用了哪个 skill
   - 为什么这次会用它
5. 如果这次没有使用任何 skill，就不要硬报。

## 推荐表达

- “这次我会用 `creator-space-ux` 和 `encoding-guard`，前者处理图文编辑逻辑，后者确保中文文案不会出编码问题。”

## 注意

- 不要把 skill 说明写成很长的前置汇报。
- 透明化是为了增强协作感，不是增加阅读负担。
- 如果多个 skill 同时触发，优先说最关键的 1 到 3 个。
