# 品牌主题护栏

当任务会影响全站可感知的视觉主题，而不只是某一处局部样式时，使用这个 skill。

## 当前主题方向

- 主主题：暖金、奶油白、浅蓝辅助
- 品牌氛围：温润、干净、内容型，而不是强平台感或科技霓虹感
- 紫色角色：保留少量记忆点或功能性入口，不再主导整站主体视觉

## 核心规则

1. 顶部导航、主要按钮、品牌标识和大面积容器优先跟随暖金主题。
2. 紫色可以保留在 AI / TTS / 次级功能提醒等小范围点缀里，但不要和主体区域形成对冲。
3. 新页面如果已经采用暖金内容主题，头部和入口按钮也应同步跟上，不要再保留旧紫色主基调。
4. 视觉统一优先级高于单页“局部好看”：页面之间要像同一个项目。
5. 对外文案不要解释设计理由，只呈现最终产品表达；需要时联动 [$public-copy-boundary](/D:/my-code/valley-mas/.codex/skills/public-copy-boundary/SKILL.md)。

## 适用场景

- Header / Footer / 全站导航改造
- 首页与内页风格统一
- 全局按钮、主题变量、品牌色更新
- 新页面承接已有暖金主题
- 怀疑“页面主体和头部像两套设计语言”时

## 不适用场景

- 纯逻辑改动
- 单个图标的微调
- 与全站主题无关的局部组件修复

## 工作顺序

1. 先判断这次改动是否会被用户明显感知为“主题变化”。
2. 优先检查头部、主按钮、页面大容器、品牌字是否统一。
3. 如需改变量，优先从全局色板和高频组件入手，而不是每个页面逐个硬改。
4. 如页面里仍需保留紫色，确认它只是点缀而不是主视觉。
5. 改完后至少跑一轮前端编译；涉及中文 UI 时再跑 [$encoding-guard](/D:/my-code/valley-mas/.codex/skills/encoding-guard/SKILL.md)。

## 最终说明

最终回复里应说明：

- 这次有没有真正把主题统一动作落到代码里
- 哪些高频区域发生了可感知变化
- 紫色被保留在什么范围，避免误解成“完全移除品牌记忆”
