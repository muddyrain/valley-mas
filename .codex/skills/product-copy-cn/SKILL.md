# 中文产品文案

当任务涉及中文 UI 文案时，使用这个技能。

## 文案规则

1. 优先使用产品自身语境，不要随意借外部平台做参照，除非用户明确要求。
2. 按钮文案保持短、直接、可执行。
3. 辅助说明应该解释功能做什么，而不是教育用户。
4. 避免出现占位感、模板感或泛 AI 营销口吻。
5. 如果一段文案会出现在创作者态和公开态，要确认它在两个场景都成立，或主动拆分。

## 推荐默认写法

- 按钮：直接动词短语
- 状态：短、稳定、可扫读
- 说明：一句话、具体、无废话
- 空状态：说明缺什么，以及下一步能做什么

## 校验

- 在实际界面上下文里重读一遍文案。
- 编辑中文后运行 [$encoding-guard](/D:/my-code/valley-mas/.codex/skills/encoding-guard/SKILL.md)。
- 如果用户要求某种产品语气，最终说明里要提一下已按该语气调整。
