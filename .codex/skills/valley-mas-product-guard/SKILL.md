# 产品行为护栏

当任务会改变产品行为，而不只是改代码结构时，使用这个技能。

## 重点关注

- 创作空间、博客、资源、图文这几条主链路
- `private / shared / public` 可见范围行为
- 从列表进入详情，再返回原上下文的链路
- 卡片摘要与详情页真实内容是否一致
- 空状态、加载态、404、兜底态是否合理

## 工作规则

1. 如果一个功能同时有创作者态和访客态，至少同时检查这两种视角。
2. 把“作者本人是否还能访问自己的内容”当成必查项。
3. 优先使用带来源上下文的返回逻辑，不要写死回列表页。
4. 如果卡片展示了预览内容，要确认它不会误导用户。
5. 推理和界面里都要把状态、可见范围、发布状态分开处理。

## 校验

- 运行最贴近改动面的 TypeScript 或 Go 校验。
- 如果改动涉及中文 UI 文案，再运行 [$encoding-guard](/D:/my-code/valley-mas/.codex/skills/encoding-guard/SKILL.md)。
- 最终说明里点出有哪些用户可感知行为发生了变化。
