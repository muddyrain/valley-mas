# LifeTrace Agent Design Brief

## 1. 产品定位

LifeTrace Agent 是一个以 Agent 为主导的个人生活记录 App。它不是让用户手动维护多个记录工具，而是把日程、倒数日、生日、资产、订阅、习惯、日记、照片、计划清单和生活碎片统一收纳到一个地方，再由多个智能体自动整理、提醒、复盘和给出建议。

一句话定位：

> 一个地方收纳所有生活碎片，由 AI Agent 帮你整理、计算、提醒和复盘。

## 2. 参考图可借鉴方向

参考图里的核心价值不是某个单独页面，而是一套“生活碎片容器”的产品范式，可以借鉴这些方向：

- 统一入口：不需要下载多个记录 App，所有生活记录都从一个地方进入。
- 模板丰富：倒数日、生日、资产、订阅、计划、打卡、日记、照片墙等都有快速模板。
- 轻量数据感：大数字、倒计时、次数、日均成本、剩余天数一眼可读。
- 卡片网格：两列卡片适合移动端快速浏览，结构清晰，操作负担低。
- 详情抽屉：点开卡片后进入半屏或整屏详情，展示字段、提醒、编辑入口。
- 图标图库：内置大量图标和贴纸，降低用户创建记录时的阻力。
- 桌面组件：记录不仅停留在 App 内，还可以成为桌面小组件。
- 图片处理：上传图片后可去背景、生成贴纸，用于生活记录和个性化展示。

需要升级的地方：

- 从“用户自己分类记录”升级为“Agent 主动理解并整理”。
- 从“模板列表”升级为“Agent 根据一句话自动选择模板”。
- 从“静态统计”升级为“智能体复盘、提醒和建议”。
- 从“记录工具”升级为“个人生活操作系统”。

## 3. Agent 设计

App 应该让 Agent 成为主角，而不是隐藏在角落里的聊天按钮。建议设计 5 个核心 Agent：

### 3.1 收纳 Agent

职责：接收用户的一句话、语音、图片或截图，自动拆分成结构化记录。

示例：

用户输入：“周末看了两集剧，打了羽毛球，iCloud 快续费了，记得给爸爸准备生日礼物。”

收纳 Agent 自动拆成：

- 追剧记录
- 运动打卡
- 订阅续费提醒
- 生日/纪念日任务

### 3.2 计划 Agent

职责：把愿望、清单、旅行、观影、健身等目标拆成可执行计划。

示例：

- “6 月前看完这部 36 集电视剧”
- “今年想去北京玩”
- “想换手机但不知道什么时候合适”

### 3.3 资产 Agent

职责：管理设备、生活用品、订阅会员、保修、折旧、日均成本和更换建议。

展示重点：

- 当前估值
- 已使用多久
- 日均成本
- 下次续费/保修到期
- 最佳更换窗口

### 3.4 习惯 Agent

职责：管理打卡、次数、连续天数、频率趋势和提醒。

记录类型：

- 喝水
- 健身
- 打球
- 打扫卫生
- 看剧
- 阅读
- 睡眠

### 3.5 复盘 Agent

职责：每天、每周、每月生成生活报告，告诉用户最近发生了什么、哪些计划延迟了、哪些资产值得关注。

输出形式：

- 本周摘要
- 重要提醒
- 趋势变化
- 下周建议
- 可执行行动清单

## 4. 信息架构

建议底部导航保留 5 个入口：

- 今日：Agent 总览、提醒、今日计划、一句话记录。
- 收纳：所有记录类型、模板库、快速创建。
- Agent：智能体工作台，查看 Agent 正在整理什么、建议什么。
- 资产：设备、订阅、日均成本、续费和更换建议。
- 我的：小组件、图标库、数据统计、偏好设置。

## 5. 核心页面设计

请优先生成移动端高保真设计稿，建议尺寸为 iPhone 15/16 竖屏。

### 5.1 今日 Agent 首页

目标：第一眼看出这是一个由 Agent 驱动的生活中枢。

必须包含：

- 顶部问候语、日期和设置入口。
- 一个明显的一句话输入区域，文案为“告诉 Agent 你刚刚做了什么，或接下来想做什么”。
- Agent 今日摘要，例如“今天有 4 件事需要你留意”。
- 四个重点卡片：倒数日、习惯、资产、订阅。
- Agent 建议条，例如“iCloud 还有 7 天续费，建议检查是否还需要 2TB 套餐”。

### 5.2 Agent 自动收纳页

目标：展示 Agent 如何把一句话拆成多个生活记录。

必须包含：

- 用户输入的一句话。
- Agent 解析过程状态，例如“识别中、分类中、生成提醒中”。
- 拆分结果卡片：运动、观影、订阅、生日提醒。
- 每张卡片显示分类、时间、图标、可编辑字段和确认按钮。
- 一个“全部收纳”主按钮。

### 5.3 生活碎片模板库

目标：参考图里的模板入口，但强调 Agent 可以自动选择模板。

必须包含：

- 搜索框。
- 横向分类 Tab：全部、倒数日、习惯、资产、订阅、日记、计划、照片。
- 模板列表或网格。
- 每个模板包含线性图标、标题、简短说明。
- 顶部提示：“也可以直接告诉 Agent，它会自动帮你选择模板。”

### 5.4 资产 Agent 页

目标：把资产记录做成生活化设备管家，不是财务软件。

必须包含：

- 顶部汇总：资产总值、资产数量、日均成本。
- 两列资产卡片：手机、电脑、平板、耳机、家电、会员订阅。
- 资产详情页或抽屉：购买日期、购买价格、已使用多久、日均成本、当前状态、保修/续费、Agent 建议。
- 示例建议：“这台手机已使用 3 年 8 个月，日均成本已降到较低水平，适合继续使用或等待下一代发布后更换。”

### 5.5 计划 Agent 页

目标：展示愿望、旅行、观影、清单如何变成可执行计划。

必须包含：

- 计划分组：旅行、观影、健身、购物、年度计划。
- 计划卡片显示总任务数、已完成数、下一步行动。
- 详情页为清单结构，支持完成状态、日期、备注。
- Agent 节奏建议，例如“如果想在 6 月 10 日前看完，还需要每周看 6 集。”

### 5.6 习惯与打卡页

目标：保留参考图里的打卡网格，但增强趋势和 Agent 提醒。

必须包含：

- 两列习惯卡片。
- 次数、最近一次、连续天数或本周完成情况。
- 小型点阵或迷你趋势图。
- 快速打卡按钮。
- Agent 提醒：“你已经 6 天没有打扫卫生了，要不要今晚安排 20 分钟？”

### 5.7 日记与照片墙页

目标：把随手记录、心情、照片、碎碎念统一展示。

必须包含：

- 时间流日记。
- 图片九宫格或照片墙。
- 标签：生活、工作、观影、旅行、心情。
- 数据统计入口：记录篇数、字数、活跃天数、内容分类。
- 支持图片贴纸感，但不要过度幼稚。

### 5.8 复盘 Agent 周报页

目标：体现 AI 的长期价值。

必须包含：

- 本周生活摘要。
- 完成事项、延迟事项、资产提醒、订阅提醒。
- 3 条 Agent 洞察。
- 3 条下周建议。
- 一个“和 Agent 复盘”按钮。

## 6. 视觉规范

整体风格：

- 高级、克制、清晰、轻量科技感。
- 可以参考图里的黑白强对比和大数字，但不要完全复刻厚重黑色海报风。
- App 内部以浅色为主，局部使用黑色强调操作和选中态。
- 页面要像真实可用的产品，而不是营销展示图。

颜色：

- 背景：#F4F4F6、#FAFAF8、#FFFFFF。
- 主文字：#111111。
- 次文字：#747474。
- 强调黑：#1F1F1F。
- Agent 强调色：柔和青色 #70D7E8 或鼠尾草绿 #8FB996，只作为少量状态色。
- 警示/到期：柔和橙色 #E7A75F。

字体：

- 中文使用现代、圆润、清晰的无衬线字体。
- 标题要有力量，但 App 内不要使用过度夸张的海报字体。
- 数字要大、清楚、有节奏。

组件：

- 两列卡片网格。
- 胶囊 Tab。
- 黑色圆形悬浮按钮。
- 详情抽屉。
- 搜索框。
- 轻量统计卡。
- 图标选择器。
- Agent 消息/建议条。

图标与插画：

- 使用统一线性图标或轻量彩色贴纸图标。
- 不使用 emoji 作为正式图标。
- 图标可以有生活感，但保持统一风格。

动效建议：

- Agent 解析时使用轻微呼吸动画或步骤进度。
- 卡片确认收纳时有轻微吸附/归档动效。
- 底部抽屉弹出要顺滑。
- 打卡完成时有短暂状态反馈。

## 7. 设计取舍

第一版不要做得太大。完整设计稿可以展示 8 个页面，但产品 MVP 建议先实现 4 个核心闭环：

1. 一句话交给 Agent。
2. Agent 自动拆分记录。
3. 今日页聚合提醒和卡片。
4. 周报页给出复盘价值。

资产、订阅、图标库、照片贴纸、小组件都适合作为第二阶段增强能力。
