# Correctness Review

- [ ] null、undefined、空数组、空字符串、零值和缺失配置有明确行为。
- [ ] 创建、更新、删除和重试操作满足预期幂等性，不会重复扣减、重复发送或重复创建。
- [ ] 分页、筛选、排序和刷新使用一致的状态真源，变更条件后页码和缓存正确重置。
- [ ] 异步错误没有被空 catch 或未处理 Promise 吞掉，用户状态与日志状态保持一致。
- [ ] 时间逻辑明确时区、日期边界和过期语义，不混用本地时间与 UTC 假设。
- [ ] 共享类型、接口或格式变化检查所有消费者，并保留必要的向后兼容路径。
- [ ] 测试覆盖需求中的成功、失败和关键边界，不只断言实现细节或 mock 调用次数。
