# React UI Review

- [ ] effect 中的事件、订阅、定时器、observer 和请求在卸载或依赖变化时正确清理。
- [ ] 并发请求、搜索和路由切换不会让较早响应覆盖较新状态；需要时使用 AbortController 或请求序号。
- [ ] URL query、浏览器前进后退、筛选、分页和页面本地状态保持一致。
- [ ] loading、空状态、错误状态和成功状态不会互相覆盖，也不会让用户重复提交。
- [ ] 按钮、输入、弹窗和自定义交互具备语义元素、键盘路径、焦点管理和可读标签。
- [ ] 文案属于用户任务语境，不把开发者分析、实现说明或操作教程直接写进界面。
- [ ] 响应式、动画、Canvas、Three.js、拖拽、滚动和布局结论有运行时证据；静态读码不能单独标记视觉验收通过。
