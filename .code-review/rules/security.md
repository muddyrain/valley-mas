# Security Review

- [ ] 源码、示例配置、日志和响应体没有真实密钥、token、密码或外部账号凭据。
- [ ] 服务端权限由认证/授权中间件和业务边界执行，不能只依赖前端隐藏入口。
- [ ] 外部输入经过类型、长度、枚举或格式校验；错误响应不泄露内部堆栈和敏感字段。
- [ ] 用户可控 URL 不会未经协议、主机和重定向限制直接访问内网或云元数据地址。
- [ ] 富文本、Markdown 或 HTML 输出在进入 `dangerouslySetInnerHTML` 前经过可信净化。
- [ ] 上传校验同时考虑文件大小、类型、扩展名、内容和存储路径，不能只信任客户端声明。
- [ ] 日志不会记录 Authorization、Cookie、邮箱授权码、AI provider key 或完整个人隐私数据。
