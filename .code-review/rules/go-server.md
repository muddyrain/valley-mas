# Go Server Review

- [ ] Gin handler 只负责协议适配，复杂业务优先复用现有 service/领域模块，不在 handler 中复制逻辑。
- [ ] GORM 多步写操作在需要原子性时使用事务，并正确返回事务内错误。
- [ ] 数据查询包含必要的用户、家庭空间或管理员作用域，避免只按资源 ID 查询造成越权。
- [ ] 外部 HTTP、AI、邮件、存储和数据库操作传递 context，并设置合理超时或复用已有超时配置。
- [ ] 上游错误映射到稳定 API 错误码，日志保留诊断信息但响应不暴露内部实现。
- [ ] 新字段、约束和迁移考虑已有数据、零值、回滚和多版本部署期间的兼容性。
- [ ] goroutine、channel、ticker 和 response body 在所有返回路径上得到释放或关闭。
