# Valley MAS Code Review 规则

本目录提供按改动风险加载的 review 清单，不要求固定数量或固定角色的审查 Agent。

## 加载规则

| 改动范围 | 加载规则 |
|---|---|
| 所有代码改动 | `rules/security.md`、`rules/correctness.md` |
| `server/**` | 追加 `rules/go-server.md` |
| React、样式或用户可见交互 | 追加 `rules/react-ui.md` |

## 输出要求

1. Findings 优先，按严重程度排序，并附文件和行号。
2. 只报告能说明具体失败路径、回归风险或维护风险的问题。
3. 不把格式偏好、未被需求支持的重构或泛泛建议升级成阻塞项。
4. 没有发现问题时明确说明，并列出尚未运行的测试或残余风险。
5. 同一类评论重复出现两次以上且可机械判断时，优先新增测试、Biome 规则或 Harness 检查，不继续扩写提示词。

## 严重程度

- `P0`：安全漏洞、数据丢失、不可恢复的生产事故。
- `P1`：确定的逻辑错误、鉴权绕过、崩溃、严重回归。
- `P2`：边界缺陷、竞态、兼容性或缺失测试导致的中等风险。
- `P3`：非阻塞的可维护性或性能建议。
